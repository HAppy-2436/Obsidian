## 六个核心方法（HOW？）

1.  **Write clear instructions** 写出清晰的指令
2.  **Provide reference text** 提供参考文本
3.  **Split complex tasks into simpler subtasks** 将复杂的任务拆分为更简单的子任务
4.  **Give the model time to "think"** 给模型时间"思考"
5.  **Use external tools** 使用外部工具
6.  **Test changes systematically** 系统地测试变更

---

### 1. 写出清晰的指令

*   **1.1 塑造角色**
    *   你是一名`XXX` / 请以`XXX`的身份…
*   **1.2 明确目标**
    *   总结… / 分析… / 讲解…
*   **1.3 提供上下文**
    *   背景是… / `XXX`指的是…
*   **1.4 约束输出**
    *   文体是… / 字数为… / 输出格式为…

### 2. 提供参考文本

*   隐含的形式规则
*   反直觉或自定义的标签映射
*   特定领域的缩写或编码
*   依赖于上下文的分类（无常规线索）
*   需要从数据中推导数学或逻辑规则
*   特定格式或结构约束
*   **可使用技巧**：少样本提示

### 3. 将复杂的任务拆分为更简单的子任务

*   拆分为多个步骤
*   使用**链式思考（CoT）**
*   规划路径，分步执行

### 4. 给模型时间"思考"

*   让模型逐步推理
*   使用"先思考，再回答"类提示词

### 5. 使用外部工具

*   例如：ASR、OCR、RAG、Python等

### 6. 系统地测试变更

*   针对内容、理解、表达、对话层面进行测试与优化

## 常见问题与解决方案

### 内容错误（Content Errors）
*   **常见问题**：幻觉（编造不存在的事实）、错误事实、数学/逻辑推理错误、推理链断裂
*   **解决方案**：让模型自检与纠错、使用替代方案进行交叉验证、要求模型逐步解释其推理

### 理解错误（Comprehension Errors）
*   **常见问题**：答非所问、误解专业术语、执行不完整、错误解读上下文
*   **解决方案**：让模型复述任务、指出误解点并要求重做、要求补全缺失部分、进行语义纠偏

### 表达问题（Expression Issues）
*   **常见问题**：回答结构与期望不同、内容空洞、推理不充分、风格不一致
*   **解决方案**：让模型按指定结构重排、要求补充深度内容、格式化重做、强制转换风格

### 对话层面问题（Dialogue-Level Issues）
*   **常见问题**：上下文丢失、自相矛盾、过度迎合、无法维持设定
*   **解决方案**：让模型总结上下文、要求解释矛盾点、中断迎合行为、要求恢复设定
