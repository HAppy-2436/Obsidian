# 引入
最近刷到了一个有趣的小项目，可以统计电脑上的chromium内核数量，确实给我带来了一些震撼，他让我留意到了这个问题：
# 你电脑里到底有多少chromium
——电脑上到底有多少chromium。而我了解到了一个试图打破这个局面的新兴技术
# Electrobun
这就是我今天为大家介绍的——Electrobun，官方的描述是“用TypeScript构建极快、极小、跨平台的桌面应用”，这里的“极快”、“极小”便和我们熟悉的electron形成了鲜明的对比
# Star History
作为一个新兴的桌面应用框架，当前10k+的star数，以及26年以来关注度的飙升，都印证了他切实的在试图解决当下桌面应用开发的痛点
# 第一部分
那么他针对的问题，就是桌面应用开发背负的历史包袱。
# 为什么要重复造轮子
站在如今的视角审视，难以理解为什么桌面应用要各自打包一个浏览器内核，为什么要重复造轮子，但事实上，现在看起来的问题，是曾经问题的解决方案
# 2013年
时间回到13年前，那时主流的系统自带的webview功能都极其有限，支持现代的HTML5、CSS3都是天方夜谭，依赖系统webview开发桌面应用完全不可行。
# 唯一的出路
于是，开源、跨平台一致性优异、性能强悍且支持最新web标准的chromium成为了当时时代下的最优解
# Node.js
浏览器不具备的读写文件、调用系统API、创建窗口等等系统能力，Nodejs也填上了最后一片拼图，让 JavaScript 能够跳出浏览器，直接与操作系统进行交互
# Electron
二者结合，便诞生了统治桌面应用开发至今的Electron框架
# 应用
我们熟知的诸多现代跨平台桌面软件，都是基于此的。但是，强大泛用的背后，一个空白的应用安装包却都要占用150MB起步
# 系统webview
但2026年的今天，我们发现，当年那个迫使我们必须打包 Chromium 的核心矛盾，已经发生了变化。现在主流的操作系统自己，已经为我们准备好了现代、高效的 WebView。意味着我们不再需要自己‘随身携带’一个上百兆的 Chromium 了
