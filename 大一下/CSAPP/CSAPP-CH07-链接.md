# CH07 链接

> 一句话总结：**链接器把多个 .o 和库合在一起**，主要干两件事：**符号解析**（同名全局符号按强弱规则选一个）和**重定位**（把占位地址改成真实地址）。库分静态（拷进可执行文件）和动态（运行时加载）。

> 相关：[[CSAPP-CH03-机器级表示]]、[[CSAPP全部测验题目]]

---

## 1. ELF 目标文件结构

### 1.1 三种目标文件

| 类型 | 后缀 | 特点 |
|------|------|------|
| **可重定位** | .o | 可被链接，地址未确定 |
| **可执行** | (无后缀) | 可直接运行，地址已定 |
| **共享（动态库）** | .so | 运行时被动态链接器加载 |

### 1.2 ELF 节（必背）

| 节 | 内容 |
|----|------|
| **`.text`** | **程序代码（机器指令）** |
| **`.rodata`** | 只读数据（字符串字面量、switch 表） |
| **`.data`** | **已初始化的全局/静态变量** |
| **`.bss`** | 未初始化的全局/静态变量（不占磁盘空间，运行时才分配） |
| **`.symtab`** | **符号表** |
| **`.rel.text`** | 代码段的重定位条目 |
| **`.rel.data`** | 数据段的重定位条目 |
| **`.debug`** | 调试符号表（-g 才有） |
| **`.strtab`** | 字符串表（符号名、节名） |
| **`.line`** | 行号表（调试用） |

⚠️ **易混点**：
- `.bss` 在磁盘上**不占空间**，只是占位符（运行时 loader 给 0）
- `.data` 才在文件里真实存数据
- **重定位条目**有 `.rel.text` 和 `.rel.data` 两个

📝 **真题**：
- `.text` 存**程序代码**（A）
- 符号表在 **`.symtab`**（D）
- 看 ELF 工具 = **readelf**（C）

## 2. 符号表

### 2.1 符号种类

| 类型 | 例子 |
|------|------|
| **全局符号** | 非 static 的函数和全局变量 |
| **外部符号** | 其他模块定义的全局符号（引用） |
| **局部符号** | static 的函数和变量，**只在定义文件内可见** |
| **节符号** | `.text` `.data` 本身 |

⚠️ **函数内的局部变量（auto）不出现在符号表里**！因为它们在运行时栈上分配。
static 局部变量 → 在符号表里。

📝 **真题**：**局部变量**不会列在符号表（D）。

## 3. 静态链接 — 符号解析

### 3.1 强弱符号规则

| 符号 | 定义 |
|------|------|
| **强符号** | 函数和已初始化的全局变量 |
| **弱符号** | 未初始化的全局变量 |

⚠️ **已初始化全局变量是强符号！** 多个模块定义同名已初始化变量 → 链接错误。

### 3.2 弱符号的存放位置

- C 默认把**未初始化的全局变量**作为**弱符号**
- 编译器把弱符号分配到 **COMMON** 块（不是 .bss！）

⚠️ **.bss 存的是已分配但未初始化的局部静态变量**。
**COMMON 存的是可能与其它 .o 同名的未初始化全局变量**（链接器选最大的）。

📝 **真题**：弱符号在 **COMMON**（D）。

### 3.3 解析冲突规则

| 情况 | 规则 |
|------|------|
| 多个强符号同名 | **链接错误** |
| 1 强 + N 弱 | 选强符号 |
| 多个弱符号 | **选 size 最大的** |

### 3.4 静态库链接顺序（⚠️ 必考）

```
g.o → libx.a → liby.a → g.o   （循环）
g.o → liby.a → libz.a → libx.a
```

链接器从左到右扫描，**左边的 .o / .a 可以引用右边的，但反过来不行**。

⚠️ **遇到循环依赖时，要在命令里重复依赖项**（让链接器再扫描一次）。

📝 **真题**：最小命令行 = `gcc g.o libx.a liby.a libz.a libx.a`（B）。
- 第一遍：`g.o libx.a liby.a libz.a` → 解析了大部分，但 `liby.a` 还需要 `libx.a`（来自 g.o 的引用间接传递）
- 第二遍：再加 `libx.a` 解决 `liby.a` 的引用

### 3.5 Module 1 vs Module 2 全局变量

```c
// Module 1
int a;          // 弱符号
int main() { a=0; p2(); a++; printf("%d\n", a); }

// Module 2
int a = 20;     // 强符号
int p2() { a = 10; }
```

- Module 2 的 `a = 20` 是强符号
- Module 1 的 `int a` 是弱符号
- 解析：选强 → a 在 Module 2
- 执行：`a = 0 → p2() 把 a 改成 10 → a++ = 11` → 输出 **11**

📝 **真题**：输出 **11**（B）。

## 4. 重定位（Relocation）

### 4.1 重定位条目格式

```
offset | symbol | type | addend
```

| 字段 | 含义 |
|------|------|
| **offset** | 待修改引用的**节内偏移** |
| **symbol** | 被修改引用应该指向的**符号** |
| **type** | 重定位类型（R_X86_64_PC32 / R_X86_64_32） |

📝 **真题**：重定位条目 `3a: R_X86_64_32 address`
- `3a` = 待修改引用的**节内偏移**（A）
- `address` = 符号名（被引用的符号）

### 4.2 重定位条目位于哪个节？

- `R_X86_64_PC32 search-0x4` 出现在 **`net.o`** 中
- 一般是 **`.rel.text`** 节里存代码段的重定位条目

📝 **真题**：`net.o` 中的重定位条目位于 **`.rel.text`**（D）。

### 4.3 PC 相对 vs 绝对地址

| 类型 | 公式 |
|------|------|
| **R_X86_64_PC32** | `*refptr = S + A - P` （PC 相对） |
| **R_X86_64_32** | `*refptr = S + A` （绝对） |

### 4.4 重定位时机器码解析

汇编指令一行 = 多字节机器码。`ja 4004d5` 是 2 字节指令 `77 02`，下一条 `pop %rbp`（1 字节 `5d`）从偏移 +2 开始。

⚠️ **倒着算位置**：已知跳转目标是 `4004d5`、偏移是 `77 02` = `+2`，所以第一条指令地址 = `4004d5 - 2 = 4004d3`。

📝 **真题**：第一条指令地址 = **4004d3**（C）。

## 5. 可执行目标文件加载

### 5.1 加载过程

```
加载器读 ELF → 拷贝 .text / .data 到内存 → 跳到入口点 (_start)
```

- `.text` `.rodata` 加载到**只读段**
- `.data .bss` 加载到**读写段**
- **程序头部表**描述如何映射

⚠️ **可重定位和可执行目标文件的节类型相似，但** **`.bss` 在可执行文件中仍然是占位**。

### 5.2 共享库（动态库）特点

| 库类型 | 加载时机 | 升级 | 磁盘占用 |
|--------|----------|------|----------|
| **静态库** | 链接时 | 需重新链接 | **大**（代码被拷） |
| **动态库** | 运行时 | 替换 .so 即可 | **小**（多个进程共享同一份代码） |

📝 **真题**：
- 静态库可执行文件**必须依赖静态库**才能运行 ❌ **错误**！静态库的代码已经拷进可执行文件了
- 动态库可执行文件必须依赖动态库 ✅
- 动态库升级方便 ✅
- 静态库占用空间大 ✅

⚠️ **B 选项错误**：静态库的内容已被链接器拷贝到可执行文件里，不再需要原 .a 文件。

## 6. PIC（位置无关代码）

`-fpic` = **生成与位置无关的代码**，用于编译共享库。

📝 **真题**：`-fpic` 用途 = **生成与位置无关的代码**（C）。

### 6.1 PIC 数据引用

- **全局偏移表 GOT**：存放全局变量的地址
- **过程链接表 PLT**：存放函数调用的跳转桩

## 7. 链接器打桩（Interposition）

| 条件 | 可打桩 |
|------|--------|
| 源代码可见 | ✅ |
| 可重定位目标文件可见 | ✅ |
| 可执行目标文件可见 | ✅（动态链接） |
| 汇编代码可见 | 一般不必要 |

📝 **真题**：通常条件 = **能访问可重定位目标文件**（B）——链接器层面打桩。

## 10. 速记口诀

> **.text 代码，.data 已初始化全局/静态，.bss 未初始化静态（占位）**
> **符号表在 .symtab，重定位条目在 .rel.text / .rel.data**
> **强弱规则：同名强 + 弱 → 选强；多弱 → 选大；多强 → 报错**
> **COMMON 存弱符号，.bss 存静态未初始化**
> **静态库从左到右扫，循环依赖就重复**
> **重定位 offset = 节内偏移，symbol = 被引用符号**
> **PC 相对地址 = S + A - P；绝对地址 = S + A**
> **静态库内容已被拷贝到可执行文件，不需要 .a 就能运行**
> **-fpic 生成位置无关代码，给共享库用**
> **链接器打桩的常用条件 = 能拿到可重定位 .o**
