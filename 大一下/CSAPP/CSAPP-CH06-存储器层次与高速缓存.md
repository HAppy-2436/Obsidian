# CH06 存储器层次结构与高速缓存

> 一句话总结：**高速缓存用时间/空间局部性把热点数据放在离 CPU 最近的存储里**，组织方式靠 **组号（s 位）+ 块偏移（b 位）+ 标记（t 位）**，不命中分冷/容量/冲突三类。

> 相关：[[CSAPP-CH03-机器级表示]]、[[CSAPP-CH09-虚拟内存与动态内存分配]]、[[CSAPP全部测验题目]]

---

## 1. 核心框架

```
┌─── CPU 寄存器 ─── 最快，最贵，最小
│
├─── L1/L2/L3 高速缓存（SRAM）─── 在 CPU 内部
│
├─── 主存（DRAM，内存条）─── 易失
│
├─── 本地 SSD / 磁盘 ─── 非易失，慢
│
└─── 远程存储
```

⚠️ **存储技术对比**：
- **SRAM**：高速缓存（贵、快、6 晶体管/位）
- **DRAM**：主存（便宜、慢、要刷新、1 晶体管+电容/位）
- **SSD**：Flash Memory，基于 NAND
- 速度：**SRAM > DRAM > SSD > 磁盘**

📝 **真题**：存取速度最快 = **SRAM**（D）；内存条 = **主存 main memory**（C）。

## 2. 局部性原理

### 2.1 两大原则

| 类型 | 含义 | 例子 |
|------|------|------|
| **时间局部性** | 一旦引用，**近期**很可能再引用 | 循环体里的变量 |
| **空间局部性** | 一旦引用，**附近地址**很可能被引用 | 顺序遍历数组 |

### 2.2 步长（stride）

- **步长越小 → 空间局部性越好**
- **循环体越小 → 时间局部性越好**

📝 **真题**：
- 步长大 → 空间局部性**差**（W12-1）
- 数组引用步长大 → 缓存命中率**倾向于越低**（W9-3）

## 3. 高速缓存组织（⚠️ 必考计算）

### 3.1 三类高速缓存

| 类型 | 每组行数 E | 区别特征 |
|------|-----------|----------|
| **直接映射** | E=1 | 每个组只有 1 行 |
| **组相联** | 1 < E < C/B | 每组 E 行 |
| **全相联** | E = C/B | 只有 1 组 |

⚠️ **直接映射与其它类型的区别 = 每组行数 E**（D）。

### 3.2 地址划分（m 位地址）

```
┌───────── t ─────────┬──── s ────┬──── b ────┐
       标记位           组索引       块内偏移
```

| 符号 | 含义 |
|------|------|
| m | 物理地址位数 |
| C | 缓存总大小（字节） |
| B | 块大小（字节） |
| E | 每组行数 |
| S | 组数 = `C / (B × E)` |
| b | `log₂(B)` |
| s | `log₂(S)` |
| t | `m - s - b` |

### 3.3 公式速记（必背）

$$S = \frac{C}{B \times E}, \quad b = \log_2 B, \quad s = \log_2 S, \quad t = m - s - b$$

📝 **真题**：m=16, C=1024, E=4, B=4
- `S = 1024 / (4×4) = 64`（**C 组**）
- `b = 2, s = 6, t = 16 - 6 - 2 = 8`（**C t=8**）

### 3.4 地址 → 标记 / 组 / 偏移（看十六进制）

把 m 位地址写成 16 进制，从**最低位**开始数 b 位 = 偏移，接下来 s 位 = 组号，剩下高位 = 标记。

📝 **真题**（p.436 练习题 6.12 风格）：
- m=16, C=1024, E=4, B=4 → S=64, s=6, b=2
- 地址 `0x123a` = `0001 0010 0011 1010` = `0001001000111010`
- 低 2 位 = `10`（偏移 2）
- 接下来 6 位 = `001110` = 14（组号）
- 标记 = `0001001000` = 72

⚠️ **题目背景不同参数不同**，必须先从题里抠出 m/C/E/B。

## 4. 三类不命中

| 类型 | 原因 | 怎么缓解 |
|------|------|----------|
| **冷不命中** | 第一次访问 | 不可避免（预取可缓解） |
| **容量不命中** | 工作集 > 缓存容量 | 换更大缓存 / 改算法 |
| **冲突不命中** | 多个块映射到同一组 | 换相联度更高的缓存 |

⚠️ **没有"热不命中"**（D 错误）。

## 5. 写策略

| 策略 | 行为 |
|------|------|
| **直写** | 写穿到内存 |
| **写回** | 等到替换时才写回（**默认**） |
| **写分配** | 写不命中时，先把块加载到缓存再写 |
| **非写分配** | 写不命中时，直接写内存（**绕过缓存**） |

## 6. LRU 替换实战（三路组相联）

📝 **真题**：48 字节容量、三路组相联、块大小 4 → **S = 48/(4×3) = 4 组**

访问序列：`0x02, 0xb4, 0x2a, 0x28, 0x13, 0x22, 0x01, 0x30`

每地址低 2 位 = 块内偏移，剩下 = 标记 + 组号。

| 地址 | 标记 | 组号 | 行为 |
|------|------|------|------|
| 0x02 | 0x0 | 0 | miss → 调入组 0 |
| 0xb4 | 0xb | 1 | miss → 调入组 1 |
| 0x2a | 0x2 | 0 | miss（组 0 已有一个）→ 调入组 0 |
| 0x28 | 0x2 | 0 | **hit**（同一组同一块） |
| 0x13 | 0x1 | 0 | miss（组 0 满 3 个）→ LRU 替换 |
| 0x22 | 0x2 | 0 | **hit** |
| 0x01 | 0x0 | 0 | miss → LRU 替换 |
| 0x30 | 0x3 | 0 | miss → LRU 替换 |

⚠️ **组号由地址中段决定**，低 b 位是块内偏移。

📝 **真题命中次数**：
- 命中：`0x28`、`0x22` → **2 次**（B）

📝 **真题最后 cache 内容**：用最后一次替换后的状态推（看选项中 `T(0)`、`T(1)`、`T(2)` 的 tag 值）。

## 7. 存储器山（Memory Mountain）

- **X 轴**：步长 stride（对数刻度）
- **Y 轴**：大小 size（对数刻度）
- **Z 轴**：吞吐率（MB/s）

**步长越大 → 同一行内的访问更分散 → 空间局部性变差 → 吞吐率下降**。

📝 **真题**：步长大 → 空间局部性**越差**（D）。

## 8. Amdahl 定律

$$\text{加速比} = \frac{1}{(1 - \alpha) + \alpha / k}$$

例：缝制 25%、消毒 75%，整体加速 4/3：
- 缝制加速比 = `s`，消毒加速比 = `t`
- `(1-0.75) + 0.75/t = 1/(4/3) = 3/4`
- `0.25 + 0.75/t = 0.75`
- `0.75/t = 0.5` → `t = 1.5`

📝 **真题**：消毒步骤加速比 = **1.50**（B）。

---

## 9. 磁盘访问时间

$$T_{access} = T_{avg\_seek} + T_{avg\_rotation} + T_{transfer}$$

$$T_{avg\_rotation} = \frac{1}{2} \times \frac{60\,\text{s}}{RPM}$$

📝 **真题**：RPM=10000, T_seek=9ms, T_access=12.02ms
- `T_rotation = (60/10000)/2 = 3ms`
- `T_transfer = 12.02 - 9 - 3 = 0.02ms`
- `扇区数 = T_rotation / T_transfer = 3 / 0.02 = 150`（单条磁道）

⚠️ **每条磁道扇区数 = 旋转一周时间 / 传输一个扇区时间**。

## 11. 速记口诀

> **SRAM 缓存，DRAM 主存，SSD 硬盘**
> **缓存三要素：C/B/E → 算出 S、b、s、t**
> **直接映射 E=1，全相联 S=1**
> **三类不命中：冷、容量、冲突（无"热"）**
> **步长小 → 空间局部性好；循环体小 → 时间局部性好**
> **Amdahl：α 改 1→k，整体 = `1/((1-α)+α/k)`**
> **磁盘访问 = 寻道 + 半圈旋转 + 传输**
> **写回 vs 直写；写分配 vs 非写分配**
