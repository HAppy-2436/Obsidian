# CH02 信息的表示和处理 — 整数与浮点

> 一句话总结：**整数用补码**（永远记住 TMin 只有一个 +2 的 n-1 次方 那个特殊位），**浮点用 IEEE 754**（符号 + 阶码 + 尾数，分规格化与非规格化）。

> 相关：[[CSAPP-CH01-漫游与信息存储]]、[[CSAPP-CH03-机器级表示]]、[[CSAPP全部测验题目]]

---

## 1. 整数编码三大体系

### 1.1 三种编码对比（w 位整数）

| 编码      | 范围                            | 关键特征                        |
| ------- | ----------------------------- | --------------------------- |
| **无符号** | `0 ~ 2^w - 1`                 | 直接二进制                       |
| **原码**  | `-(2^(w-1)-1) ~ +(2^(w-1)-1)` | 符号位 + 绝对值，**正 0 负 0**       |
| **补码**  | `-2^(w-1) ~ 2^(w-1) - 1`      | 最高位权重 `-2^(w-1)`，**只有一个 0** |

⚠️ **原码最小整数的陷阱**：原码最小 = `-(2^(w-1) - 1)`，**不是** `-2^(w-1)`（符号位占 1 位）。

📝 **真题**：10 位原码最小 = `-(2^9 - 1) = -511`。10 位补码 TMin = `-2^9 = -512`。

### 1.2 关键值速记（w 位）

| 名称 | 无符号 | 补码 |
|------|--------|------|
| **UMax** | `2^w - 1` | — |
| **TMax** | — | `2^(w-1) - 1` |
| **TMin** | — | `-2^(w-1)` |
| **-1** | — | 全 1 |
| **0** | 全 0 | 全 0 |

### 1.3 TMax 的加法逆元（必考）

> **TMax + TMax = -2 ≡ TMax**（在补码下）→ **TMax 的加法逆元是它自己**。

📝 **真题**：TMax 的加法逆元 = **TMax**（选项 A）。

## 2. 补码运算性质（⚠️ 全部不满足）

| 性质 | 整数 | 浮点 |
|------|------|------|
| 交换律 `a+b=b+a` | ✅ | ✅ |
| 结合律 `(a+b)+c=a+(b+c)` | ❌（溢出） | ❌（精度） |
| 分配律 `a·(b+c)=a·b+a·c` | ❌（溢出） | ❌ |

📝 **真题**：补码运算**只满足交换律**，结合律和分配律都可能因为溢出被破坏。

## 3. 无符号 / 补码 转换规则

### 3.1 转换函数

| 函数 | 作用 |
|------|------|
| `U2T_w(x)` | 无符号 → 补码：`x < 2^(w-1)` 则不变，否则减 `2^w` |
| `T2U_w(x)` | 补码 → 无符号：`x ≥ 0` 则不变，否则加 `2^w` |
| `T2B_w(x)` | 补码 → 位向量（最高位权重 `-2^(w-1)`） |

📝 **真题**：w=8 时 `U2T(180)`：`180 > 127` → `180 - 256 = -76`。

### 3.2 char / unsigned char 转换

- C 语言赋值时**位模式不变**，只是解读方式变。
- `char x = -100`（位模式 `10011100`）→ 赋给 `unsigned char ux` → `ux = 156`（同位模式）。

⚠️ **不是 100，也不是非法数**，是 `256 - 100 = 156`。

### 3.3 关系比较陷阱（必考）

| 表达式 | 比较方式 | 结果 |
|--------|----------|------|
| `unsigned i = -1; i >= 0` | **按无符号** | **true**（-1 当作 UMax） |
| `-1 < 1U` | 都转无符号 | false（4294967295 < 1 为假，即结果 0） |

📝 **真题**：
- `unsigned i = -1; i >= 0` → **true**
- `-1 < 1U` → 0（false）

## 4. 整数运算速算

### 4.1 移位

| 操作 | 行为 |
|------|------|
| `x << k` | 左移 k 位，**右侧补 0**，溢出截断高位 |
| `x >> k`（逻辑） | 右移 k 位，**左侧补 0** |
| `x >> k`（算术） | 右移 k 位，**左侧补符号位** |

⚠️ **C 标准未规定有符号数右移是逻辑还是算术** → 题目说 "算术移位" 才能确定。

📝 **真题**：`[10100011] << 3`（8 位）→ 高位 101 移出截掉 → `[00011000]`（答案 A）。

### 4.2 乘法优化（用移位加）

把常数 52 = `110100`₂ 拆成 2 的幂之和：
- `52 = 64 - 16 + 4 = 2^6 - 2^4 + 2^2`
- → `(x<<6) - (x<<4) + (x<<2)`

📝 **真题**：`x*52` 用 +、-、<< → `(x<<6) - (x<<4) + (x<<2)`。

### 4.3 除以 2 的 k 次幂（补码偏置）

负数除以 `2^k` 时，**右移前加偏置 `2^k - 1`**：

| k | 偏置 |
|---|------|
| 128 = 2^7 | **127** |
| 64 = 2^6 | 63 |
| 8 = 2^3 | 7 |

📝 **真题**：负整数除以 128 → 偏置 = `2^7 - 1 = 127`。

### 4.4 加法溢出判定

| 类型 | 溢出判定 |
|------|----------|
| **补码正溢出** | `x > 0 && y > 0 && x+y ≤ 0` |
| **补码负溢出** | `x < 0 && y < 0 && x+y ≥ 0` |
| **无符号溢出** | `x + y < x`（或 ` < y`）→ 真实结果要 mod 2^w |

📝 **真题**：无符号加法溢出判据 = **和小于 x 或小于 y**（选项 D `x+y < x`）。

### 4.5 截断

把 w 位补码截到 k 位：保留低 k 位，**重新解读为 k 位补码**。
例：8 位补码 33 = `00100001`，截到 6 位 = `100001` → 6 位补码 = `-31`（最高位 1 是负号，`100001 = -32 + 1 = -31`）。

## 5. 浮点数 IEEE 754

### 5.1 格式（k 位阶码，n 位小数）

```
┌─s─┬─────exp─────┬───────frac───────┐
 1        k              n
```

| 类型 | exp | frac | 数值公式 |
|------|-----|------|----------|
| **规格化** | 既不全 0 也不全 1 | 任意 | `(-1)^s × 2^(exp-bias) × (1 + frac)` |
| **非规格化** | 全 0 | 任意 | `(-1)^s × 2^(1-bias) × (0 + frac)` |
| **无穷** | 全 1 | 全 0 | ±∞ |
| **NaN** | 全 1 | 非 0 | NaN |

**偏置 bias = `2^(k-1) - 1`**。

### 5.2 浮点 ↔ 十进制

例：23 → `10111` → 规格化 → `1.0111 × 2^4` → **frac 字段 = `0111`**（小数点后 4 位补齐到 n 位）。

📝 **真题**：数字 11 → `1011` → `1.011 × 2^3` → 阶码 = `3 + bias` → 偏置后字段值 `= 3`（题中可能给的是 K 位偏置后的字段值）。

### 5.3 8 位自定义浮点（练手）

例：1 符号 + 3 阶码 + 4 小数，编码 `01010101`：
- s=0, exp=`101`=5, frac=`0101`
- bias = `2^2 - 1 = 3`，真阶 = `5 - 3 = 2`
- 数值 = `1.0101₂ × 2^2 = 1.3125 × 4 = 5.25`

📝 **真题**：同款格式 `01010101` → 选 **C (5/4 不对) 或 D (21/4 = 5.25)**，看小数位数。

⚠️ **公式记牢**：规格化值 = `(1 + 0.f₁f₂...) × 2^(exp-bias)`。

### 5.4 最大非规格数（必考）

**非规格最大** = `2^(1-bias) × (1 - 2^(-n))` ≈ `2^(1-bias) × (2^n - 1) / 2^n`

例：1+6+4 格式（11 位），非规格最大 = `2^(1-31) × (1 - 2^-4) = 2^-30 × 15/16 = 2^-34 × 15`

📝 **真题**：11 位（1+6+4）最大非规格数 = `2^-34 × 15`（选 B）。

### 5.5 最小正数（非规格最小）

非规格最小 = `2^(1-bias) × 2^(-n) = 2^(-bias-n)`。

例：1+2+3 格式（6 位），bias=`2^1-1=1`，n=3 → 最小正数 = `2^(-1-3) = 2^-4 = 1/16`。

📝 **真题**：6 位浮点最小正数 = `1/16`（选 B）。

### 5.6 向偶数舍入（默认）

- 舍入位 = 1：看保留位最低位，**偶数 → 0，奇数 → 1**
- 舍入位 = 0：直接截断

例：`11.011` 保留 2 位小数 → 看 `.011` 末位 `.1` 触发 → 保留位是 `1`（奇）→ 进位 → `11.10`。
例：`10.1011` 保留 2 位小数 → 看 `.11` → 进位 → `11.00`。

📝 **真题**：`11.011` → `11.10`；`10.1011` → `11.00`。

### 5.7 浮点运算特性

- **不满足结合律**：`1e20 + (-1e20 + 1.25) = 1e20 + (-1e20) = 0` ≠ `(1e20 + -1e20) + 1.25 = 1.25`
- **满足交换律**（加法）
- **`x*x ≥ 0`** 对 float **永远成立**（除 inf/NaN），对 int 不一定（溢出变负）

📝 **真题**：
- `1e20 + (-1e20 + 1.25) = 0.0`
- 仅命题 (2) 成立：`f*f ≥ 0.0` 对 float 恒真；`i*i ≥ 0` 对 int 不一定（例 50000*50000 溢出变负）

### 5.8 类型转换相等性（必考）

| 表达式 | 是否总成立 | 原因 |
|--------|----------|------|
| `x == (int)(double)x` | ✅ 几乎总成立 | double 范围大 |
| `x == (int)(float)x` | ❌ 可能不成立 | float 只能精确表示到 2^24 |
| `d == (double)(float)d` | ❌ 可能不成立 | float → double 保留近似值 |
| `(f+d) - f == d` | ❌ | 浮点加法不结合 |

⚠️ **double 比 int 大很多但 float 不够大**：超过 2^24 的 int 转 float 会丢精度。

📝 **真题**：四个选项**只有 A 总为真**。

## 7. 速记口诀

> **原码最小：负 2^(w-1) - 1；补码最小：负 2^(w-1)**
> **TMax 逆元是它自己，TMin 逆元是它自己**
> **位模式不变，只是解读方式变**
> **无符号比较 -1 > 0；-1 < 1U 是 false**
> **`x*52 = (x<<6) - (x<<4) + (x<<2)`**
> **负数除以 2^k → 加偏置 2^k - 1**
> **补码运算只满足交换律；浮点运算连交换律都不绝对满足（NaN 时）**
> **浮点非规格最小 = 2^(-bias-n)；非规格最大 = 2^(1-bias) × (1-2^-n)**
> **向偶舍入看保留位最低位：偶不变，奇进位**
> **double 比 int 大很多，float 不够 → 转 float 可能丢精度**
