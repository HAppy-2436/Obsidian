### Slide 8

```
指针声明

C++ uses an asterisk * to denote a pointer. If Item is a type, then a pointer to such an Item object has the type Item *. For example,
Item *item_ptr;
declares item_ptr as a pointer variable to an Item object.

Creating dynamic objects

Item_ptr = new Item;
creates a new dynamic object of type Item and assigns its location to the pointer variable item_ptr.

Deleting dynamic objects

delete item_ptr;

disposes of the dynamic object to which item_ptr points and returns the space it occupies to the free store so it can be used again.

上述语句在释放指针item_ptr指向的空间。指针类型变量item_ptr可以继续在后续的程序中使用。
可以在释放之后，将item_ptr设置为null。
```

```
指针运算

If i is an integer value and p is a pointer to an Item：
 then (p + i) is an expression of type Item *.
The value of p + i gives the memory address offset from p by i Item objects.
即： the expression (p + i) actually yields the address (p + n*i), where n is the number of bytes of storage occupied by a simple object of type Item.
```

```
Linked Stacks

考虑一下在设计链式的栈时：
栈顶应该在链表的头部还是链表的尾部比较方便。

如果栈顶在链表的尾部，入栈时比较方便，出栈时却比较麻烦，因为出栈之后。栈顶的指针需要从n移动到n-1，这个步骤需要遍历从头部到n-1处的所有节点，才能完成。

如果栈顶在链表的头部，入栈时比较方便，出栈时也比较方便，所以选择这种情况有利于操作。
```

### Slide 37

```
Error_code Stack::pop()
/*
Post: The top of the Stack is removed.  If the Stack
      is empty the method returns underflow; otherwise it returns success.
*/
{
   Node<Node_entry> *old_top = top_node;
   if (top_node == NULL) return underflow;
   top_node = old_top->next;
   delete old_top; //释放被删除结点的空间
   return success;
}
```
