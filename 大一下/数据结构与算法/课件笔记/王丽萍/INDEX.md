## 解析说明

### 技术栈
- **Python 库**：`olefile` (OLE2 解析) + 自实现 MS-PPT 二进制扫描
- **目标文件**：旧版 `.ppt` (PowerPoint 97-2003, OLE2 Compound File 格式)
- **抽取原子**：TextCharsAtom (0x0FA0) + TextBytesAtom (0x0FA8)，文本以 UTF-16-LE 编码

### 已知限制
1. **页面分组是启发式的**：基于文本原子在流中的偏移间距推测，可能过度切分（实际 1 张幻灯片可能被分成 2-3 个组）。如需精确按页还原，需要完整解析 MS-PPT 容器树（`SlideListWithText`、`SlideContainer` 等）。
2. **特殊字符乱码**：原 PPT 中部分特殊符号（—、""、""、…）在抽取过程中显示为 `�?`，因为原 PPT 的字体子集可能使用了私有 Unicode 编码。建议结合课堂笔记使用。
3. **图表 / 图片 / 公式**：本次只抽取文本框内容，图片中的 OCR 文字、数学公式的精确表达（如集合论符号 Σ）无法保留。
4. **OneDrive 占位文件**：3 个失败的课件实际上是 HTML 错误页，需要手动补齐源文件。
