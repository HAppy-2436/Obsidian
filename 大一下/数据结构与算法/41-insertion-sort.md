# 运用逆向思维：插入排序

## 学习目标

- 理解 **插入排序（Insertion Sort）** 的核心思想：往 **已排序区间** 插入新元素
- 掌握插入排序的标准 C++ 实现
- 认识到插入排序是 **稳定 + 原地** 的自适应排序，对 **部分有序数组** 特别友好
- 理解「**有序度越高，效率越高**」的本质原因

## 一句话总结

**插入排序** 的核心思路是：维护 `[0, sortedIndex)` 为有序区间，每次把 `nums[sortedIndex]` 插入到左侧有序区间的合适位置。它是 **稳定 + 原地** 的 **自适应排序**——数组有序度越高，效率越高（最好 O(N)），适合 **接近有序的小数据**。

## 1. 反向思维：从「右往左插」到「左往右插」

### 1.1 两条不同的路

回顾 [[40-bubble-sort|冒泡排序]] 的优化过程——我们原本想「**在未排序区间找最小值，插入到分界线**」，但因为数据搬移代价太大，改成了「**通过交换相邻逆序对让最小值冒泡上来**」。

能不能反过来想？既然 `[0, sortedIndex)` 已经是 **有序** 的，那为什么不直接把 `nums[sortedIndex]` **插入到左侧有序数组的合适位置** 呢？

```text
已排序区间：[2, 5, 7, 9]
            0  1  2  3

要插入：3（sortedIndex = 4）

查找 3 应该插入的位置：2 和 5 之间 → 下标 1
插入：   [2, _, 5, 7, 9]
            ↑
            3
最终：   [2, 3, 5, 7, 9]
```

### 1.2 为什么不直接二分？

你可能会想：用 [[#二分插入|二分搜索]] 找到插入位置，把第一个 for 循环（O(N)）优化到 O(log N) 不就行了？

**答案是没必要**。因为即便二分找到了位置，**插入仍然需要搬移元素**（O(N)），整体复杂度还是 O(N)。所以还不如「**边遍历边交换，遇到有序就停**」来得简单高效。

## 2. 标准代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace dsa {

// 插入排序（Insertion Sort）
// 时间复杂度：最好 O(N)（已排序），平均/最坏 O(N²)
// 空间复杂度：O(1)，原地排序
// 稳定性：✅ 稳定（只在 nums[i] < nums[i-1] 时才交换）
void insertionSort(vector<int>& nums) {
    int n = nums.size();
    // 维护 [0, sortedIndex) 是有序区间
    int sortedIndex = 0;
    while (sortedIndex < n) {
        // 把 nums[sortedIndex] 插入到左侧有序数组 [0, sortedIndex)
        for (int i = sortedIndex; i > 0; i--) {
            if (nums[i] < nums[i - 1]) {
                swap(nums[i], nums[i - 1]);
            } else {
                break;  // 左侧已经有序，无需继续
            }
        }
        sortedIndex++;
    }
}

} // namespace dsa
```

> [!tip] 与冒泡排序的核心区别
> 冒泡：内层循环从尾部向 sortedIndex 走，每趟冒一个最小值上来。
> 插入：内层循环从 sortedIndex 向头部走，每趟把新元素「沉」到正确位置。
> 两者都是 O(N²)，但 **插入排序的常数更小**，因为「有序就 break」。

## 3. 与冒泡排序的对比

| 维度 | [[40-bubble-sort\|冒泡排序]] | 插入排序 |
|------|---------|---------|
| 内层循环方向 | 从尾 → sortedIndex | 从 sortedIndex → 0 |
| 内层循环终止条件 | 必定跑满 sortedIndex+1..n | 遇到有序就 break |
| 实际比较次数 | ~n²/2 | ≤ n²/2 |
| 稳定性 | ✅ | ✅ |
| 原地性 | ✅ | ✅ |
| 对部分有序的友好度 | 一般（交换次数多） | 很强（接近 O(N)） |

## 4. 自适应排序：有序度越高效率越高

插入排序最大的特点是 **自适应**：

| 输入类型 | 比较次数 |
|---------|---------|
| 完全有序 | N - 1 次 |
| 接近有序（少量逆序对） | 接近 N |
| 完全逆序 | n²/2 次 |

### 4.1 为什么？

每次插入新元素时，内层循环从 sortedIndex 往左走，**只要遇到不小于左侧元素的，就 break**。

```text
[1, 2, 3, 4, 5, 6]  ← 已有序
                  7  ← 要插入
新元素 7 进来，内层循环：7 < 6？否 → 立刻 break
仅 1 次比较 → O(N) 总计
```

而 [[40-bubble-sort|冒泡排序]] 没有这个 early break 机制，每趟内层都必须跑完。

### 4.2 极端例子

```text
完全有序：[1, 2, 3, 4, 5]
  第 1 趟：插入 2，与 1 比较，break
  第 2 趟：插入 3，与 2 比较，break
  ...
  共 4 次比较 → O(N)

完全逆序：[5, 4, 3, 2, 1]
  第 1 趟：插入 4，与 5 比较，swap → 与 3 比较（无），停
  第 2 趟：插入 3，与 4 swap，与 3 swap，与 2 swap，停
  ...
  共 1+2+3+4 = 10 次比较 → O(N²)
```

## 5. 改进：二分插入排序

如果你仍然想用二分优化查找位置，可以把「比较次数」降到 O(N log N)（但搬移仍是 O(N)，所以整体还是 O(N²)）。

```cpp
// 二分插入排序：仅减少比较次数，不改变总复杂度
void binaryInsertionSort(vector<int>& nums) {
    int n = nums.size();
    for (int i = 1; i < n; i++) {
        int target = nums[i];
        // 二分查找 [0, i) 中 target 应插入的位置 lo
        int lo = 0, hi = i;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (nums[mid] <= target) lo = mid + 1;
            else hi = mid;
        }
        // 把 [lo, i) 整体后移一位
        for (int j = i; j > lo; j--) {
            nums[j] = nums[j - 1];
        }
        nums[lo] = target;
    }
}
```

> [!warning] 二分插入的工程意义有限
> 比较在 CPU 看来极快，搬移才是大头。实际工程上很少使用。

## 6. 复杂度表

| 指标 | 值 |
|------|---|
| 最好时间 | O(N)（已排序数组） |
| 平均时间 | O(N²) |
| 最坏时间 | O(N²)（完全逆序） |
| 空间复杂度 | O(1) |
| 稳定性 | ✅ 稳定 |
| 原地性 | ✅ 原地 |
| 自适应性 | ✅ 强（有序度越高越快） |

## 7. 适用场景

虽然插入排序平均是 O(N²)，但有以下不可替代的价值：

1. **N 很小（N ≤ 50）**：常数极小，往往比 [[44-quick-sort|快排]] 还快
2. **接近有序的数据**：通过 [[41-insertion-sort|插入排序]] 可达 O(N) 级别
3. **混合排序算法的兜底**：`std::sort` 和 `Arrays.sort` 在递归深度达到某个阈值时，会 **切换到插入排序** 作为子数组的排序方式
4. **在线排序（Online Sort）**：数据流式到来时，每来一个就插入到已排序部分，是 **唯一合适的算法**

> [!tip] STL 的 sort 内部就用插入排序兜底
> `std::sort` 是 introsort（快排 + 堆排 + 插入排序混合），当递归子数组长度 < 16 时，会切换到插入排序。这印证了插入排序在「小规模 + 部分有序」场景的优势。

| 维度 | 插入排序 | 快速排序 | 归并排序 |
|------|---------|---------|---------|
| 平均时间 | O(N²) | O(N log N) | O(N log N) |

| 大数据 | ❌ 慢 | ✅ | ✅ |
