# 队列/栈的基本原理

## 学习目标

读完本章后，你应该能够：

- 清楚地说出「队列」和「栈」的核心区别（FIFO vs LIFO）。
- 准确画出队列和栈的基本 API，理解它们都是**操作受限**的线性结构。
- 理解「为什么」数组和链表能作为队列/栈的底层存储。
- 在合适的场景选择队列或栈解决问题（任务调度、撤销/重做、DFS/BFS、表达式求值等）。

## 一句话总结

> **队列 = 一端进、另一端出（FIFO）；栈 = 同一端进出（LIFO）。** 它们都是把数组/链表的「任意位置增删查改」砍掉、只保留头尾操作的「操作受限」结构。

阅读本章前，建议先掌握：

- [[05-linkedlist-basic|链表（链式存储）的原理]]

## 为什么需要队列和栈？

在 [[02-array-basic|数组]] 和 [[05-linkedlist-basic|链表]] 里，我们实现了「在任意合法索引上增删查改」的 API。这些 API 功能强大，但**太自由了**——很多真实场景里，我们只想要一小部分操作：

- 排队买票：只想「队尾进、队头出」，不想让中间的人插队。
- 函数调用栈：只想「栈顶进、栈顶出」，LIFO 语义符合调用/返回的嵌套顺序。
- 浏览器后退按钮：只想看最近访问的页面，最近的先被弹出。

这些「操作受限」的线性结构就是**队列（Queue）**和**栈（Stack）**。受限 = 简单 = 容易分析、容易并发、容易用硬件/操作系统支持。

## 一、队列（Queue）：先进先出 FIFO

> 类比：**排队买票**。先来的人先被服务，后来的人排在队尾。

### 1.1 队列的约束

- **入队（enqueue/push）**：只能在**队尾**插入元素。
- **出队（dequeue/pop）**：只能从**队头**删除元素。
- **查看（peek/ front）**：只能看队头元素，不能看中间。
- **size()**：返回元素个数。

不允许随机访问队头以外的元素，不允许从队头插入（那是「插队」）。

### 1.2 队列的基本 API

```cpp
// 队列的基本 API（C++ 伪代码示意）
template <typename E>
class MyQueue {
public:
    // 向队尾插入元素，时间复杂度 O(1)
    void push(const E& e);

    // 从队头删除元素，时间复杂度 O(1)
    E pop();

    // 查看队头元素（不删除），时间复杂度 O(1)
    E& peek();

    // 返回队列中的元素个数，时间复杂度 O(1)
    int size() const;
};
```

> [!info] 不同语言命名不同
> `push` / `offer` / `enqueue` 都可能表示入队；`pop` / `poll` / `dequeue` 都可能表示出队。要点是**尾部进、头部出**这个语义。

### 1.3 队列的典型应用

- **任务调度**：操作系统进程队列、线程池任务队列、打印队列。
- **消息队列（Message Queue）**：Kafka、RabbitMQ、Redis Stream 等中间件的核心抽象。
- **BFS（广度优先搜索）**：[[34-graph-traverse-basic|图的 BFS 遍历]]、二叉树层序遍历都靠队列实现「一层一层」展开。
- **滑动窗口 / 单调队列**：LeetCode 239「滑动窗口最大值」是单调队列的经典题。
- **生产者-消费者模型**：一个线程往队尾 push，另一个线程从队头 pop。

## 二、栈（Stack）：先进后出 LIFO

> 类比：**一摞盘子**。最后放上去的盘子在最上面，下次拿走的也是它。

### 2.1 栈的约束

- **入栈（push）**：只能在**栈顶**插入元素。
- **出栈（pop）**：只能从**栈顶**删除元素。
- **查看（peek/top）**：只能看栈顶元素。
- **size()**：返回元素个数。

同样不允许随机访问栈顶以外的元素。

### 2.2 栈的基本 API

```cpp
// 栈的基本 API（C++ 伪代码示意）
template <typename E>
class MyStack {
public:
    // 向栈顶插入元素，时间复杂度 O(1)
    void push(const E& e);

    // 从栈顶弹出元素，时间复杂度 O(1)
    E pop();

    // 查看栈顶元素（不弹出），时间复杂度 O(1)
    E& peek();

    // 返回栈中的元素个数，时间复杂度 O(1)
    int size() const;
};
```

### 2.3 栈的典型应用

- **函数调用栈**：每次函数调用压栈，return 时弹栈。递归本质上就是栈。
- **撤销/重做（Undo/Redo）**：编辑器、IDE、Photoshop 全部用栈记录操作历史。
- **括号匹配**：LeetCode 20「有效的括号」是栈最直白的应用。
- **表达式求值**：中缀转后缀（逆波兰式）、计算器实现都靠栈。
- **DFS（深度优先搜索）**：[[34-graph-traverse-basic|图的 DFS 遍历]]、二叉树前/中/后序遍历的非递归实现都用栈。
- **浏览器前进/后退**：每个标签页维护两个栈（back stack + forward stack）。

## 三、队列 vs 栈：一张表对比

| 维度 | 队列 (Queue) | 栈 (Stack) |
| --- | --- | --- |
| 中文 | 队列 | 栈（堆栈） |
| 英文 | Queue | Stack |
| 数据进出顺序 | 先进先出 FIFO | 先进后出 LIFO |
| 插入端 | 队尾 | 栈顶 |
| 删除端 | 队头 | 栈顶 |
| 类比 | 排队买票 | 一摞盘子 |
| 时间复杂度（push/pop/peek） | O(1) | O(1) |
| 典型算法 | BFS、单调队列、滑动窗口 | DFS、括号匹配、表达式求值 |
| 操作系统实例 | 任务就绪队列、消息队列 | 函数调用栈、中断处理栈 |

## 四、底层实现的选择

队列和栈都只是「API 约定」，**没有规定底层用什么存**。常见选择有：

- **动态数组（顺序存储）**：栈用 `vector` 的 `push_back` + `pop_back` 非常自然；队列要 `pop_front` 就需要 [[04-cycle-array|环形数组]] 技巧才能保持 O(1)。
- **双向链表（链式存储）**：栈/队列用双向链表实现都很简单，头尾操作都是 O(1)，且天然支持无限扩容。

这两种实现的具体代码分别在 [[09-array-queue-stack|数组实现队列/栈]] 和 [[10-linked-queue-stack|链表实现队列/栈]] 中给出。

## 五、复杂度小结

| 操作 | 队列 | 栈 | 说明 |
| --- | --- | --- | --- |
| push（入队/入栈） | O(1) | O(1) | 增 |
| pop（出队/出栈） | O(1) | O(1) | 删（默认实现下） |
| peek（看头/看顶） | O(1) | O(1) | 读 |
| size | O(1) | O(1) | 维护一个计数器即可 |
| 随机访问 | 不支持 | 不支持 | 这是「操作受限」的代价 |

> **关键点**：队列和栈的 push/pop/peek 都是 **O(1)**，这是它们被广泛使用的基础。代价是失去了随机访问能力——你不能 O(1) 地取「队头后第 3 个元素」或「栈底元素」。

## 六、为什么说它们是「操作受限」结构？

对照一下：

| 数据结构 | insert | delete | 随机访问 | 是否受限 |
| --- | --- | --- | --- | --- |
| 动态数组 | O(n) 任意位 | O(n) 任意位 | O(1) | 否 |
| 双向链表 | O(1) 任意位 | O(1) 已知节点 | O(n) | 否 |
| 队列 | O(1) 仅队尾 | O(1) 仅队头 | ✗ | **是** |
| 栈 | O(1) 仅栈顶 | O(1) 仅栈顶 | ✗ | **是** |

队列/栈把 API 砍掉了一大部分，只留头尾/顶部的 O(1) 操作。这种「减法」反而带来了**简洁性、可分析性、可并发性**——这是数据结构设计里非常常见的 trade-off。

## 七、C++ 标准库中的队列和栈

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    // 栈
    stack<int> st;
    st.push(1);
    st.push(2);
    st.push(3);
    cout << st.top() << "\n"; // 3
    st.pop();                 // 弹出 3

    // 队列
    queue<int> q;
    q.push(1);
    q.push(2);
    q.push(3);
    cout << q.front() << "\n"; // 1
    q.pop();                   // 弹出 1
    cout << q.size() << "\n";  // 2
}
```

> `std::stack` 默认基于 `std::deque` 实现，`std::queue` 默认也基于 `std::deque`（容器适配器）。如果想自己用 `vector` 或 `list` 当底层，传第二个模板参数即可：
> `stack<int, vector<int>> st;`

## 八、易混淆点与踩坑提示

1. **`stack` 的 `top()` 不会删除元素**，`pop()` 才删除但**不返回值**（沿用早期 STL 风格，为了异常安全）。
2. **`queue` 的 `front()`/`back()`** 分别返回队头/队尾元素。
3. **不要把栈的 `push/pop` 和 vector 的 `push_back/pop_back` 弄混**——`std::vector` 也没有 `push` 这个名字。
4. **递归 = 栈**：写不出递归就想想调用栈怎么走的；写递归转非递归，本质就是把系统调用栈换成自己维护的栈。
5. **空队列/空栈访问 front/top 是未定义行为**，先 `empty()` 检查。

## 九、应用场景汇总

| 场景 | 用栈还是队列 | 关键原因 |
| --- | --- | --- |
| 函数调用 | 栈 | 嵌套调用/返回天然 LIFO |
| 撤销/重做 | 栈 | 最近操作最先回退 |
| 括号匹配 | 栈 | 嵌套结构 LIFO |
| BFS | 队列 | 一层一层扩散 FIFO |
| DFS | 栈 | 深度优先一路走到底 |
| 滑动窗口最值 | 单调队列 | 队头始终是当前窗口最值 |
| 消息缓冲 | 队列 | 先到先服务 |
| 表达式求值 | 栈 | 运算符优先级管理 |
