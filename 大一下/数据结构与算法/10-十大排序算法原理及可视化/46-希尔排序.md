## 希尔排序原理

### 为什么需要希尔排序？

插入排序在最坏情况（逆序数组）下需要 O(N²) 次比较和交换，主要原因是每次插入只能移动一个位置。

**希尔排序的核心洞察**：如果一个数组**大致有序**，插入排序非常快（接近 O(N)）。那么能不能先做几次"粗略排序"，让数组大致有序，最后再做一次插入排序？

### 核心思想：间隔分组

1. 选一个**间隔序列**（gap / h）：h₁, h₂, ..., hk，最后一个 hk = 1
2. 对每个 hi，把数组分成 hi 个子序列，每个子序列由相距 hi 的元素组成
3. 对每个子序列做**插入排序**（这叫 "h-排序"）
4. 缩小 hi，重复步骤 2-3，直到 hi = 1（最后一次就是普通插入排序）

每次 h-排序后，数组都比上一次更"接近有序"。最后一次 hi = 1 时数组已基本有序，插入排序飞快。

### h 有序数组

一个数组如果按间隔 h 是有序的（即任意两个相距 h 的元素都有序），就称它为 **h 有序**。

希尔排序的核心就是：h₁ 有序的数组再做 h₂ 排序（h₂ < h₁）后，仍然保持 h₁ 有序。这叫**h₁, h₂ 不相互干扰**——这是希尔排序正确性的关键。

## 增量序列的选择

增量序列的选择直接影响希尔排序的效率。常用三种：

### 1. Shell 原始序列：N/2, N/4, ..., 1
最简单，但效率较差，最坏 O(N²)。

### 2. Knuth 序列：1, 4, 13, 40, 121, ...
公式：`h = 3*h + 1`，即 `h = (3^k - 1) / 2`。
生成方式：从 h=1 开始，反推最大的 h < N/3。
**平均 O(N^1.5)**，是工业界最常用的序列。

### 3. Sedgewick 序列：性能最优但复杂
公式：`h = 4^k + 3*2^(k-1) + 1` 或 `9*4^k - 9*2^k + 1`
最坏 O(N^(4/3))。

## 完整 C++ 实现

### 基础版（Shell 序列）

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace dsa {

// 希尔排序：Shell 增量序列（N/2, N/4, ..., 1）
void shellSortBasic(vector<int>& nums) {
    int n = nums.size();
    // 从 n/2 开始，每次减半
    for (int h = n / 2; h > 0; h /= 2) {
        // 对每个子序列做插入排序
        for (int i = h; i < n; i++) {
            int temp = nums[i];
            int j = i;
            while (j >= h && nums[j - h] > temp) {
                nums[j] = nums[j - h];
                j -= h;
            }
            nums[j] = temp;
        }
    }
}

} // namespace dsa
```

### 工业推荐版（Knuth 序列）

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace dsa {

// 希尔排序：Knuth 增量序列（推荐）
void shellSort(vector<int>& nums) {
    int n = nums.size();
    // 1. 计算最大的 h：1, 4, 13, 40, ...
    int h = 1;
    while (h < n / 3) h = h * 3 + 1;

    // 2. 从 h 开始，不断除以 3 直到 1
    while (h >= 1) {
        // h-排序：对每个子序列做插入排序
        for (int i = h; i < n; i++) {
            int temp = nums[i];
            int j = i;
            while (j >= h && nums[j - h] > temp) {
                nums[j] = nums[j - h];
                j -= h;
            }
            nums[j] = temp;
        }
        h /= 3;
    }
}

// 通用希尔排序：自定义增量序列
void shellSortGeneric(vector<int>& nums, vector<int>& gaps) {
    int n = nums.size();
    for (int h : gaps) {
        if (h <= 0 || h >= n) continue;
        for (int i = h; i < n; i++) {
            int temp = nums[i];
            int j = i;
            while (j >= h && nums[j - h] > temp) {
                nums[j] = nums[j - h];
                j -= h;
            }
            nums[j] = temp;
        }
    }
}

} // namespace dsa
```

## 复杂度分析

| 增量序列 | 最好 | 平均 | 最坏 | 空间 |
|---------|------|------|------|------|
| Shell (N/2^k) | O(N log N) | O(N²) | O(N²) | O(1) |
| Knuth | O(N) | O(N^1.5) | O(N²) | O(1) |
| Sedgewick | O(N) | O(N^(7/6)) | O(N^(4/3)) | O(1) |

**为什么希尔排序比插入排序快？**

| 维度 | 插入排序 | 希尔排序 |
|------|----------|----------|
| 比较次数 | O(N²) | O(N^1.5) |
| 移动步长 | 每次 1 步 | 最大可达 h 步 |
| 数据移动 | 频繁小幅移动 | 大步跳跃，最后微调 |
| 缓存友好 | 较好 | 较好 |

关键：**最后一次 h=1 的插入排序时，数组已经基本有序**，所以工作量很小。前面的 h-排序把元素"大致归位"，最后微调一下就完成了。

## 应用场景

### 何时用希尔排序？

- **中等规模数据**（10³ ~ 10⁵）：比插入排序快，比快排简单
- **代码长度有限**：希尔排序的代码比快排、堆排序短
- **不需要稳定排序**：希尔排序是不稳定的

### 何时不用？

- 数据规模 > 10⁶：用快排或归并
- 需要稳定排序：用归并
- 对性能极致要求：用 introsort（C++ std::sort 用的）

### 工业使用案例

- **uClibc**：早期 Linux C 库用希尔排序作为 qsort 备选
- **嵌入式系统**：资源紧张时，希尔排序是 O(N^1.5) 中代码最简的
- **Java Arrays.sort 早期版本**：对部分类型使用希尔排序

## 稳定性分析

希尔排序是**不稳定**的。

原因：当 h > 1 时，相距 h 的元素可能交换位置，破坏相等元素的相对顺序。例如：
```
排序前：[2, 2', 1]  (2 和 2' 是相等的两个元素)
h=1 后：[1, 2, 2']  ← 顺序保持
但 h=2 时（如果是这样分）[1, 2, 2'] → [1, 2', 2] ← 顺序变了
```

## 实战建议

1. **首选 Knuth 序列**：代码短、效率好
2. **小数组直接插入排序**：N < 50 时插入排序更快
3. **大数组用 std::sort**：C++17 的 std::sort 是 introsort（快排+堆排+插入排序混合），工业级最优
4. **调试时打印中间结果**：希尔排序的 h-排序过程容易让人迷惑
