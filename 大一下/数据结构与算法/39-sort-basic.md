# 排序算法的关键指标

## 学习目标

- 理解评估排序算法的 **三大核心指标**：时空复杂度、稳定性、是否原地
- 掌握「稳定排序」与「不稳定排序」的概念及其现实意义
- 理解「原地排序」与「非原地排序」的区别及空间复杂度含义
- 能够在对比不同排序算法时，依据这三大指标分析其优劣

## 一句话总结

排序算法的优劣可从三个维度评判：**时空复杂度**（效率和资源消耗）、**稳定性**（相同元素的相对位置是否改变）、**原地排序**（是否需要 O(N) 额外空间）。后文所有排序算法的分析都会围绕这三个指标展开。

## 1. 为什么需要「指标」？

排序是计算机科学最基础的问题之一，没有之一。

刷题和面试/笔试中，你也许不会直接被要求手写一个排序算法，但 **理解排序算法的原理、复杂度、稳定性**，对于理解其他算法（如 [[44-quick-sort|快速排序]] 引出的分治思想，[[48-bucket-sort|桶排序]] 引出的哈希分布思想）有莫大帮助。

更重要的是，排序问题的多种解法体现了「**同一个问题，可以从不同角度切入**」的算法思维。

> [!tip] 排序章节的脉络
> 本文先建立评估指标；后续 [[42-select-sort|选择排序]] → [[40-bubble-sort|冒泡排序]] → [[41-insertion-sort|插入排序]] → [[46-shell-sort|希尔排序]] → [[44-quick-sort|快速排序]] → [[43-merge-sort|归并排序]] → [[45-heap-sort|堆排序]] 一路演进，最终进入 [[47-counting-sort|计数排序]] / [[48-bucket-sort|桶排序]] / [[49-radix-sort|基数排序]] 的非比较类排序世界。

## 2. 指标一：时空复杂度

排序算法的 **时间复杂度** 和 **空间复杂度** 是最直观的指标。

时间复杂度越低，排序越快；空间复杂度越低，辅助存储越少。这点和 [[01-complexity|其他算法]] 完全一致。

排序算法按时间复杂度可分为两大阵营：

| 阵营 | 时间复杂度 | 代表算法 |
|------|----------|---------|
| **比较类排序** | 最优 O(N log N) | [[44-quick-sort|快排]]、[[43-merge-sort|归并]]、[[45-heap-sort|堆排]] |
| **非比较类排序** | 可达 O(N) | [[47-counting-sort|计数]]、[[48-bucket-sort|桶排]]、[[49-radix-sort|基数]] |

按是否原地又可分为：

- **原地排序**：O(1) 额外空间（如快排、堆排）
- **非原地排序**：O(N) 额外空间（如归并、计数）

## 3. 指标二：排序稳定性

> **定义**：对于序列中的相同元素，如果排序之后它们的相对位置没有发生改变，则称该排序算法为「**稳定排序**」，反之则为「**不稳定排序**」。

### 3.1 直观示例

排序整数时，稳定性没有意义，因为没有「区分两个相同整数」的标识。但排序 **结构体/对象** 时，稳定性就至关重要。

```text
排序前（按交易日期排好序）：

   Date        UserID
2020-01-01    1001
2020-01-02    1001
2020-01-03    1001
2020-02-01    1002
2020-02-02    1002
2020-02-03    1002

按 UserID 稳定排序后：

   Date        UserID
2020-01-01    1001   ← 日期仍有序
2020-01-02    1001
2020-01-03    1001
2020-02-01    1002
2020-02-02    1002
2020-02-03    1002
```

> [!success] 稳定排序的妙处
> 由于之前已经按日期排好序，对 UserID 稳定排序后，相同 UserID 的订单的相对位置保持不变，**日期的有序性也保留下来**。这种「在已有序基础上再排序」的累加效果，是稳定排序的核心价值。

如果用 **不稳定排序**，相同 UserID 的订单相对位置可能变化，日期的有序性会丧失，相当于你之前对日期的排序白做了。

### 3.2 常见排序算法的稳定性

| 算法 | 是否稳定 | 说明 |
|------|---------|------|
| [[40-bubble-sort\|冒泡排序]] | ✅ 稳定 | 只交换相邻逆序对，不动相同元素 |
| [[41-insertion-sort\|插入排序]] | ✅ 稳定 | 同上 |
| [[42-select-sort\|选择排序]] | ❌ 不稳定 | 交换跨度大，会跳过相同元素 |
| [[46-shell-sort\|希尔排序]] | ❌ 不稳定 | 多趟插入会打乱相对位置 |
| [[43-merge-sort\|归并排序]] | ✅ 稳定 | 合并时左优先于右 |
| [[44-quick-sort\|快速排序]] | ❌ 不稳定 | partition 阶段会换位 |
| [[45-heap-sort\|堆排序]] | ❌ 不稳定 | sink 过程会换位 |
| [[47-counting-sort\|计数排序]] | ✅ 稳定 | 反向填充可保证稳定 |
| [[48-bucket-sort\|桶排序]] | ✅ 稳定 | 桶内排序 + 按桶序合并 |
| [[49-radix-sort\|基数排序]] | ✅ 稳定 | 对每位用稳定计数排序 |

> [!info] 「为什么要求稳定」的本质
> 当你 **多次按不同 key 排序** 时，只有稳定排序能让你之前的排序工作不被浪费。这是数据库 ORDER BY、多级排序、分组聚合等场景的基础。

## 4. 指标三：是否原地排序

> **原地排序**：排序过程中 **不需要额外的辅助空间**，只使用常数级别的额外空间，直接操作原数组。
>
> 关键不是「是否返回新数组」，而是「**是否需要 O(N) 的额外数组**」。

```cpp
// ❌ 非原地排序：需要 O(N) 的辅助数组
void sort(vector<int>& nums) {
    vector<int> tmp(nums.size());   // O(N) 额外空间
    // ... 用 tmp 辅助排序
}

// ✅ 原地排序：只用 O(1) 额外空间
void sort(vector<int>& nums) {
    // 直接在 nums 上交换、搬移
}
```

### 4.1 为什么原地排序很重要？

- **大数据量**：10 亿个整数排序时，多一个 10GB 的辅助数组意味着要向 OS 多申请 10GB 内存。
- **硬件友好**：原地排序对缓存友好，因为始终在原数组上操作，CPU cache 命中率更高。
- **嵌入式/受限环境**：在内存极其有限的场景（如嵌入式设备），原地排序几乎是唯一选择。

### 4.2 常见排序算法的「原地性」

| 算法 | 是否原地 | 额外空间 |
|------|---------|---------|
| [[42-select-sort\|选择排序]] | ✅ | O(1) |
| [[40-bubble-sort\|冒泡排序]] | ✅ | O(1) |
| [[41-insertion-sort\|插入排序]] | ✅ | O(1) |
| [[46-shell-sort\|希尔排序]] | ✅ | O(1) |
| [[44-quick-sort\|快速排序]] | ✅ | O(log N)（递归栈） |
| [[45-heap-sort\|堆排序]] | ✅ | O(1) |
| [[43-merge-sort\|归并排序]] | ❌ | O(N) |
| [[47-counting-sort\|计数排序]] | ❌ | O(max - min) |
| [[48-bucket-sort\|桶排序]] | ❌ | O(N + k) |
| [[49-radix-sort\|基数排序]] | ❌ | O(N + k) |

## 5. 三个指标的综合权衡

实际工程中，排序算法的选择往往是 **三个指标的权衡**，没有绝对的「最优」：

| 场景 | 推荐算法 | 理由 |
|------|---------|------|
| 通用、大数据量 | [[44-quick-sort\|快速排序]] | O(N log N) + 原地 + 缓存友好 |
| 要求稳定 + 大数据量 | [[43-merge-sort\|归并排序]] | O(N log N) + 稳定，可外部排序 |
| 要求稳定 + 数据有限 | [[47-counting-sort\|计数排序]] | O(N + k)，数据范围小时无敌 |
| Top K 问题 | [[45-heap-sort\|堆排序]] / [[22-binary-heap-implement\|优先级队列]] | O(N log K) |
| 几乎有序的小数据 | [[41-insertion-sort\|插入排序]] | 接近 O(N)，常数极小 |

## 6. 排序算法对比总结表

| 算法 | 平均时间 | 最坏时间 | 最好时间 | 空间 | 稳定性 | 是否原地 |
|------|---------|---------|---------|------|--------|---------|
| 选择排序 | O(N²) | O(N²) | O(N²) | O(1) | ❌ | ✅ |
| 冒泡排序 | O(N²) | O(N²) | O(N)（加 flag） | O(1) | ✅ | ✅ |
| 插入排序 | O(N²) | O(N²) | O(N) | O(1) | ✅ | ✅ |
| 希尔排序 | O(N log N) ~ O(N^1.5) | O(N²) | O(N) | O(1) | ❌ | ✅ |
| 快速排序 | O(N log N) | O(N²) | O(N log N) | O(log N) | ❌ | ✅ |
| 归并排序 | O(N log N) | O(N log N) | O(N log N) | O(N) | ✅ | ❌ |
| 堆排序 | O(N log N) | O(N log N) | O(N log N) | O(1) | ❌ | ✅ |
| 计数排序 | O(N + k) | O(N + k) | O(N + k) | O(k) | ✅ | ❌ |
| 桶排序 | O(N + k) | O(N²) | O(N) | O(N + k) | ✅ | ❌ |
| 基数排序 | O(N * d) | O(N * d) | O(N * d) | O(N + k) | ✅ | ❌ |

> 上表中 `k` 为数据范围，`d` 为位数。
