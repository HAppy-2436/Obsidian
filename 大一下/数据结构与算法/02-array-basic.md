# 数组（顺序存储）的原理

> [!info] 章节定位
> 本章是「数据结构的第一个」。从数组入手，理解「连续内存 + 随机访问 + 增删搬移 + 扩容」这四个核心概念，为后面 [[03-array-implement|动态数组实现]]、[[04-cycle-array|环形数组]]、[[11-hashtable-with-array|数组哈希表]] 打基础。

## 📚 学习目标

- 区分「静态数组」与「动态数组」两种语境
- 理解数组「随机访问 O(1)」的原理：首地址 + 偏移量
- 掌握数组「增、删、查、改」的时间复杂度及其根本原因
- 理解「数据搬移」和「扩容」的概念
- 知道标准库（C++ `std::vector` / Java `ArrayList` / Python `list`）底层就是动态数组

## 🎯 一句话总结

> 数组是「**一块连续的内存**」，靠「**首地址 + 索引 × 元素大小**」实现 O(1) 随机访问；连续性同时让它在中间增删时要搬移数据，满了还要扩容。

## 📖 正文

### 1. 静态数组 vs 动态数组

我们平时说「数组」时，含义往往是模糊的。本章先把说辞统一：

| 类别 | 含义 | 例子 |
| --- | --- | --- |
| 静态数组 | 一块**固定大小、连续**的内存，按索引访问 | C/C++ `int arr[10]`、Java `int[] arr = new int[10]` |
| 动态数组 | 静态数组 + 增删查改 API（push / insert / remove）+ 自动扩缩容 | C++ `std::vector`、`java.util.ArrayList`、Python `list` |

> 静态数组是「数组」的**原始形态**；动态数组是「在静态数组之上加了一组管理 API」。

本章先讲静态数组的原理（理解随机访问和搬移的根源），下一章 [[03-array-implement]] 再动手实现一个动态数组。

### 2. 静态数组的内存模型

```text
int arr[5] = {10, 20, 30, 40, 50};
```

这行代码做了三件事：

1. **在内存中开辟一段连续的 5 × 4 = 20 字节**（`int` 占 4 字节）。
2. **把数组名 `arr` 作为指针**指向这段内存的**首地址**。
3. **把这 5 个 int 依次写入**这 20 字节。

从硬件视角看，数组就是一段「首地址 + 长度 + 元素宽度」的连续字节流：

```text
首地址  +0  +1  +2  +3  |  +4  +5  +6  +7  |  +8  ...
        [   int 0   ]     [   int 1   ]      [   int 2  ]
          10  00  00  00     14  00  00  00     1E  00  00  00
        (小端序示意)
```

### 3. 数组索引为什么从 0 开始

`arr[i]` 的寻址方式是：

```text
arr + i * sizeof(element)
```

`arr` 是首地址，乘 `sizeof(int) = 4` 得到偏移量，加起来就是要访问的字节位置。

如果索引从 **1** 开始，那 `arr[1]` 应该是第一个元素，但 `arr + 1 * 4` 实际指向的是第二个元素。要让「索引 1 = 第一个元素」，只能改成 `arr + (i-1) * 4`，每次都多减一次。这是「0-based」成为主流的根本原因——**少一次运算**。

对应的推论：

- `arr` 本身就是指向首元素的指针，所以 `*arr == arr[0]`。
- `arr + 1` 指向第二个元素，`*(arr + 1) == arr[1]`。
- `&arr[i] == arr + i`，是「第 i 个元素的地址」。

> [!tip] 给初学者
> 指针运算和数组下标在 C/C++ 中几乎是一回事：编译器把 `arr[i]` 翻译成 `*(arr + i)`。理解这个等式，就理解了为什么 `2[arr]` 在 GCC 下也能编译通过（虽然别这么写）。

### 4. 随机访问为什么是 O(1)

「随机访问」（random access）= 给定索引 `i`，**直接拿到元素**。

由上面的寻址公式，只要知道 `i`，一步就能算出内存地址，然后 CPU 用一次访存指令读出来。**与数组大小无关**。

这就是「**连续内存的代价是搬移，回报是 O(1) 随机访问**」。

> [!warning] 反过来：值找索引不是 O(1)
> 如果给你一个值 `v`，让你找它在数组里的下标，你**只能**遍历数组，时间复杂度是 `O(n)`。这是初学者最容易混淆的点：「数组是 O(1) 查」严格说是「按索引查」是 O(1)，「按值查」是 O(n)。

### 5. 增删查改 API 复杂度

数据结构的职责是「**增删查改**」。下面把数组这四个操作摊开看。

#### 5.1 增（insert）

| 场景 | 复杂度 | 是否需要搬移 |
| --- | --- | --- |
| 末尾追加（append） | `O(1)` | 否 |
| 中间插入 | `O(n)` | 是，要给新元素腾位置 |
| 头部插入 | `O(n)` | 是，要搬移全部 |
| 空间已满 | `O(n)` | 是，要扩容并拷贝 |

**末尾追加的伪代码**：

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int arr[10] = {1, 2, 3, 4, 0, 0, 0, 0, 0, 0};
    int size = 4;            // 已有 4 个元素
    arr[size++] = 5;          // 末尾追加
    arr[size++] = 6;
    return 0;
}
```

**中间插入（索引 2 位置插 666）的伪代码**：

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int arr[10] = {1, 2, 3, 4, 0, 0, 0, 0, 0, 0};
    int size = 4;

    // 在索引 2 插入 666：先把索引 2 之后的元素整体后移一位
    for (int i = size - 1; i >= 2; i--) {
        arr[i + 1] = arr[i];
    }
    arr[2] = 666;
    size++;
    return 0;
}
```

> [!warning] 必须从后往前遍历
> 如果从前往后遍历，`arr[2] = arr[1]` 会把 `arr[1]` 的原始值覆盖，后续就丢了。从后往前可以保证「被搬的元素」还没被覆盖。

**扩容（resizing）的伪代码**：

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int arr[10];
    for (int i = 0; i < 10; i++) arr[i] = i;   // 已经装满

    // 申请一块 2 倍大小的新数组
    int newArr[20];
    for (int i = 0; i < 10; i++) newArr[i] = arr[i];
    // arr 此刻之后由 GC / 操作系统回收
    newArr[10] = 10;  // 在新数组末尾追加
    return 0;
}
```

扩容 = 申请新空间 + 拷贝旧数据 + 释放旧空间。`O(n)`。

> [!tip] 扩容的「摊销复杂度」
> 扩容单次很贵（O(n)），但不是每次插入都扩容——通常是 2 倍扩容。所以把 N 次插入的总开销均摊到每次，就是 O(1) 「摊销时间复杂度」。这个概念在 [[39-sort-basic|排序]] 章节会再次出现。

#### 5.2 删（remove）

| 场景 | 复杂度 | 是否需要搬移 |
| --- | --- | --- |
| 末尾删除 | `O(1)` | 否 |
| 中间删除 | `O(n)` | 是，要填补空位 |
| 头部删除 | `O(n)` | 是，要搬移全部 |

**中间删除（删 `arr[1]`）的伪代码**：

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int arr[10] = {1, 2, 3, 4, 5, 0, 0, 0, 0, 0};
    int size = 5;

    // 删除 arr[1]：把 [2..size-1] 整体前移一位
    for (int i = 1; i < size - 1; i++) {
        arr[i] = arr[i + 1];
    }
    arr[size - 1] = 0;  // 占位标记（动态数组里要清空引用）
    size--;
    return 0;
}
```

> [!tip] 必须从前往后遍历
> 删除时从前往后搬移可以保证「被搬的元素」是「源」而不是「目的」。如果从后往前搬，会先把 `arr[4] = arr[5]`（无意义），再 `arr[3] = arr[4]`，把刚搬过来的值覆盖了。

#### 5.3 查 / 改（get / set）

给定索引，**一步算地址** + **一次访存**，所以：

| 操作 | 复杂度 | 说明 |
| --- | --- | --- |
| `arr[i]` 读 | `O(1)` | 随机访问 |
| `arr[i] = v` 写 | `O(1)` | 随机访问 |
| 按值查索引 | `O(n)` | 必须遍历 |
| 范围查 `[l, r]` | `O(r - l)` | 部分遍历 |

### 6. 静态数组的本质总结

```text
随机访问的超能力 ←—— 连续的内存空间 ——→ 增删时的搬移代价
```

> 一个人最大的优势往往也是他的最大劣势。数组的连续性给了它 O(1) 随机访问，但也让它在增删时不得不搬移数据、在满了之后不得不扩容。

### 7. 动态数组：把「增删查改」封装成 API

静态数组太「原始」，写算法题时几乎不用。**实际工作**中我们用动态数组，它的本质是「**底层一个静态数组 + 增删查改 API + 自动扩缩容**」。

下面用 C++ 的 `std::vector` 演示动态数组的常见操作（其他语言 API 几乎一一对应）：

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    // 创建动态数组（不用显式指定大小）
    vector<int> arr;

    // 末尾追加：摊销 O(1)
    for (int i = 0; i < 10; i++) {
        arr.push_back(i);
    }

    // 中间插入：O(n)
    arr.insert(arr.begin() + 2, 666);

    // 头部插入：O(n)
    arr.insert(arr.begin(), -1);

    // 末尾删除：O(1)
    arr.pop_back();

    // 中间删除：O(n)
    arr.erase(arr.begin() + 2);

    // 按索引访问：O(1)
    int a = arr[0];

    // 按索引修改：O(1)
    arr[0] = 100;

    // 按值查找索引：O(n)
    auto it = find(arr.begin(), arr.end(), 666);
    int index = (it != arr.end()) ? (int)(it - arr.begin()) : -1;

    return 0;
}
```

> [!tip] 动态数组没有解决「搬移」
> 很多人误以为动态数组 = 增删头部也 O(1)，这是错的。**动态数组的底层仍然是静态数组**，它在中间增删时一样要搬移，只是把搬移藏在了 API 内部。
>
> 如果你**真的**需要「头尾都 O(1)」，见 [[04-cycle-array|环形数组]] 和 [[07-deque-implement|双端队列]]。

### 8. 数组 vs 链表：开篇对照

数组的「连续」既给了它超能力也带来了局限；后面的 [[05-linkedlist-basic|链表]] 章节会讲「**链式存储**」——元素分散在内存中、靠指针串起来，恰好与数组互补。

提前预告一下复杂度对照（详细对比见 [[05-linkedlist-basic|链表原理]]）：

| 操作 | 数组 | 链表 |
| --- | --- | --- |
| 按索引读 | `O(1)` | `O(n)` |
| 末尾增 | 摊销 `O(1)` | `O(n)`（无尾指针） |
| 头部增 | `O(n)` | `O(1)` |
| 中间增 | `O(n)` | `O(n)`（要先找位置） |
| 按值查 | `O(n)` | `O(n)` |
| 内存占用 | 紧凑、无额外开销 | 额外存指针（双链表两倍） |

## 📊 复杂度一览

| 操作 | 静态数组 | 动态数组（摊销） | 是否搬移 |
| --- | --- | --- | --- |
| 按索引读 / 写 | `O(1)` | `O(1)` | 否 |
| 末尾追加 | `O(1)` | **摊销 `O(1)`**（单次扩容 O(n)） | 偶尔搬移 |
| 头部插入 | `O(n)` | `O(n)` | 全部搬移 |
| 中间插入 | `O(n)` | `O(n)` | 部分搬移 |
| 末尾删除 | `O(1)` | **摊销 `O(1)`**（含缩容） | 否 |
| 头部 / 中间删除 | `O(n)` | `O(n)` | 全部 / 部分搬移 |
| 按值查索引 | `O(n)` | `O(n)` | — |

## 🛠️ 应用场景

- **数据缓冲 / 批量访问**：图像像素、矩阵、CSV 行——天然连续，索引访问频繁。
- **栈 / 队列的底层**（静态版本）：用数组实现栈，push/pop 都是 O(1)，见 [[08-queue-stack-basic|队列/栈原理]]。
- **哈希表的桶数组**：见 [[11-hashtable-with-array|数组哈希表]]、[[13-hashmap-basic|哈希表原理]]。
- **堆的底层存储**（完全二叉树）：见 [[21-binary-heap-basic|二叉堆]]，堆用数组存比链表省一半内存。
- **前缀和 / 差分**：数组的随机访问让 O(1) 区间查询成为可能。

> 不要用的场景：**频繁在头部插入**（改用 [[04-cycle-array|环形数组]] / [[06-linkedlist-implement|双链表]] / [[07-deque-implement|Deque]]）；**大小事先未知且增长极不可预测**（改用链表或预分配大数组）。
