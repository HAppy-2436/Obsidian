# 图的基本术语

## 学习目标

- 理解图（Graph）的数学定义：**节点（Vertex）+ 边（Edge）**
- 掌握**有向/无向、加权/无权**这四种基本组合
- 理解**度、入度、出度**的概念
- 理解**稠密图 vs 稀疏图**的判定方式
- 掌握**子图、连通图、连通分量**等基本术语
- 区分**强连通、弱连通、强连通分量（SCC）、弱连通分量（WCC）**

## 一句话总结

图（Graph）由**节点（Vertex）**和**边（Edge）**组成；边可以有方向（**有向图**）和权重（**加权图**），并据此派生出**度/入度/出度、连通性、子图、连通分量、SCC** 等术语——这些是后续图论算法（最短路、最小生成树、欧拉图）的**共同词汇表**。

## 1. 图的数学定义

一幅图（Graph）由两个集合构成：

- **节点集 V（Vertex）**：图中的顶点，每个顶点有**唯一 ID**（通常是 0 ~ V-1 的整数）。
- **边集 E（Edge）**：节点对 `(u, v)` 的集合，表示节点之间的连接关系。

> 图论起源于 18 世纪欧拉对**哥尼斯堡七桥问题**的研究，是数据结构中**最一般化**的结构——树、链表、数组都可以视为图的特例。

```
图 = (V, E)
V = {0, 1, 2, 3, 4}     ← 5 个节点
E = {(0,1), (1,2), (2,3)}  ← 3 条边
```

## 2. 边的方向：四种基本图

### 2.1 有向图 vs 无向图

| 类型 | 边的形式 | 含义 |
|------|----------|------|
| **有向图（Directed Graph）** | `<u, v>` 是一条有方向的边 | 可以从 u 走到 v，但不一定能从 v 走到 u |
| **无向图（Undirected Graph）** | `(u, v)` 是一条无方向的边 | u 和 v 互相可达（等价于两条方向相反的有向边） |

> [!tip] 代码实现上的小技巧
> 在代码中我们**一般不区分有向/无向图**，而是统一用有向边存储。无向图的边 `(u, v)` 就**存两条有向边** `<u, v>` 和 `<v, u>` 即可。

### 2.2 加权图 vs 无权图

| 类型 | 边上的属性 | 应用 |
|------|----------|------|
| **加权图（Weighted Graph）** | 边带**权重**（距离、成本、时间） | 地图导航、网络路由、电路布线 |
| **无权图（Unweighted Graph）** | 边没有权重，只表示**连通性** | 社交网络、网页链接 |

### 2.3 四种组合

边**有方向**和**有权重**是两个**正交**的属性，组合后得到 4 种图：

1. **有向无权图**（最常见，比如 Twitter 关注关系）
2. **无向无权图**（比如 Facebook 好友关系）
3. **有向加权图**（比如带单行线的地图）
4. **无向加权图**（比如无向带距离的地图）

> 加权图衍生出大量经典算法：**最短路径（Dijkstra / Floyd）、最小生成树（Kruskal / Prim）** 等，这些会在 [[36-graph-minimum-spanning-tree|最小生成树]] 和 [[37-graph-shortest-path|最短路径]] 章节详细讲解。

## 3. 度（Degree）

### 3.1 无向图的度

对于无向图中的节点 v，**度（degree）** 就是与 v 相连的**边的条数**。例如下面的无向图中，节点 1 的度为 2，节点 4 的度为 4（连接到其他 4 个节点）。

### 3.2 有向图的入度和出度

有向图的边有方向，所以度被细分为：

- **入度（in-degree）**：有多少条边**指向**该节点
- **出度（out-degree）**：该节点有多少条边**指向**别人

例如节点 3 的入度为 2（两条边指向它），出度为 1（它有一条边指向别的节点）。

> **重要公式**：对于**任意无向图**，`Σ degree(v) = 2 × |E|`（每条边对度数的总贡献为 2）。这个等式在证明很多图论性质时非常有用。

## 4. 边和节点的数量关系

### 4.1 简单图（Simple Graph）

我们一般讨论的都是**简单图（Simple Graph）**——即：

- **没有自环边（Self-loop）**：不存在 `v -> v` 的边
- **没有多重边（Multiple edges）**：任意两个节点之间至多一条边

### 4.2 边的取值范围

在有 V 个节点的简单图中：

- **E 的最小值** = 0（图中只有孤立节点）
- **E 的最大值** = V(V-1) ≈ V²（任意两节点之间都有一条边）

### 4.3 稠密图 vs 稀疏图

根据 E 接近最大值还是最小值，分为：

| 类型 | E 的量级 | 典型存储 |
|------|---------|----------|
| **稠密图（Dense Graph）** | E ≈ V² | [[33-graph-basic\|邻接矩阵]] |
| **稀疏图（Sparse Graph）** | E ≪ V² | [[33-graph-basic\|邻接表]] |

> 一个常用的经验阈值：**E > V × log V** 时倾向于稠密，否则倾向于稀疏。但实际工程中**几乎所有的图都是稀疏的**（比如社交网络、道路网），所以邻接表是最常用的存储方式。

## 5. 子图（Subgraph）

**子图（Subgraph）**：如果图 G' 的所有节点和边都包含在图 G 中，则称 G' 是 G 的一个子图。简单来说，子图是从原图中**删除一些节点和边**后得到的图。

子图有两种特殊类型：

### 5.1 生成子图（Spanning Subgraph）

- 包含**原图所有节点**
- 只包含**部分边**
- 例：图 G 的一个生成子图，移除了节点 3 和节点 4 之间的边

### 5.2 导出子图（Induced Subgraph）

- 选择原图的**一部分节点**
- 以及这些节点之间在原图中的**所有边**
- 例：图 G 的一个导出子图，包含节点 {1,2,3,4} 及它们之间在原图中的所有边

> **应用**：[[36-graph-minimum-spanning-tree|最小生成树]] 实际上就是**带权最小的生成子图**（包含所有节点，但只选 V-1 条边使权重和最小）。

## 6. 连通性（Connectivity）

连通性是图论中**最重要的概念**之一，描述了节点之间是否存在路径。

### 6.1 无向图的连通性

**连通图（Connected Graph）**：如果无向图中**任意两个节点**之间都存在一条路径，我们称这个图是连通的。

**连通分量（Connected Component）**：对于**非连通**的无向图，其中的多个连通子图被称为**连通分量**。一个图可以有多个连通分量。

例：一幅图有 2 个连通分量——节点 {1,2,3,4,5} 形成一个，节点 {6,7} 形成另一个。

### 6.2 有向图的连通性

有向图的连通性因为**边的方向**而变得复杂，分为强连通和弱连通。

| 概念 | 定义 |
|------|------|
| **强连通图（Strongly Connected Graph）** | 有向图中**任意两个节点**之间都存在**有向路径**（u→v 且 v→u 都有） |
| **弱连通图（Weakly Connected Graph）** | 把所有有向边**都变成无向边**后，图变成连通的 |
| **强连通分量（SCC）** | 有向图中的**最大的强连通子图** |
| **弱连通分量（WCC）** | 把有向图的所有有向边变为无向边后，形成的连通分量 |

> [!tip] 面试重点
> 实际算法面试中**主要考察无向图的连通性**（连通分量数量、最大连通子图等），有向图的 SCC 在高级算法（Tarjan / Kosaraju）中才会涉及。

## 7. 用 C++ 表示"图的术语"

下面给出一段 C++ 代码，把上面提到的**度、入度、出度、加权**等概念用 [[33-graph-basic|邻接表]] 落到了代码层面：

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace dsa {

// 一条带权重的有向边
struct Edge {
    int to;       // 邻居节点
    int weight;   // 权重（无权图可填 1）
    Edge(int t, int w = 1) : to(t), weight(w) {}
};

// 简化版图：邻接表 + 节点数 + 边数
class SimpleGraph {
public:
    SimpleGraph(int n) : n_(n), adj_(n) {}

    // 加有向边 u -> v
    void addEdge(int u, int v, int w = 1) { adj_[u].push_back(Edge(v, w)); }

    // 节点 v 的度 = 邻接表长度（无向图）；有向图为出度
    int outDegree(int v) const { return (int)adj_[v].size(); }

    // 节点 v 的入度 = 遍历所有邻接表统计 v 出现的次数
    int inDegree(int v) const {
        int cnt = 0;
        for (int u = 0; u < n_; ++u) {
            for (const auto& e : adj_[u]) if (e.to == v) ++cnt;
        }
        return cnt;
    }

    // 节点总数 / 边总数
    int V() const { return n_; }
    int E() const {
        int cnt = 0;
        for (int v = 0; v < n_; ++v) cnt += (int)adj_[v].size();
        return cnt;
    }

    // 简单打印邻接表
    void print() const {
        for (int v = 0; v < n_; ++v) {
            cout << v << " : ";
            for (const auto& e : adj_[v]) cout << "(" << e.to << ", w=" << e.weight << ") ";
            cout << endl;
        }
    }

private:
    int n_;
    vector<vector<Edge>> adj_;
};

} // namespace dsa

int main() {
    using namespace dsa;
    SimpleGraph g(4);
    g.addEdge(0, 1, 5);
    g.addEdge(0, 2, 3);
    g.addEdge(1, 2, 2);
    g.addEdge(2, 3, 7);

    cout << "节点数: " << g.V() << "  边数: " << g.E() << endl;
    cout << "节点 2 的出度: " << g.outDegree(2) << endl;
    cout << "节点 2 的入度: " << g.inDegree(2) << endl;
    g.print();
    return 0;
}
```

**输出**：

```text
节点数: 4  边数: 4
节点 2 的出度: 1
节点 2 的入度: 2
0 : (1, w=5) (2, w=3) 
1 : (2, w=2) 
2 : (3, w=7) 
3 : 
```

注意：**入度需要遍历整个邻接表**统计（O(V+E)），这是邻接表唯一的"短板"——对应到下面的"复杂度表"中邻接表的入度查询是 O(V+E)，而邻接矩阵是 O(V)。

## 8. 复杂度表

| 操作 | 邻接表 | 邻接矩阵 |
|------|--------|----------|
| 求某个节点的度（出度） | O(1) ~ O(deg(v)) | O(V) |
| 判断两节点是否有边 | O(deg(v)) | **O(1)** |
| 遍历所有边 | O(V + E) | O(V²) |
| 空间 | **O(V + E)** | O(V²) |

## 9. 应用场景

- **社交网络**：节点 = 用户，边 = 好友/关注关系
- **地图导航**：节点 = 地点，边 = 道路，权重 = 距离/时间
- **网页链接**：节点 = 网页，边 = 超链接（构成有向图，Google PageRank）
- **任务调度**：节点 = 任务，边 = 先后顺序（拓扑排序）
- **电路设计**：节点 = 元器件，边 = 导线
