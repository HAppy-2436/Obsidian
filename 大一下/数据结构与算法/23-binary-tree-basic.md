# 二叉树基本概念及特殊二叉树

## 学习目标

- 理解二叉树为什么是「最重要的基本数据结构」
- 掌握二叉树的 **节点 / 边 / 深度 / 高度 / 叶子 / 子树** 等基本术语
- 熟悉几种 **特殊二叉树**：满二叉树、完全二叉树、二叉搜索树（BST）、高度平衡二叉树、自平衡二叉树
- 理解二叉树的 **链式存储** vs **数组存储** vs **邻接表存储** 三种实现方式
- 了解为什么 **完全二叉树能用数组存储**（父子节点索引的数学关系）

## 一句话总结

二叉树是最重要的基本数据结构——既是后续 [[31-rbtree-basic|红黑树]]、[[21-binary-heap-basic|二叉堆]]、[[30-union-find-basic|并查集]]、[[26-segment-tree-basic|线段树]] 等高级结构的基底，也是 [[34-graph-traverse-basic|DFS / BFS]]、[[44-quick-sort|快速排序]]、[[43-merge-sort|归并排序]] 等算法的「思维模型」（一切暴力穷举 = 把问题抽象成树）。

## 1. 为什么二叉树这么重要？

二叉树本身比较简单，但它有两个不可替代的价值：

### 1.1 二叉树是高级数据结构的基底

很多复杂数据结构都是基于二叉树的：

| 高级结构 | 简述 |
|---------|------|
| [[31-rbtree-basic|红黑树]] | 自平衡的二叉搜索树 |
| [[21-binary-heap-basic|二叉堆]] | 完全二叉树 + 堆序 |
| [[26-segment-tree-basic|线段树]] | 二叉树 + 区间信息 |
| [[27-tree-map-basic|二叉搜索树]] | 左小右大的二叉树 |
| [[29-trie-map-basic|Trie 字典树]] | 多叉树（本质是 26 叉树） |
| [[30-union-find-basic|并查集]] | 用森林（多棵二叉树）表示集合 |

把二叉树玩明白，这些高级结构都不是问题；不把二叉树搞明白，它们也都驾驭不了。

### 1.2 二叉树是算法思维的核心

一切暴力穷举算法，比如 [[34-graph-traverse-basic|DFS / BFS]]、回溯、动态规划，**本质上都是把具体问题抽象成树结构**。只要抽象出来了，这些问题最终都回归二叉树的问题。一段算法代码在别人眼里是一串文本，在你眼里就是一棵树——这就是二叉树思维带来的「降维打击」。

## 2. 二叉树的基本术语

下面是一棵普通的二叉树（每个节点最多两个子节点）：

```
            1           ← 根节点（root）
          /   \
         2     3        ← 2 是 1 的左子，3 是 1 的右子
        / \   / \
       4  ?  5   6       ← 4 是 2 的左子，2 是 4 的父节点
```

### 2.1 核心术语

- **节点（Node）**：树中的基本单位，存储数据和指向子节点的指针。
- **父节点 / 子节点**：直接相连的两个节点中，上方的是父节点，下方的是子节点。
- **根节点（Root）**：最上方那个没有父节点的节点。
- **叶子节点（Leaf）**：最下层没有子节点的节点。
- **子树（Subtree）**：以某个节点为根的整棵树，包括它的所有后代。
- **深度 / 高度**：从根节点到该节点的路径上节点的个数（根节点深度 = 1）。
- **层（Level）**：根节点在第 1 层，子节点在第 2 层，依此类推。

### 2.2 树 vs 二叉树

- **树（Tree）**：每个节点的子节点数量不限（多叉树）。
- **二叉树（Binary Tree）**：每个节点**最多**有 2 个子节点（左、右）。
- **满二叉树（Perfect Binary Tree）**：除叶子外每个节点都有 2 个子节点，且所有叶子在同一层。
- **完全二叉树（Complete Binary Tree）**：除最后一层外每层都是满的，且最后一层的节点紧凑靠左排列。

## 3. 几种特殊的二叉树

### 3.1 满二叉树（Perfect Binary Tree）

```
              1
            /   \
           2     3
          / \   / \
         4   5 6   7
        /\  /\ /\  /\
       8 9 ...
```

满二叉树就像一个正三角形。深度为 `h` 的满二叉树总节点数为 `2^h - 1`（等比数列求和）。

> [!note] 中英文术语差异
> 中文「满二叉树」对应英文 **Perfect Binary Tree**（所有叶子在同一层）。英文的 **Full Binary Tree** 指的是「所有节点要么没孩子、要么有 2 个孩子」（不要求叶子在同一层）。

### 3.2 完全二叉树（Complete Binary Tree）

```
              1
            /   \
           2     3
          / \   /
         4  5  6
        /
       8
```

完全二叉树 = 满二叉树 + 最后一层可以不满，但必须 **紧凑靠左排列**。不难发现，**满二叉树是特殊的完全二叉树**。

完全二叉树的核心性质：**父子节点的索引存在数学关系**（这就是 [[21-binary-heap-basic|二叉堆]] 和 [[26-segment-tree-basic|线段树]] 能用数组存储的根本原因）。

完全二叉树的另一个隐秘性质：完全二叉树的左右子树中，**至少有一棵是满二叉树**。这个性质在做算法题（如 LeetCode 222「完全二叉树的节点个数」）时会用到。

### 3.3 二叉搜索树（BST）

**二叉搜索树（Binary Search Tree, BST）** 的定义：对于树中每个节点，**其左子树的每个节点的值都小于该节点的值，右子树的每个节点的值都大于该节点的值**。简称「左小右大」。

```
              7
            /   \
           4     9
          / \   / \
         1   5 8   10
```

注意：「子树的每个节点」加粗了，这是初学者常犯的错误——不要只看子节点，要看 **整棵子树的所有节点**。比如下图中节点 7 的左子树中有个节点 8 > 7，就 **不** 满足 BST：

```
              7
            /   \
           4     9
          / \   / \
         1   8 5   10   ← 8 在 7 的左子树，但 8 > 7，违反 BST
```

BST 的优势：左小右大让我们可以 **O(log N)** 定位任意节点、找某范围的所有节点。普通二叉树就只能 O(N) 遍历。

### 3.4 高度平衡二叉树（Height-Balanced Binary Tree）

**定义**：每个节点的左右子树的高度差 ≤ 1（注意是「每个节点」，不仅仅是根节点）。

```
              1           ← 左右子树高度差 = |2-3| = 1 ✓
            /   \
           2     3        ← 节点 2: |1-0|=1 ✓；节点 3: |2-1|=1 ✓
          /     / \
         4     5   6
        /
       7
```

**核心性质**：包含 N 个节点的高度平衡二叉树，高度是 **O(log N)**。

这是非常重要的性质——本站后面讲到的 [[27-tree-map-basic|二叉搜索树]] 和 [[31-rbtree-basic|红黑树]]，如果能保证树的高度是 O(log N)，那么增删查改效率就很高。反之，如果树退化成链表（极端不平衡），效率就降到 O(N)。

### 3.5 自平衡二叉树（Self-Balanced Binary Tree）

如果可以在增删节点时对树的结构做一些调整，让树的高度始终保持平衡，这就是 **自平衡二叉树**。

最经典的自平衡二叉树就是 [[31-rbtree-basic|红黑树]]——一种自平衡的二叉搜索树。保持平衡的关键操作叫 **「旋转」**（左旋 / 右旋），通过旋转可以在不破坏 BST 性质的前提下调整树的高度分布。

## 4. 二叉树的实现方式

### 4.1 方式一：链式存储（最常见）

每个节点有指向左右子节点的指针，这就是力扣/LeetCode 上的标准 `TreeNode`：

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace dsa {

// 二叉树节点（链式存储）
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

// 构建示例：    1
//             / \
//            2   3
//           /   / \
//          4   5   6
void buildDemoTree() {
    TreeNode* root = new TreeNode(1);
    root->left  = new TreeNode(2);
    root->right = new TreeNode(3);
    root->left->left = new TreeNode(4);
    root->right->left = new TreeNode(5);
    root->right->right = new TreeNode(6);
    // 使用完记得 delete（这里省略，避免代码冗长）
}

} // namespace dsa
```

### 4.2 方式二：数组存储（仅完全二叉树）

**完全二叉树** 因为节点紧凑，可以用数组存储，无需指针：

- **0-indexed**：`parent(i) = (i-1)/2`，`left(i) = 2i+1`，`right(i) = 2i+2`。
- **1-indexed**：`parent(i) = i/2`，`left(i) = 2i`，`right(i) = 2i+1`。

```cpp
// 完全二叉树的数组表示（1-indexed，下标 0 留空作为哨兵）
// 树形:
//              1
//            /   \
//           2     3
//          / \   /
//         4  5 6
// 数组: [_ 1 2 3 4 5 6]
int tree[7] = {0, 1, 2, 3, 4, 5, 6};
```

[[21-binary-heap-basic|二叉堆]] 和 [[26-segment-tree-basic|线段树]] 都用数组存储，因为它们本质是完全二叉树。

### 4.3 方式三：邻接表存储（哈希表模拟）

在算法题中经常需要把「逻辑上」的树结构映射到哈希表，而不需要真的创建 `TreeNode`：

```cpp
// 用 unordered_map 模拟一棵「父节点 → 子节点列表」的多叉树
// 树形:  1 → [2, 3]; 2 → [4]; 3 → [5, 6]
// 等价于:
//         1
//        / \
//       2   3
//      /   / \
//     4   5   6
unordered_map<int, vector<int>> tree;
tree[1] = {2, 3};
tree[2] = {4};
tree[3] = {5, 6};
```

这种表示法的另一个名字叫 **「邻接表」**，会在 [[33-graph-basic|图结构]] 章节里详细讲解。

### 4.4 方式四：递归树（可视化场景）

在递归算法的可视化中，每次递归调用都会生成一个「递归树节点」，这本身也算一种树的实现方式。这种递归树是 [[43-merge-sort|归并排序]]、[[44-quick-sort|快速排序]] 和 [[34-graph-traverse-basic|DFS 算法]] 的可视化基础。

## 5. 复杂度表

| 操作 | 平衡二叉树 | 完全二叉树 | 普通二叉树 |
|------|----------|----------|----------|
| 查找最值 | O(log N) | O(1)（堆顶） | O(N) |
| 插入/删除 | O(log N) | O(log N) | O(N)（退化为链表时） |
| 遍历全部节点 | O(N) | O(N) | O(N) |
| 空间 | O(N) | O(N) | O(N) |

## 6. 应用场景

- **二叉搜索树（BST）**：见 [[27-tree-map-basic|BST 原理]]，O(log N) 增删查改。
- **红黑树**：见 [[31-rbtree-basic|红黑树原理]]，自平衡 BST，C++ `std::map` / `std::set` 底层。
- **二叉堆**：见 [[21-binary-heap-basic|二叉堆原理]]，O(1) 取最值 + O(log N) 插入。
- **线段树**：见 [[26-segment-tree-basic|线段树原理]]，区间查询 / 区间更新。
- **Trie**：见 [[29-trie-map-basic|Trie 字典树]]，多叉树，前缀匹配。
- **并查集**：见 [[30-union-find-basic|并查集原理]]，用森林表示不相交集合。
- **算法思维**：DFS / BFS / 回溯 / 动态规划的「可视化模型」。
