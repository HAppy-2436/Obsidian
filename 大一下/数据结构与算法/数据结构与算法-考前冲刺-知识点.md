## 第 1 章 课程介绍

### 1.1 理论全掌握

#### 1.1.1 数据结构基本概念

- **数据（Data）**：客观事物的符号表示，是所有能输入到计算机并被程序处理的符号的总称。
- **数据元素（Data Element）**：数据的基本单位，通常作为一个整体考虑和处理（如一个学生、一条记录）。
- **数据项（Data Item）**：数据元素的不可分割的最小单位（如学生的学号、姓名）。
- **数据对象（Data Object）**：性质相同的数据元素的集合（如所有学生）。
- **数据结构（Data Structure）**：**相互之间存在一种或多种特定关系的数据元素的集合**。包括三方面：
  1. **数据的逻辑结构**：数据元素之间的逻辑关系（线性、树、图、集合）。
  2. **数据的存储结构（物理结构）**：数据在计算机中的存储方式（顺序、链式、索引、散列）。
  3. **数据的运算**：对数据施加的操作（增删改查、排序等）。

#### 1.1.2 逻辑结构四大类

| 类型 | 关系特征 | 典型 |
|---|---|---|
| **集合** | 同属一个集合，无其他关系 | 集合 |
| **线性** | 一对一 | 数组、链表、栈、队列 |
| **树** | 一对多 | 二叉树、堆 |
| **图** | 多对多 | 社交网络、地图 |

#### 1.1.3 存储结构四种

- **顺序存储**：逻辑上相邻的元素物理上也相邻（数组）。优点：随机访问 O(1)。缺点：插入删除 O(n)，需要连续空间。
- **链式存储**：元素任意存储，靠指针维系关系（链表）。优点：插入删除 O(1)。缺点：不能随机访问，额外指针开销。
- **索引存储**：建立索引表（关键字 → 地址）。优点：查找快。缺点：索引占空间、维护索引开销。
- **散列（哈希）存储**：用哈希函数直接算出地址。优点：期望 O(1) 查找。缺点：冲突处理复杂、不支持顺序访问。

#### 1.1.4 抽象数据类型（ADT）

- **定义**：一个数学模型 + 定义在该模型上的一组操作。**只关心"做什么"，不关心"怎么做"**。
- **三要素**：
  1. **数据对象（Data Object）**
  2. **数据关系（Data Relation）**
  3. **基本操作（Basic Operations）**
- **表示格式**：
  ```
  ADT 抽象数据类型名
  Data
      数据对象的定义
  Relation
      数据关系的定义
  Operation
      操作 1：初始条件 / 操作结果
      操作 2：...
  endADT
  ```
- **实现**：ADT 通过具体的存储结构和算法实现（C++ 的 class / struct）。

#### 1.1.5 算法五大特性

1. **有穷性**：必须在有限步后终止。
2. **确定性**：每一步骤无歧义。
3. **可行性**：每一步都能执行。
4. **输入**：零个或多个输入。
5. **输出**：一个或多个输出。

#### 1.1.6 算法评价指标

- **正确性**：满足问题需求。
- **可读性**：清晰易理解。
- **健壮性**：对非法输入能处理而非崩溃。
- **效率**：
  - **时间复杂度 T(n)**：基本操作次数随 n 的增长趋势（用大 O 表示）。
  - **空间复杂度 S(n)**：算法占用的额外存储空间。

#### 1.1.7 时间复杂度分析

- **常见量级**（从快到慢）：
  `O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)`
- **计算方法**：
  - 顺序语句：加法
  - 选择语句：取 max
  - 循环语句：乘积
  - 递归：递推方程（主定理）
- **最好 / 最坏 / 平均情况**：
  - 最好：最佳输入
  - 最坏：最差输入
  - 平均：所有输入等概率时的期望
- **摊还时间**：一系列操作的总成本平均到每次（splay 树摊还 O(log n)）。

#### 1.1.8 主定理（Master Theorem）

递归式 `T(n) = aT(n/b) + O(n^d)`：
- `d > log_b a` → `T(n) = O(n^d)`
- `d = log_b a` → `T(n) = O(n^d log n)`
- `d < log_b a` → `T(n) = O(n^log_b a)`

**例**：归并排序 `T(n) = 2T(n/2) + O(n)`，a=2, b=2, d=1，`log_b a = 1 = d` → O(n log n)。

#### 1.1.9 C++ 类（Lab 必写）

```cpp
class MyClass {
public:
    MyClass();                                  // 默认构造
    MyClass(const MyClass& other);              // 拷贝构造
    MyClass(MyClass&& other);                   // 移动构造（C++11）
    MyClass& operator=(const MyClass& other);   // 拷贝赋值
    MyClass& operator=(MyClass&& other);        // 移动赋值
    ~MyClass();                                 // 析构
    // 业务方法
    void push(int x);
    int pop();
    int top() const;                            // const 成员函数不修改对象
    int size() const;
    bool empty() const;
private:
    int* data;
    int sz;
    int cap;
};
```

- **Rule of Three**：如果定义了析构、拷贝构造、拷贝赋值中任何一个，通常三个都要定义（资源管理）。
- **Rule of Five**（C++11）：再加移动构造、移动赋值。
- **const 正确性**：不修改对象的成员函数加 `const`。
- **RAII**：资源获取即初始化（智能指针 `unique_ptr` / `shared_ptr`）。

#### 1.1.10 数组

- **一维数组**：连续存储，类型相同。下标 `0..n-1`。
- **二维数组**：行优先 / 列优先存储。多维 → 一维映射：`a[i][j]` = `*(base + i*col + j)`。
- **时间复杂度**：
  - 随机访问 O(1)
  - 查找（无序）O(n)
  - 插入/删除 O(n)
  - 遍历 O(n)
- **特殊矩阵压缩**：
  - 对称矩阵：存一半（下三角）
  - 三角矩阵
  - 稀疏矩阵：三元组 / 十字链表

#### 1.1.11 指针与内存

- **指针**：存地址的变量。`int* p = &x;`
- **空指针**：`nullptr`（C++11 起，替代 `NULL`）
- **指针运算**：`p + 1` 跳过一个类型大小
- **动态内存**：`new` / `delete`（C++）或 `malloc` / `free`（C）
- **内存泄漏**：申请了没释放
- **悬挂指针**：指向已释放的内存
- **栈内存**：函数局部变量、自动管理
- **堆内存**：`new` 出来的、手动管理（智能指针更好）

#### 1.1.12 Life Game 通用思路

- **规则核心**：每个 cell 根据邻居状态决定下一代的生死。
- **编程套路**：
  1. 维护**两份网格**（当前 + 下一代），避免原地修改
  2. 每代遍历所有 cell，统计邻居数
  3. 按规则更新下一代
  4. 交换指针 / 数组，重复
- **关键技巧**：
  - 边界处理（无限网格 vs 有限网格）
  - 邻居方向（4 邻 / 8 邻 / 自定义）
  - 输出格式（`-` / `*` / `0` / `1`）



## 第 2 章 栈

### 2.1 理论全掌握

#### 2.1.1 栈的定义与性质

- **栈（Stack）**：限定仅在**表尾**进行插入和删除操作的线性表。
- **LIFO**（Last In First Out）：后进先出。
- **栈顶（Top）**：允许操作的一端。
- **栈底（Bottom）**：固定不动的一端。
- **关键术语**：
  - **入栈 / 压栈（Push）**：栈顶插入元素
  - **出栈 / 弹栈（Pop）**：栈顶删除元素
  - **读栈顶（Top/Peek）**：看栈顶元素但不删除
  - **栈空（Empty）**、**栈满（Full）**（仅顺序栈）
- **ADT 接口**：
  ```
  ADT Stack
  Data
      一维数组 + 栈顶指针
  Operation
      InitStack(&S)
      DestroyStack(&S)
      ClearStack(&S)
      StackEmpty(S)
      StackLength(S)
      GetTop(S, &e)
      Push(&S, e)
      Pop(&S, &e)
  ```

#### 2.1.2 顺序栈（顺序存储）

- **结构**：数组 + 栈顶指针 `top`。
- **约定**：
  - `top = -1` 表示空栈
  - `top` 指向**栈顶元素**（不是下一个空位）
  - 进栈：`data[++top] = x`
  - 出栈：`return data[top--]`
  - 栈满：`top == capacity - 1`
- **另一种约定**（栈顶指向下一个空位）：
  - `top = 0` 表示空栈
  - 进栈：`data[top++] = x`
  - 出栈：`return data[--top]`
  - 栈满：`top == capacity`
- **时间复杂度**：所有操作 O(1)
- **空间复杂度**：O(capacity)
- **优缺点**：
  - ✅ 简单、随机访问、CPU 缓存友好
  - ❌ 容量固定（浪费 or 溢出）

**C++ 实现**：
```cpp
class Stack {
    static const int CAP = 105;
    int data[CAP];
    int top;
public:
    Stack(): top(-1) {}
    bool empty() const { return top == -1; }
    bool full()  const { return top == CAP-1; }
    int  size()  const { return top + 1; }
    void clear() { top = -1; }
    bool push(int x) { if (full()) return false; data[++top] = x; return true; }
    bool pop()        { if (empty()) return false; top--; return true; }
    int  peek() const { return data[top]; }
};
```

#### 2.1.3 两栈共享空间

- **场景**：两个**同类型**栈，需求此消彼长（如一个栈大时另一个小）。
- **方法**：一个数组从两端向中间生长。
  - 栈 1 从左端：`top1 = -1` 初始
  - 栈 2 从右端：`top2 = n` 初始
  - 栈满条件：`top1 + 1 == top2`
- **优点**：节省空间
- **缺点**：只能两栈共享

#### 2.1.4 链栈（链式存储）

- **结构**：单链表，**头插法**（头节点处即栈顶）。
- **优点**：
  - 不需要预先分配空间
  - 不存在栈满（除非系统内存耗尽）
  - 长度动态
- **缺点**：
  - 每个节点多一个指针
  - 不可随机访问
- **时间复杂度**：所有操作 O(1)
- **C++ 实现**：
```cpp
struct Node { int val; Node* next; Node(int v): val(v), next(nullptr) {} };
class LinkedStack {
    Node* topNode;
    int cnt;
public:
    LinkedStack(): topNode(nullptr), cnt(0) {}
    ~LinkedStack() { clear(); }
    bool empty() const { return cnt == 0; }
    int  size()  const { return cnt; }
    void clear() { while (topNode) { Node* t = topNode; topNode = topNode->next; delete t; } cnt = 0; }
    void push(int x) { Node* n = new Node(x); n->next = topNode; topNode = n; cnt++; }
    void pop()        { Node* t = topNode; topNode = topNode->next; delete t; cnt--; }
    int  top() const  { return topNode->val; }
};
```

#### 2.1.5 顺序栈 vs 链栈

| 维度 | 顺序栈 | 链栈 |
|---|---|---|
| 存储 | 数组 | 链表 |
| 容量 | 固定 | 动态 |
| 空间利用率 | 可能浪费 | 灵活 |
| 访问速度 | 快（连续） | 慢（指针） |
| 判满 | 需要 | 不需要 |
| 实现难度 | 简单 | 中等 |

#### 2.1.6 栈的应用（核心）

**(1) 括号匹配**
- 思路：扫描字符串，遇左括号入栈，遇右括号弹栈顶匹配
- 时间 O(n)，空间 O(n)
- 关键：处理嵌套、注释、字符串字面量

**(2) 表达式求值**
- **中缀表达式**（人写）：`2+3*4`
- **前缀表达式 / 波兰式**（运算符前）：`+ 2 * 3 4`
- **后缀表达式 / 逆波兰式**（运算符后）：`2 3 4 * +`
- **后缀求值算法**：
  - 扫描 token
  - 数字：入栈
  - 运算符：弹两个数（b = 栈顶, a = 次栈顶），计算 `a op b`，结果入栈
  - 结束：栈底为结果
- **中缀 → 后缀（调度场算法，Shunting-yard）**：
  - 数字：直接输出
  - `(`：入栈
  - `)`：弹栈到 `(` 为止
  - 运算符：弹出栈顶优先级 ≥ 当前的所有运算符，再把当前入栈
  - 结束：弹出栈中剩余运算符

**(3) 递归**
- **递归调用栈**：每次递归调用都把现场（参数、局部变量、返回地址）压栈
- **递归 = 栈的应用**（隐式）
- **递归转非递归**：用显式栈模拟

**(4) 深度优先搜索（DFS）**
- 用栈实现非递归 DFS

**(5) 函数调用 / 撤销操作（undo）**
- 撤销栈：每次操作前入栈当前状态，撤销时弹栈恢复

**(6) 浏览器前进/后退**
- 两个栈：back 栈和 forward 栈

**(7) 中缀表达式求值的双栈法**
- 一个**操作数栈**，一个**操作符栈**
- 数字入操作数栈
- 运算符按优先级比较，决定计算还是入栈

#### 2.1.7 常见考点

- 后缀表达式求值（必考）
- 栈的入栈出栈合法序列判断（Catalan 数：`C_n = (2n)! / ((n+1)! n!)`）
- 栈与递归的等价性
- 双端栈 / 共享栈
- 中缀→后缀转换



## 第 3 章 队列

### 3.1 理论全掌握

#### 3.1.1 队列的定义与性质

- **队列（Queue）**：只允许在**一端**（队尾）插入、在**另一端**（队首）删除的线性表。
- **FIFO**（First In First Out）：先进先出。
- **队首（Front/Head）**：允许删除的一端。
- **队尾（Rear/Tail）**：允许插入的一端。
- **关键术语**：
  - **入队（Enqueue）**：队尾插入
  - **出队（Dequeue）**：队首删除
  - **读队首（Front/Peek）**：看队首元素
  - **队空（Empty）**、**队满（Full）**（仅顺序队列）
- **ADT 接口**：
  ```
  ADT Queue
  Data
      一维数组 + 队首、队尾指针
  Operation
      InitQueue(&Q)
      DestroyQueue(&Q)
      ClearQueue(&Q)
      QueueEmpty(Q)
      QueueLength(Q)
      GetHead(Q, &e)
      EnQueue(&Q, e)
      DeQueue(&Q, &e)
  ```

#### 3.1.2 顺序队列与假溢出

- **普通顺序队列**：数组 + `front` + `rear`
  - 初始：`front = rear = 0`（指向 0 号位置或 0 号前）
  - 入队：`data[rear++] = x`
  - 出队：`return data[front++]`
  - 判空：`front == rear`
  - 判满：`rear == MAXSIZE`（这是**假溢出**问题）
- **假溢出**：经过多次入队出队后，`rear` 到了数组末尾但前面有空位
- **解决**：循环队列（环形队列）

#### 3.1.3 循环队列（环形队列）—— 重点

- **核心思想**：用**模运算**把数组首尾相接。
- **结构**：数组 + `front` + `rear`，都从 0 增长，到 `MAXSIZE` 模回 0。
- **入队**：`rear = (rear + 1) % MAXSIZE; data[rear] = x;`
- **出队**：`front = (front + 1) % MAXSIZE;`
- **判空**：`front == rear`
- **判满**（两种方法）：
  - **方法 1（牺牲一个存储格）**：`(rear + 1) % MAXSIZE == front`
    - 容量实际可用 = MAXSIZE - 1
  - **方法 2（额外 count 变量）**：`count == MAXSIZE`
    - 容量实际可用 = MAXSIZE
- **队列长度**：
  - 方法 1：`(rear - front + MAXSIZE) % MAXSIZE`
  - 方法 2：直接 `count`
- **时间复杂度**：所有操作 O(1)
- **C++ 实现**（方法 2，更直观）：
```cpp
class CirQueue {
    static const int MAXN = 105;
    int data[MAXN];
    int frontIdx, rearIdx, cnt;
public:
    CirQueue(): frontIdx(0), rearIdx(-1), cnt(0) {}
    bool empty() const { return cnt == 0; }
    bool full()  const { return cnt == MAXN; }
    int  size()  const { return cnt; }
    void clear() { frontIdx=0; rearIdx=-1; cnt=0; }
    void enqueue(int x) { rearIdx = (rearIdx+1)%MAXN; data[rearIdx] = x; cnt++; }
    void dequeue()      { frontIdx = (frontIdx+1)%MAXN; cnt--; }
    int  front() const  { return data[frontIdx]; }
};
```

#### 3.1.4 链式队列

- **结构**：单链表 + 头指针（队首）+ 尾指针（队尾）。
- **优点**：
  - 长度动态，不存在假溢出
  - 永不满（除内存耗尽）
- **缺点**：
  - 每个节点多一个指针
  - 不可随机访问
- **关键操作**：
  - 入队：从尾部插入，`tail->next = new; tail = new;`
  - 出队：从头部删除，`head = head->next;`（空链表时 tail 也要置空）
- **时间复杂度**：所有操作 O(1)

#### 3.1.5 双端队列（Deque）

- **定义**：两端都能入队和出队的队列。
- **操作**：
  - `push_front` / `pop_front` / `front()`
  - `push_back` / `pop_back` / `back()`
- **应用**：
  - 既是栈也是队列的泛化
  - 滑动窗口（单调队列）
  - 撤销/重做（双栈效果）
- **C++ STL**：`std::deque<T>`（双端队列）

#### 3.1.6 优先队列（Priority Queue）

- **定义**：元素带优先级，出队时优先级最高者先出。
- **实现**：
  - **堆（最常用）**：插入 O(log n)，删除最值 O(log n)
  - **有序数组**：插入 O(n)，删除 O(1)
  - **无序数组**：插入 O(1)，删除 O(n)
- **应用**：
  - 任务调度（操作系统）
  - Dijkstra 算法的核心
  - Huffman 编码
  - 数据流的中位数 / Top K 问题
- **C++ STL**：`std::priority_queue<T>`（默认大根堆）

#### 3.1.7 队列的应用

- **BFS（广度优先搜索）**：用队列实现，层序遍历
- **操作系统进程就绪队列**
- **消息队列（Message Queue）**：进程间通信
- **缓冲区管理**：生产者-消费者模型
- **打印机任务队列**
- **滑动窗口 / 单调队列**：求最值
- **树的层序遍历**
- **图的 BFS 最短路（无权图）**

#### 3.1.8 顺序队列 vs 链式队列

| 维度 | 顺序（循环） | 链式 |
|---|---|---|
| 存储 | 数组 | 链表 |
| 容量 | 固定 | 动态 |
| 假溢出 | 用循环解决 | 不存在 |
| 判满 | 需要 | 不需要 |
| 额外空间 | 0 | 每节点 1 指针 |
| 访问速度 | 快 | 慢 |
| 实现难度 | 模运算易错 | 中等 |

#### 3.1.9 常见考点

- 循环队列的判空判满（必考，注意**牺牲存储格**约定）
- 队列与栈的相互实现（两个栈实现队列 / 两个队列实现栈）
- BFS 用队列
- 优先队列的实现
- 双端队列的滑动窗口



## 第 4 章 链式存储的栈与队列（含循环链表）

### 4.1 理论全掌握

#### 4.1.1 链栈（详见 §2.1.4）

- 单链表头插法实现
- 优点：动态长度、永不栈满
- 缺点：指针开销、不能随机访问
- 适用：栈深度不可预知

#### 4.1.2 链式队列（详见 §3.1.4）

- 单链表 + 头尾指针
- 优点：动态长度、无假溢出
- 关键：**尾指针维护**（不然入队要 O(n) 遍历）

#### 4.1.3 循环链表

- **单循环链表**：尾节点 `next` 指向头
- **双循环链表**：头节点 `prev` 指向尾，尾 `next` 指向头
- **优点**：
  - 从任意节点出发可遍历全表
  - 尾插 / 头插 O(1)
  - 适合"绕圈"操作（如约瑟夫）
- **判空**：
  - 单循环：`head == head->next`（头节点同时指向自己）
  - 双循环：`head->next == head && head->prev == head`
- **判尾**：
  - 单循环：`p->next == head`
  - 双循环：`p->next == head`
- **约瑟夫问题**：用循环单链表，O(n·m)

#### 4.1.4 链表常见操作

- **头插**：O(1)
- **尾插**（带尾指针）：O(1)
- **按位查找**：O(n)
- **按值查找**：O(n)
- **中间插入**（已知前驱）：O(1)
- **删除**（已知前驱）：O(1)
- **链表反转**：三指针迭代 O(n) / 递归 O(n)
- **快慢指针找中点**：slow 走一步、fast 走两步
- **判环**：fast 与 slow 相遇则有环

#### 4.1.5 链式结构的应用

**(1) 约瑟夫问题（Josephus）**
- n 个人围圈，从 1 报数到 m 出列，循环往复
- 用循环单链表实现
- 时间复杂度：O(n·m) 朴素 / O(n log n) 数学公式

**(2) 多项式运算**
- 节点 `{coef, exp, next}`，按指数排序
- 加减：双指针归并
- 乘法：每个 A 项 × 整个 B 累加

**(3) 大整数运算**
- 节点存一位（十进制）或一长块（万进制）
- 加减：从低位加起 + 进位
- 乘法：每位 × 整个数 + 进位

**(4) 字符串的链式存储**
- 一般用顺序存储（`char[]` 或 `string`）即可
- 链式适合频繁插入/删除字符

#### 4.1.6 顺序 vs 链式 综合对比

| 操作 | 顺序（数组） | 链式（链表） |
|---|---|---|
| 访问第 i 个 | O(1) | O(n) |
| 头部插入/删除 | O(n) | O(1) |
| 尾部插入/删除 | O(1) | O(1)（带尾指针） |
| 中间插入/删除 | O(n) | O(1)（已知前驱） |
| 查找（无序） | O(n) | O(n) |
| 空间 | 预先固定 | 动态 |
| 额外开销 | 0 | 指针 |
| 适用场景 | 长度已知、查询多 | 长度变化、增删多 |

#### 4.1.7 常见考点

- 循环链表的建立与判空判尾
- 约瑟夫问题
- 链表反转（一趟 / 递归）
- 快慢指针（找中点、判环、找倒数第 k 个）
- 哑节点（dummy head）的好处



## 第 5 章 递归

### 5.1 理论全掌握

#### 5.1.1 递归的定义

- **递归（Recursion）**：一个函数**直接或间接**调用自己。
- **递归的本质**：把大问题分解为**同类子问题**。
- **直接递归**：`f() { f(); }`
- **间接递归**：`f() { g(); } g() { f(); }`
- **数学基础**：数学归纳法。

#### 5.1.2 递归三要素

1. **递归终止条件（Base Case / 出口）**：最小问题的解，不再递归
2. **递归假设**：假设子问题已经解决
3. **递归方程（递推式）**：用子问题构造原问题

**例子：阶乘**
```cpp
int fact(int n) {
    if (n <= 1) return 1;          // 终止
    return n * fact(n - 1);        // 假设 fact(n-1) 已求
}
```

#### 5.1.3 递归 vs 迭代

| 维度 | 递归 | 迭代 |
|---|---|---|
| 思想 | 自顶向下分解 | 自底向上递推 |
| 代码 | 简洁 | 繁琐 |
| 性能 | 函数调用开销 | 无额外开销 |
| 栈空间 | O(深度) | O(1) |
| 适用 | 问题天然递归 | 简单循环 |
| 风险 | 栈溢出 | 几乎无 |

**能递归一定能迭代，迭代更高效**。

#### 5.1.4 递归调用栈（Call Stack）

- 每次递归调用都把**现场**压栈：
  - 参数值
  - 局部变量
  - 返回地址
- 函数返回时弹栈恢复
- **栈深度 = 递归深度**；深度过大会**栈溢出**（stack overflow）
- C++ 默认栈 8MB 左右；递归 10⁵ 层可能溢出

#### 5.1.5 递归的两种视角

- **自顶向下（Top-down）**：从原问题出发，分解到子问题
- **自底向上（Bottom-up）**：从最小子问题出发，构造原问题
- 递归算法是**自顶向下**写出来的
- 迭代 / 动态规划是**自底向上**实现

#### 5.1.6 尾递归

- **定义**：递归调用是函数的**最后一步操作**
- **特点**：编译器可以优化为循环（栈帧复用）
- **C++ 不保证尾递归优化**（GCC/O2 通常会优化）
- **例子**：
  ```cpp
  // 尾递归
  int fact(int n, int acc) {
      if (n <= 1) return acc;
      return fact(n - 1, n * acc);   // 最后一步是递归调用
  }
  ```

#### 5.1.7 递归时间复杂度分析

**展开法**（代入法、递归树）：

例：`T(n) = 2T(n/2) + O(n)`（归并排序）
- 第 0 层：1 个 O(n) = O(n)
- 第 1 层：2 个 O(n/2) = O(n)
- ...
- 第 log n 层：n 个 O(1) = O(n)
- 总计：`O(n log n)`

**主定理**：`T(n) = aT(n/b) + O(n^d)`
- 若 `d > log_b a`：`T(n) = O(n^d)`
- 若 `d = log_b a`：`T(n) = O(n^d log n)`
- 若 `d < log_b a`：`T(n) = O(n^log_b a)`

#### 5.1.8 经典递归问题

| 问题 | 终止 | 递推 | 时间 | 空间 |
|---|---|---|---|---|
| 阶乘 | n=1 → 1 | n * fact(n-1) | O(n) | O(n) |
| 斐波那契 | f(0)=0, f(1)=1 | f(n-1) + f(n-2) | O(2ⁿ) 朴素 | O(n) |
| 整数划分 | n=0 → 1, m=0 → 0（n>0）| 多种 | O(?) | O(?) |
| 全排列 | 长度到 n | 选未用元素 | O(n!) | O(n) |
| 组合 | 选够 k 个 | 选/不选 | O(C(n,k)) | O(n) |
| 汉诺塔 | n=1 移动一次 | 三步递归 | O(2ⁿ) | O(n) |
| N 皇后 | 放完 / 冲突 | 逐行放 | O(n!) | O(n²) |
| 迷宫 DFS | 到出口 / 死路 | 4 方向 | O(4^n) | O(n²) |

#### 5.1.9 递归设计步骤

1. **找重复**：子问题是什么？
2. **找变化**：哪些参数随递归变化？
3. **找边界**：什么条件下不再递归？

**例子：二叉树遍历**
```cpp
void preorder(Node* p) {
    if (!p) return;            // 边界：空树
    visit(p);                  // 访问根
    preorder(p->left);         // 重复 1：左子树
    preorder(p->right);        // 重复 2：右子树
}
```

#### 5.1.10 递归 vs 回溯

- **回溯（Backtracking）**：递归的"剪枝版"
- 在递归基础上：
  - 走不通时**撤销选择**（状态恢复）
  - 剪去明显不合法的分支
- 经典：八皇后、数独、组合/排列、子集和

```cpp
void backtrack(vector<int>& path, vector<bool>& used) {
    if (path.size() == n) {  // 终止
        // 记录答案
        return;
    }
    for (int i = 0; i < n; i++) {
        if (used[i]) continue;   // 剪枝
        used[i] = true;          // 做选择
        path.push_back(i);
        backtrack(path, used);   // 递归
        path.pop_back();         // 撤销选择
        used[i] = false;
    }
}
```

#### 5.1.11 递归的应用

- **数学**：阶乘、斐波那契、组合数、Catalan 数
- **数据结构**：树的遍历、链表反转、图的 DFS
- **分治**：归并、快排、二分查找
- **回溯**：排列、子集、八皇后
- **动态规划**（递归+记忆化）
- **编译原理**：语法分析（递归下降）

#### 5.1.12 常见考点

- 写递归求阶乘/斐波那契/全排列
- 写递归求二叉树高度/遍历
- 写递归判断回文 / 字符串反转
- 分析递归的时间空间复杂度
- 递归转非递归（用栈模拟）
- 八皇后、迷宫



## 第 6 章 列表 List

### 6.1 理论全掌握

#### 6.1.1 List 的定义

- **List（线性表）**：n 个数据元素的**有限序列**。
- 元素之间有**顺序**（线性关系）。
- 一般操作：增、删、改、查、遍历、求长度。
- **ADT 接口**：
  ```
  ADT List
  Data
      a[0..n-1] 线性序列
  Operation
      InitList(&L)
      DestroyList(&L)
      ClearList(&L)
      ListEmpty(L)
      ListLength(L)
      GetElem(L, i, &e)
      LocateElem(L, e, compare)        // 查找
      PriorElem(L, cur_e, &pre_e)
      NextElem(L, cur_e, &next_e)
      ListInsert(&L, i, e)
      ListDelete(&L, i, &e)
      ListTraverse(L, visit)
  ```

#### 6.1.2 顺序表（顺序存储）

- **结构**：数组 + 长度变量
- **特点**：
  - 逻辑相邻 = 物理相邻
  - 随机访问 O(1)
  - 插入/删除 O(n)
- **实现要点**：
  - 插入位置 i（0..n）：从后往前移
  - 删除位置 i（0..n-1）：从前往后移
  - **判满**：`size == capacity`
  - **判空**：`size == 0`
- **顺序表的操作时间复杂度**：
  | 操作 | 时间 |
  |---|---|
  | 初始化 | O(1) |
  | 销毁 | O(1) |
  | 清空 | O(1) |
  | 求长度 | O(1) |
  | 判空 | O(1) |
  | 读第 i 个 | O(1) |
  | 查找 | O(n) |
  | 插入 | O(n) |
  | 删除 | O(n) |
  | 遍历 | O(n) |

#### 6.1.3 链表（链式存储）

- **结构**：节点 `{data, *next}`，通过指针连接
- **种类**：
  - **单链表**：每个节点一个 next
  - **双链表**：每个节点 prev + next
  - **循环链表**：尾节点 next 回头
- **特点**：
  - 动态长度，无需预先分配
  - 插入/删除（已知前驱）O(1)
  - 查找 O(n)
  - 不能随机访问
- **关键概念**：
  - **头节点**：第一个节点（可存数据或不存）
  - **哑节点（dummy head）**：在头节点前的辅助节点，统一插入删除逻辑
  - **尾指针**：指向最后一个节点，便于 O(1) 尾插
  - **最后访问位置**：Lab 要求的优化，记录上次访问的节点和索引

#### 6.1.4 顺序表 vs 链表 详细对比

| 维度 | 顺序表 | 链表 |
|---|---|---|
| 存储空间 | 连续 | 离散 |
| 空间大小 | 固定 | 动态 |
| 随机访问 | O(1) | O(n) |
| 顺序访问 | O(n) | O(n) |
| 头部插入/删除 | O(n) | O(1) |
| 尾部插入/删除 | O(1) | O(1)（带尾指针）|
| 中间插入/删除 | O(n) | O(1)（已知前驱）|
| 空间开销 | 0 | 指针 |
| 缓存友好 | ✅ | ❌ |
| 适用场景 | 长度已知、查询多 | 长度变化、增删多 |

#### 6.1.5 单链表的改进

- **头节点 + 哑节点**：所有操作统一为"找到前驱"
- **循环链表**：尾插 O(1)，适合约瑟夫
- **双链表**：可双向遍历，删除给定节点 O(1)
- **静态链表**：用数组模拟链表（无指针的语言）
- **跳跃表（Skip List）**：多级索引的链表，O(log n) 查找
- **最后访问位置**：记录访问过的节点，下次访问就近

#### 6.1.6 链表反转

- **迭代版（三指针）**：
  ```cpp
  Node* reverse(Node* head) {
      Node *pre = nullptr, *cur = head;
      while (cur) {
          Node* nxt = cur->next;
          cur->next = pre;
          pre = cur;
          cur = nxt;
      }
      return pre;
  }
  ```
- **递归版**：
  ```cpp
  Node* reverse(Node* head) {
      if (!head || !head->next) return head;
      Node* newHead = reverse(head->next);
      head->next->next = head;
      head->next = nullptr;
      return newHead;
  }
  ```
- **时间 O(n)，空间 O(1) 迭代 / O(n) 递归**

#### 6.1.7 快慢指针

- **找中点**：slow 走一步，fast 走两步，fast 到尾时 slow 在中点
- **判环**：fast 与 slow 相遇则有环
- **找环入口**：相遇后一个从起点走，一个从相遇点走，再次相遇处即为入口
- **找倒数第 k 个**：fast 先走 k 步，然后 slow 跟上，fast 到尾时 slow 即是

#### 6.1.8 大数运算

- **存储**：`vector<int>` 存十进制位，**低位在前**（便于进位）
- **加法**：从低位加，处理进位
- **减法**：被减数 ≥ 减数（否则取负），借位处理
- **乘法**：每位 × 整个数 + 进位，O(n²)
- **除法**：试商法（Lab 选做）
- **去除前导零**：`while (d.size() > 1 && d.back() == 0) pop_back();`
- **零的判断**：`d.size() == 1 && d[0] == 0`
- **优化**：万进制或亿进制（每位存 4-8 位十进制）减少节点数

#### 6.1.9 字符串（String）

- **C 风格**：`char[]` + 长度 + `\0` 结尾
- **C++ STL**：`std::string`（封装好，自动管理）

- **自实现 String**：
  - 内部用 `List<char>` 或 `char* + length`
  - 重载运算符
  - 实现各种字符串函数

## 第 7 章 查找

### 7.1 理论全掌握

#### 7.1.1 查找的基本概念

- **查找表（Search Table）**：由同一类型的数据元素（或记录）组成的集合。
- **关键字（Key）**：数据元素中某个数据项的值，用以标识一个数据元素。
  - **主关键字**：唯一标识
  - **次关键字**：不唯一
- **查找（Searching）**：根据关键字，确定查找表中是否存在该记录，以及其位置。
- **查找表分类**：
  - **静态查找表**：只查不改（顺序/二分/索引）
  - **动态查找表**：可插入删除（BST/AVL/哈希）

#### 7.1.2 查找的评价指标

- **平均查找长度 ASL（Average Search Length）**：查找过程中关键字比较次数的期望。
  - 成功 ASL：`Σ(查找次数 × 概率) / n`
  - 失败 ASL：基于失败情况的平均
- **时间复杂度**：T(n) 随 n 增长
- **空间复杂度**：S(n)

#### 7.1.3 顺序查找（线性查找）

- **思想**：从表的一端开始，逐个比较
- **时间复杂度**：O(n)
- **空间复杂度**：O(1)
- **优点**：对数据无要求（有序无序都行）
- **缺点**：效率低
- **改进**：
  - 设哨兵：从尾部开始放哨兵，省去边界检查
  - 概率优化：高频数据放前面
  - 分块查找：每块内顺序、块间索引

#### 7.1.4 折半查找（二分查找）—— 重点

- **前提**：顺序存储 + **有序**
- **思想**：每次与中间元素比较，缩小一半范围
- **核心循环**：
  ```cpp
  int lo = 0, hi = n - 1;
  while (lo <= hi) {
      int mid = lo + (hi - lo) / 2;   // 防溢出
      if (a[mid] == target) return mid;
      else if (a[mid] < target) lo = mid + 1;
      else hi = mid - 1;
  }
  return -1;
  ```
- **时间复杂度**：O(log n)
- **空间复杂度**：O(1) 迭代 / O(log n) 递归
- **判定树**：高度 ⌈log₂(n+1)⌉，平衡二叉树

#### 7.1.5 二分查找的 6 大变体（必背）

**1. 标准查找**：找任意一个等于 target 的位置
```cpp
while (lo <= hi) {
    int mid = (lo+hi)/2;
    if (a[mid] == target) return mid;
    else if (a[mid] < target) lo = mid+1;
    else hi = mid-1;
}
```

**2. 找第一个 ≥ target 的位置（lower_bound）**
```cpp
int lo=0, hi=n;
while (lo < hi) {
    int mid = (lo+hi)/2;
    if (a[mid] < target) lo = mid+1;
    else hi = mid;
}
return lo;   // 可能 == n
```

**3. 找第一个 > target 的位置（upper_bound）**
```cpp
int lo=0, hi=n;
while (lo < hi) {
    int mid = (lo+hi)/2;
    if (a[mid] <= target) lo = mid+1;
    else hi = mid;
}
return lo;
```

**4. 找最后一个 ≤ target 的位置**
```cpp
// = upper_bound(target) - 1
```

**5. 找最后一个 < target 的位置**
```cpp
// = lower_bound(target) - 1
```

**6. 找第一个 = target（不存在则 -1）**
```cpp
int p = lower_bound(a, target);
if (p < n && a[p] == target) return p;
return -1;
```

#### 7.1.6 插值查找

- **思想**：按 key 的分布估计位置（类似字典查词）
- **公式**：`mid = lo + (target - a[lo]) * (hi - lo) / (a[hi] - a[lo])`
- **适用**：数据分布均匀
- **时间复杂度**：O(log log n) 均匀分布 / O(n) 最坏

#### 7.1.7 斐波那契查找

- **思想**：用斐波那契数列分割
- **优点**：避免除法
- **时间复杂度**：O(log n)

#### 7.1.8 分块查找（索引顺序查找）

- **思想**：把表分成若干块，块内无序、块间有序；建索引表
- **步骤**：
  1. 在索引表（有序）里二分找块
  2. 在块内顺序找
- **时间复杂度**：O(√n) 平均

#### 7.1.9 树形查找（BST/AVL/红黑树）—— 见第 10 章

#### 7.1.10 哈希查找 —— 见第 9 章

#### 7.1.11 各种查找的复杂度对比

| 查找方式 | 平均时间 | 最坏时间 | 适用 |
|---|---|---|---|
| 顺序 | O(n) | O(n) | 无序 |
| 二分 | O(log n) | O(log n) | 有序 |
| 插值 | O(log log n) | O(n) | 均匀分布 |
| 斐波那契 | O(log n) | O(log n) | 有序 |
| 分块 | O(√n) | O(n) | 大表 |

## 第 8 章 排序

### 8.1 理论全掌握

#### 8.1.1 排序的基本概念

- **排序（Sorting）**：将一组数据按关键字递增/递减的顺序重新排列。
- **稳定性**：相等关键字排序前后相对位置不变。
- **内排序 vs 外排序**：
  - **内排序**：数据全部在内存（O(n²) ~ O(n log n)）
  - **外排序**：数据在外存（归并思想，复杂度以磁盘 I/O 衡量）
- **原地排序**：辅助空间 O(1)
- **比较排序下界**：Ω(n log n)（决策树证明）

#### 8.1.2 排序总览（必背）

| 排序 | 平均 | 最坏 | 空间 | 稳定 | 类别 |
|---|---|---|---|---|---|
| **冒泡** | O(n²) | O(n²) | O(1) | ✅ | 交换 |
| **插入** | O(n²) | O(n²) | O(1) | ✅ | 插入 |
| **希尔** | O(n^1.3) | O(n²) | O(1) | ❌ | 插入改进 |
| **选择** | O(n²) | O(n²) | O(1) | ❌ | 选择 |
| **堆排** | O(n log n) | O(n log n) | O(1) | ❌ | 选择 |
| **归并** | O(n log n) | O(n log n) | O(n) | ✅ | 分治 |
| **快排** | O(n log n) | O(n²) | O(log n) | ❌ | 分治 |
| **计数** | O(n+k) | O(n+k) | O(k) | ✅ | 分配 |
| **基数** | O(d·n) | O(d·n) | O(n+k) | ✅ | 分配 |
| **桶** | O(n+k) | O(n²) | O(n+k) | ✅ | 分配 |

#### 8.1.3 冒泡排序（Bubble Sort）

- **思想**：相邻元素两两比较，逆序则交换；每轮将最大元素"冒"到末尾
- **步骤**：
  - 外层 i = 0..n-1
  - 内层 j = 0..n-i-1
  - `if (a[j] > a[j+1]) swap(a[j], a[j+1])`
- **优化**：加 `swapped` 标志，本轮无交换则已排好
- **时间**：O(n²) 平均/最坏 / O(n) 已有序
- **空间**：O(1)
- **稳定**：✅

#### 8.1.4 插入排序（Insertion Sort）—— 重点

- **思想**：把数组分为已排序区（左侧）和未排序区；依次将未排序元素插入已排序区
- **步骤**：
  - `for i = 1..n-1`：
    - `key = a[i]`
    - `j = i - 1`
    - `while j >= 0 && a[j] > key: a[j+1] = a[j]; j--`
    - `a[j+1] = key`
- **时间**：O(n²) 平均/最坏 / O(n) 已有序
- **空间**：O(1)
- **稳定**：✅
- **应用**：数据基本有序时效率高；小规模数据；其他排序（如希尔、归并）的子过程

#### 8.1.5 希尔排序（Shell Sort）

- **思想**：插入排序的改进；按一定**增量**分组，对每组做插入排序；增量逐步缩小直到 1
- **增量序列**（影响性能）：
  - Shell 原始：n/2, n/4, ..., 1
  - Hibbard：1, 3, 7, 15, ...
  - Sedgewick：复杂序列
- **时间**：
  - Shell 增量：O(n²) 最坏
  - Hibbard 增量：O(n^1.5) 最坏
  - 平均：O(n^1.3)
- **空间**：O(1)
- **稳定**：❌
- **意义**：第一个突破 O(n²) 的排序

#### 8.1.6 选择排序（Selection Sort）

- **思想**：每轮从未排序区找最小元素，与未排序区第一个元素交换
- **步骤**：
  - `for i = 0..n-1`：
    - `mi = i`
    - `for j = i+1..n-1: if a[j] < a[mi] mi = j`
    - `swap(a[i], a[mi])`
- **时间**：O(n²) 永远（无论数据如何）
- **空间**：O(1)
- **稳定**：❌（交换破坏顺序）
- **优点**：交换次数少（最多 n-1 次）

#### 8.1.7 堆排序（Heap Sort）—— 重点

- **堆的定义**：
  - **大根堆**：父 ≥ 子，根 = 最大
  - **小根堆**：父 ≤ 子，根 = 最小
- **数组存储完全二叉树**：
  - 0-indexed：父 `(i-1)/2`，左子 `2i+1`，右子 `2i+2`
  - 1-indexed：父 `i/2`，左子 `2i`，右子 `2i+1`
- **`siftDown(i, n)`**：把 i 向下调整（与较大子节点交换）
- **`siftUp(i)`**：把 i 向上调整（与父节点比较）
- **`buildHeap`**：从最后一个非叶节点 `(n-1)/2` 倒序 siftDown，O(n)
- **`heapSort`**：
  1. buildHeap（大根堆）
  2. 反复 `swap(0/1, n-1)`，n--，siftDown
- **时间**：O(n log n) 永远
- **空间**：O(1)
- **稳定**：❌
- **应用**：Top K、堆、优先队列

#### 8.1.8 归并排序（Merge Sort）—— 重点

- **思想**：分治——把数组对半拆分，递归排序，合并两个有序子数组
- **步骤**：
  1. `mergeSort(lo, hi)`：`mid = (lo+hi)/2`
  2. `mergeSort(lo, mid); mergeSort(mid+1, hi)`
  3. `merge(lo, mid, hi)`：双指针合并
- **合并**：
  - 临时数组存合并结果
  - 双指针 i, j 指向两个子数组
  - 每次取较小者放入临时数组
  - 最后把剩余部分复制回
- **时间**：O(n log n) 永远
- **空间**：O(n)（合并需要临时数组）
- **稳定**：✅
- **应用**：外排序（多路归并）

#### 8.1.9 快速排序（Quick Sort）—— 重点

- **思想**：分治——选轴点（pivot），划分（partition）为左右两半，递归排序
- **轴点选择**：
  - 第一个 / 最后一个 / 中间
  - 随机选
  - 三数取中（首/中/尾取中位数）
- **划分算法**：
  - **Lomuto**（单指针）
  - **Hoare**（双指针）
  - **课本"末元素为轴"**（Lab 用）
- **课本版本步骤**（Lab15 用）：
  - 设轴 `pivot = a[hi]`
  - `i = lo, j = hi`
  - 从左找到第一个 > 轴；从右找到第一个 < 轴
  - 若 i < j，交换；继续
  - 结束交换 `a[i]` 与 `a[hi]`
  - 返回 i 作为新轴点位置
- **递归**：`quickSort(lo, p-1); quickSort(p+1, hi)`
- **时间**：
  - 平均 O(n log n)
  - 最坏 O(n²)（已排序 / 逆序 / 全相同）
- **空间**：O(log n) 递归栈
- **稳定**：❌
- **优化**：
  - 随机轴点
  - 小规模用插入排序
  - 三数取中
  - 尾递归优化

#### 8.1.10 计数排序（Counting Sort）

- **思想**：统计每个值出现的次数，再按顺序输出
- **步骤**：
  1. 找 max
  2. 建 count[0..max]
  3. 扫描 a 统计 count[a[i]]++
  4. 累加 count 得到位置
  5. 反向填充（保持稳定）
- **时间**：O(n + k)
- **空间**：O(k)
- **稳定**：✅
- **适用**：数据范围小（k << n）

#### 8.1.11 基数排序（Radix Sort）

- **思想**：按位（个位、十位、百位…）依次排序（用稳定的子排序）
- **LSD（Least Significant Digit）**：从低位到高位
- **MSD**：从高位到低位
- **子排序**：计数排序
- **时间**：O(d·n)
- **空间**：O(n + k)
- **稳定**：✅
- **适用**：整数、字符串（按字符位）

#### 8.1.12 桶排序（Bucket Sort）

- **思想**：分桶（按范围）→ 各桶内排序 → 合并
- **时间**：O(n + k) 平均
- **空间**：O(n + k)
- **适用**：均匀分布的浮点数

#### 8.1.13 优先队列（Priority Queue）

- **定义**：元素带优先级，出队时优先级最高者先出
- **实现**：
  - **堆（最常用）**
  - 有序数组
  - 无序数组
  - 二项堆 / 斐波那契堆
- **基本操作**：
  - `insert(x)`：插入元素
  - `findMax()` / `findMin()`：取最值
  - `deleteMax()` / `deleteMin()`：删除最值
- **时间（堆实现）**：
  - 插入 O(log n)
  - 取最值 O(1) 或 O(log n)
  - 删除最值 O(log n)
  - 建堆 O(n)

#### 8.1.14 各种排序的选用指南

| 场景 | 推荐 |
|---|---|
| 一般情况 | 快排 |
| 要求稳定 | 归并 |
| 空间有限 | 堆排 |
| 数据基本有序 | 插入 |
| 大数据 + 内存不够 | 归并（外排序）|
| 范围小 | 计数 |
| 整数且位数少 | 基数 |
| Top K 问题 | 堆 |

- 各种排序的算法步骤（必会手写）
- 时间/空间复杂度
- 稳定性判断
- 适用场景对比

## 第 9 章 哈希表

### 9.1 理论全掌握

#### 9.1.1 哈希表的基本概念

- **哈希表（Hash Table）**：根据关键码值（Key）直接访问的数据结构。通过**哈希函数**将 key 映射到数组下标。
- **优点**：期望 O(1) 的查找、插入、删除
- **缺点**：冲突处理复杂、不支持顺序访问、空间利用率不高
- **关键概念**：
  - **哈希函数 H(k)**：key → [0, m) 的映射
  - **冲突（Collision）**：不同 key → 同一地址
  - **同义词（Synonym）**：哈希值相同的不同 key
  - **装填因子 α = n/m**：数据数 / 表大小（α 越小越好）

#### 9.1.2 哈希函数设计要求

1. **简单**：计算快
2. **均匀**：地址分布均匀，少冲突
3. **一致**：相同 key → 相同地址
4. **满射**：尽量用满所有地址

#### 9.1.3 常见哈希函数

| 方法 | 公式 | 适用 | 评价 |
|---|---|---|---|
| **直接定址法** | `H(k) = a·k + b` | key 分布连续 | 简单但不通用 |
| **除留余数法** | `H(k) = k mod p`（p 最好是质数） | 整数 key | **最常用** |
| **数字分析法** | 取 key 的若干位 | 已知 key 分布 | 特殊场景 |
| **平方取中法** | 取 k² 的中间几位 | 关键字位数不多 | 均匀 |
| **折叠法** | 分段求和 | 关键字位数多 | 简单 |
| **旋转法** | 末位移到首位 | 字符串 | 特定 |
| **BKDRHash** | `h = h*seed + ch` | **字符串 key** | **Lab 必背** |
| **DJB2** | `h = h*33 + ch` | 字符串 | 简单 |
| **SDBM** | `h = ch + h<<6 + h<<16 - h` | 字符串 | 较均匀 |
| **FNV-1** | `h = (h ^ ch) * FNV_prime` | 字符串 | 较均匀 |

**BKDRHash 模板（C++ 必背）**：
```cpp
unsigned int BKDRHash(const char* str, unsigned int size) {
    unsigned int seed = 31;        // 31, 131, 1313, 13131...
    unsigned int h = 0;
    while (*str) h = (h * seed + (unsigned char)(*str++)) % size;
    return h;
}
// 或 std::string 版本
unsigned int BKDR(const string& s, unsigned int size) {
    unsigned int h = 0, seed = 31;
    for (char c : s) h = (h * seed + (unsigned char)c) % size;
    return h;
}
```

#### 9.1.4 冲突解决方法

**(1) 链地址法（Separate Chaining）**
- 每个桶是一个**链表**
- 插入：查到链表（头插法）
- 查找/删除：遍历链表
- 优点：简单、不会"装满"、删除简单
- 缺点：指针开销、链表长 → 性能退化
- 优化：链表 → 红黑树 / 跳表（Java HashMap 8 后）

**(2) 开放定址法（Open Addressing）**
- 冲突时按规则探测下一个空位
- **三种探测**：
  - **线性探测**：`H_i = (H + i) mod m`
    - 优点：简单
    - 缺点：**堆积（clustering）** 严重
  - **平方探测（Quadratic Probing）**：`H_i = (H + i²) mod m` 或 `H_i = (H ± i²) mod m`
    - 优点：缓解堆积
    - 缺点：不能探遍所有位置
  - **双散列（Double Hashing）**：`H_i = (H + i * H₂) mod m`
    - 优点：均匀
    - 缺点：需要第二个哈希函数

**(3) 再哈希法（Rehashing）**
- 冲突时用另一个哈希函数
- 较少用

**(4) 建立公共溢出区**
- 基本表 + 溢出表
- 较少用

#### 9.1.5 哈希表删除问题

- **链地址法**：直接删节点，O(1)
- **开放定址法**：**不能直接置空**（否则查找链断），需用**墓碑（tombstone）标记**：
  - 状态：`EMPTY` / `OCCUPIED` / `DELETED`
  - 删除时设为 `DELETED`
  - 查找时遇到 `DELETED` 继续
  - 插入时遇到 `DELETED` 可复用
- 缺点：墓碑积累影响性能 → 定期 rebuild

#### 9.1.6 装填因子与性能

| 解决方式 | 成功 ASL | 失败 ASL |
|---|---|---|
| 链地址法 | `1 + α/2` | `α` |
| 线性探测 | `(1 + 1/(1-α))/2` | `(1 + 1/(1-α)²)/2` |
| 平方探测 | `-ln(1-α)/α` | `1/(1-α)` |

- α 越接近 1，性能越差
- α > 0.7 时通常需要**扩容（rehash）**
- 一般保持 α < 0.5 ~ 0.75

#### 9.1.7 动态哈希表

- **扩容策略**：当 α 超过阈值（如 0.75）时：
  1. 开新表（容量 2 倍）
  2. 重新计算所有 key 的哈希
  3. 重新插入
- **摊还成本**：每次插入 O(1) 摊还
- **C++ STL `unordered_map`**：自动扩容，负载因子 1.0

#### 9.1.8 一致性哈希（进阶）

- 用于**分布式系统**的缓存/数据库分片
- 把哈希空间组织成环（0 ~ 2³²）
- 节点和数据都映射到环上
- 数据归属：顺时针第一个节点
- **虚拟节点**：解决数据倾斜

#### 9.1.9 哈希表的应用

- 编译器符号表
- 数据库索引（哈希索引）
- 缓存（Memcached / Redis）
- 集合 / 字典
- 字符串去重 / 计数
- LRU 缓存（哈希 + 双向链表）

#### 9.1.11 哈希函数冲突率深度分析

**直接定址法**
- 公式：`H(k) = a·k + b`
- 冲突率：理论上 0（key 唯一映射），但 key 稀疏时**空间浪费严重**
- 适用：key 分布密集、连续（如学号 1..10000）

**除留余数法**
- 公式：`H(k) = k mod p`，p 是质数
- 为什么 p 必须是质数？——若 p = 合数 m = a·b，则所有 `k ≡ 0 (mod a)` 的 key 会冲突到 0, a, 2a...
- 例：m=12（非质数），12 = 3×4，所有 key 模 3 同余会冲突到 {0, 3, 6, 9}
- 最佳实践：取不超过表大小的最大质数（如 1009 比 1000 好）

**数字分析法**
- 适用：key 已知且分布有规律（如手机号前 7 位、身份证后 4 位）
- 选取分布均匀的几位作为哈希地址

**平方取中法**
- 取 `k²` 的中间几位（避开边界，更均匀）
- 例：k=1234, k²=1522756，取中间 3 位 227 作为地址

**折叠法**
- 把 key 分成若干段，每段相加（适合位数多的 key）
- 例：k=123456789，分成 123|456|789 → 123+456+789 = 1368

**BKDRHash 详解（Lab 必背）**
- 公式：`h = h * seed + ch`
- seed 选择：31, 131, 1313, 13131...
  - 31 是 Java `hashCode()` 的 seed（小质数 + 奇数）
  - 131 是《算法竞赛入门经典》推荐（实验证冲突率最低）
- 模运算位置：**每步都取模** vs **最后取模**——前者避免整数溢出
- 字符串 vs 整数：
  ```cpp
  // C 字符串版本
  unsigned int BKDRHash(const char* str, unsigned int size) {
      unsigned int seed = 31;
      unsigned int h = 0;
      while (*str) h = (h * seed + (unsigned char)(*str++)) % size;
      return h;
  }
  // std::string 版本（Lab 常用）
  unsigned int BKDR(const string& s, unsigned int size) {
      unsigned int h = 0, seed = 31;
      for (char c : s) h = (h * seed + (unsigned char)c) % size;
      return h;
  }
  ```
- **踩坑**：负数 key 必先转 `(unsigned)k % size`，否则负数模出负数

**DJB2 / SDBM / FNV 简介**
- DJB2：`h = h*33 + ch`（简单）
- SDBM：`h = ch + (h<<6) + (h<<16) - h`（数据库常用）
- FNV-1：`h = (h ^ ch) * FNV_prime`（密码学场景）

#### 9.1.12 冲突解决方法深度对比

**(1) 链地址法（Separate Chaining）—— 最常用**

**结构**：
```
index 0: [A] -> [D] -> null
index 1: [B] -> null
index 2: null
index 3: [C] -> [E] -> [F] -> null
```

**查找过程**：
1. `h = H(key)` 计算桶号
2. 在 `table[h]` 指向的链表中顺序查找

**插入过程**（头插法）：
1. 计算桶号 h
2. 创建新节点
3. `new->next = table[h]; table[h] = new;`

**删除过程**：
1. 计算桶号 h
2. 遍历链表，找到前驱 prev
3. `prev->next = cur->next; delete cur;`

**性能**：
- 平均链长：α = n/m
- 成功 ASL：1 + α/2
- 失败 ASL：α

**Java 8+ 优化**：链表长 > 8 时转**红黑树**，长 < 6 时退回链表（避免频繁转换）

**C++ STL 实现**：`unordered_map<K, V>` 就是链地址法，每个桶是一个单向链表

**(2) 开放定址法（Open Addressing）**

**统一公式**：`H_i(k) = (H(k) + d_i) mod m`，d_i 是探测序列

**(2.1) 线性探测**
- `d_i = 0, 1, 2, ..., m-1`
- 例：m=10, H=3, 已被占用 → 试 4, 5, 6, ... 直到找到空位
- **堆积（Primary Clustering）** 现象：
  - 已占位置形成"块"，新 key 落到块尾概率高
  - 块越长，新 key 加长块概率越大（**正反馈**）
  - 堆积严重时性能急剧下降
- ASL 公式：
  - 成功：(1 + 1/(1-α)) / 2
  - 失败：(1 + 1/(1-α)²) / 2

**(2.2) 平方探测（二次探测）**
- `d_i = 1², -1², 2², -2², 3², -3², ...` 或 `d_i = 1², 2², 3², ...`
- 缓解堆积：相邻探测位置间隔平方增长，不会形成大块
- **不能探遍所有位置** —— 证明：
  - 若 m = 4k+3 质数（如 7, 11, 19, 23...），可探遍
  - 否则最多探 (m+1)/2 个位置
- 例：m=10, H=3 → 探 4, 6, 0, 6, 4, 4, ... （撞已探过的位置）
- ASL 公式：
  - 成功：`-ln(1-α) / α`
  - 失败：`1 / (1-α)`

**(2.3) 双散列**
- `H_i = (H_1(k) + i * H_2(k)) mod m`
- H_2 必须与 m 互质，否则探不全
- 常用：`H_2(k) = q - (k mod q)`，q 是小于 m 的质数
- 性能接近"随机探测"（理想情况）
- 实现复杂，需两个哈希函数

**三种探测方法对比**：

| 方法 | 探遍性 | 抗堆积 | 实现复杂度 | 适用 |
|---|---|---|---|---|
| 线性 | ✅ | ❌ 严重 | ⭐ | 简单场景 |
| 平方 | ❌（m=4k+3 才行） | ✅ 缓解 | ⭐⭐ | 中等 |
| 双散列 | ✅ | ✅✅ 最好 | ⭐⭐⭐ | 性能要求高 |

#### 9.1.13 哈希表删除的陷阱与墓碑机制

**为什么开放定址法不能直接置空？**

考虑探测序列：A → 5, B → 5, 6, C → 6, 7
- 删 A：直接把 5 置空
- 查 B：探测到 5（空）→ 停止 → **认为 B 不存在** ❌

**墓碑（tombstone）机制**：

```cpp
enum State { EMPTY, OCCUPIED, DELETED };

struct Entry {
    K key;
    V val;
    State state;
};

bool find(int h, K key) {
    int i = h;
    while (table[i].state != EMPTY) {
        if (table[i].state == OCCUPIED && table[i].key == key)
            return true;
        i = (i + 1) % m;
    }
    return false;
}

void remove(int h, K key) {
    int i = h;
    while (table[i].state != EMPTY) {
        if (table[i].state == OCCUPIED && table[i].key == key) {
            table[i].state = DELETED;  // 不删数据，只标记
            return;
        }
        i = (i + 1) % m;
    }
}
```

**墓碑积累问题**：
- 多次插入+删除后，DELETED 标记越来越多
- 查找时遇到 DELETED 仍要继续探测 → 性能下降
- 解决方案：定期 rebuild（清空所有墓碑，重新插入存活 key）

**链地址法删除的便利**：
- 直接 `prev->next = cur->next; delete cur;`
- 不影响其他 key 的查找
- 这是链地址法相对开放定址的**重要优势**之一

#### 9.1.14 装填因子与扩容策略

**装填因子 α = n/m**

**为什么 α 不能太大？**

以线性探测为例，失败 ASL = (1 + 1/(1-α)²) / 2：
- α = 0.5 → ASL = 5
- α = 0.7 → ASL ≈ 12
- α = 0.9 → ASL ≈ 50
- α = 0.95 → ASL ≈ 200

**经验值**：
- α ≤ 0.5：性能优
- α = 0.5 ~ 0.75：可用
- α > 0.7：**必须扩容**
- α > 0.9：性能急剧恶化

**扩容（rehash）步骤**：
1. 开新表（容量通常是 2 倍 + 找下一个质数）
2. 重新计算所有 key 的哈希值
3. 重新插入（用新表大小取模）

**为什么 2 倍？**——摊还分析表明平均每次插入 O(1)

**Java HashMap 默认负载因子 0.75**——平衡空间与时间

**C++ `unordered_map` 默认 max_load_factor = 1.0**

#### 9.1.15 一致性哈希详解（进阶）

**场景**：分布式缓存，需要把数据分布到 N 个节点上

**传统哈希的问题**：
- `node = hash(key) % N`
- 节点增减时，几乎所有 key 的映射都要变 → **缓存雪崩**

**一致性哈希的解决**：

**步骤**：
1. 把哈希空间组织成环 [0, 2³²)
2. 每个节点哈希到环上：`h(node) = hash(IP)`
3. 每个数据 key 也哈希到环上
4. 数据沿环顺时针找到的第一个节点就是它的归属

**示例**（3 个节点）：
```
节点 A → hash(A) = 100
节点 B → hash(B) = 5000
节点 C → hash(C) = 9000

key "x" → hash("x") = 3000 → 顺时针找 → B
key "y" → hash("y") = 7000 → 顺时针找 → C
key "z" → hash("z") = 50 → 顺时针找 → A
```

**节点增减的影响**：
- 加节点 D（hash=2000）：只影响 [100, 2000) 区间的 key（重新映射到 D）
- 删节点 A：只影响 [A 位置, B 位置) 区间的 key（重新映射到 B）
- **只影响一部分数据**，不是全部 → 避免雪崩

**虚拟节点（Virtual Node）解决数据倾斜**：
- 真实问题：节点少时，每个节点承担的区间大小不均 → 数据倾斜
- 解决：每个真实节点虚拟出 V 个节点（如 V=200）
- 每个 key 顺时针找最近的**虚拟节点**，再映射到对应的真实节点

**应用**：Redis Cluster、Memcached、CDN 内容分发

#### 9.1.16 哈希攻击与防御

**哈希攻击（Hash DoS）**：
- 攻击者构造大量 key，使它们全部哈希到同一桶
- 哈希表退化为链表 → O(1) 变 O(n)
- 用大量恶意请求击垮服务器

**典型攻击**：发送 N 个精心构造的 key，每个 key 哈希值相同 → 服务器处理每个请求都遍历 N 长度链表

**防御方法**：
1. **加盐随机化**：每个进程启动时用随机 seed 初始化哈希函数
   - Python 3.3+：`hash()` 默认带进程随机 seed
   - Java `HashMap`：字符串哈希每次加随机扰动
2. **双哈希**：第二个哈希函数用不同算法
3. **限制输入长度**：key 过长直接拒绝
4. **改用红黑树**：Java 8+ HashMap 链表长 > 8 转红黑树（O(logn) 兜底）

#### 9.1.17 完美哈希（Perfect Hashing）

**定义**：哈希函数 H 使 n 个 key 全部映射到不同地址 → 无冲突

**适用场景**：key 集合**静态且已知**（如编译器关键字、字典）

**构造方法**：
1. **两级哈希**：第一级把 key 分到 m 个桶，第二级每个桶独立构造完美哈希
2. **概率构造**：随机选 H 直到无冲突，期望 O(n) 即可找到

**应用**：
- gperf 工具（C++ 关键字哈希）
- 编译器关键字查找表

#### 9.1.18 布隆过滤器（Bloom Filter）

**场景**：判断元素是否在集合中，**允许误判但不允许漏判**

**原理**：
- 一个 m 位的位数组 + k 个哈希函数
- 插入：对元素算 k 个哈希，每位设为 1
- 查找：算 k 个哈希，**全为 1 则认为存在**（可能有误判），任一为 0 则肯定不在

**特点**：
- ✅ 空间效率高（位数组）
- ✅ 插入/查找 O(k)
- ❌ 不支持删除（除非用计数布隆过滤器）
- ❌ 有误判率（可调）

**应用**：
- 网页爬虫：URL 去重
- 垃圾邮件过滤
- 数据库：减少不必要的磁盘查询

#### 9.1.19 Lab 实战技巧（哈希表）

**BKDRHash 调参**：
```cpp
// Lab 9.3 字符串哈希模板
struct StrHash {
    size_t operator()(const string& s) const {
        size_t h = 0;
        for (char c : s) h = h * 131 + (unsigned char)c;
        return h;
    }
};
unordered_map<string, int, StrHash> mp;
```

**链地址法完整模板**：
```cpp
class HashMap {
    static const int M = 1009;  // 质数
    vector<list<pair<string,int>>> buckets;  // 用 list 而非 vector
public:
    HashMap() : buckets(M) {}
    void put(string k, int v) {
        int h = BKDR(k, M);
        for (auto& p : buckets[h])
            if (p.first == k) { p.second = v; return; }
        buckets[h].push_back({k, v});
    }
    int get(string k) {
        int h = BKDR(k, M);
        for (auto& p : buckets[h])
            if (p.first == k) return p.second;
        return -1;
    }
};
```

**链地址法删除**（Lab q203 重点）：
```cpp
void remove(string k) {
    int h = BKDR(k, M);
    auto& lst = buckets[h];
    for (auto it = lst.begin(); it != lst.end(); ++it)
        if (it->first == k) { lst.erase(it); return; }
}
```

**避免 hash collision 攻击**：Java `HashMap` 用 `(h ^ (h >>> 16))` 扰动

**调试技巧**：
- `cout << "h=" << h << " size=" << buckets[h].size()` 看哪个桶太长
- 长链 → 调大 M 或换更好的哈希函数
## 第 10 章 二叉树

### 10.1 理论全掌握

#### 10.1.1 树的术语

- **节点（Node）**：树的基本单位
- **边（Edge）**：节点之间的连线
- **根（Root）**：唯一，没有父节点
- **父节点 / 子节点 / 兄弟节点**
- **度（Degree）**：节点的子树个数
- **叶子（Leaf）**：度为 0 的节点
- **内部节点**：度 ≠ 0
- **层次（Level）**：根为第 1 层
- **深度（Depth）**：根到节点的路径长度
- **高度（Height）**：节点到叶子的最长路径
- **森林（Forest）**：m 棵互不相交树的集合

#### 10.1.2 二叉树的定义

- **二叉树**：每个节点 ≤ 2 个子树（左、右），子树有顺序。
- **5 种形态**：
  - 空树
  - 只有根
  - 只有左子树
  - 只有右子树
  - 完整二叉树（左 + 右）

#### 10.1.3 特殊二叉树

- **满二叉树**：所有层都满。节点数 = `2^(h+1) - 1`（h 从 0 开始）。
- **完全二叉树**：除最后一层外全满，最后一层从左到右排列。
  - 数组存储最优
  - 高度 h = `⌊log₂ n⌋`
- **平衡二叉树**：左右子树高度差 ≤ 1
- **二叉查找树（BST）**：左 ≤ 根 ≤ 右
- **线索二叉树**：空指针指向前驱/后继
- **Huffman 树**：带权路径长度最小的二叉树

#### 10.1.4 二叉树的性质（必背）

- **第 i 层最多 2^(i-1) 个节点**（i ≥ 1）
- **高度 h 的二叉树最多 2^(h+1) - 1 个节点**（h ≥ 0，根高度 = 0）
- **叶子数 = 度为 2 的节点数 + 1**（`n₀ = n₂ + 1`）
- **总节点数 = n₀ + n₁ + n₂**
- **完全二叉树**：节点 i（0-indexed）的父 = (i-1)/2，左子 = 2i+1，右子 = 2i+2
- **完全二叉树高度**：`h = ⌊log₂ n⌋`（n 节点）

#### 10.1.5 二叉树的存储

**(1) 顺序存储**
- 用数组存（完全二叉树最合适）
- 节点 i 的左子 = 2i+1，右子 = 2i+2（0-indexed）
- 父 = (i-1)/2
- 一般二叉树会浪费空间（空位用特殊值）

**(2) 链式存储**
```cpp
struct Node {
    T data;
    Node *left, *right;
    Node(T v): data(v), left(nullptr), right(nullptr) {}
};
```
- 优点：节省空间、灵活
- 缺点：不能随机访问
- **三叉链表**：再加 `parent` 指针

#### 10.1.6 二叉树的遍历（必背）

**(1) 深度优先（DFS）**
- **前序（Pre-order）**：根 → 左 → 右
- **中序（In-order）**：左 → 根 → 右
- **后序（Post-order）**：左 → 右 → 根

**递归版**：
```cpp
void preorder(Node* p)  { if (!p) return; visit(p); preorder(p->left); preorder(p->right); }
void inorder(Node* p)   { if (!p) return; inorder(p->left); visit(p); inorder(p->right); }
void postorder(Node* p) { if (!p) return; postorder(p->left); postorder(p->right); visit(p); }
```

**非递归版**（用栈）：
```cpp
// 前序
vector<int> preorder(Node* root) {
    vector<int> res;
    stack<Node*> st;
    st.push(root);
    while (!st.empty()) {
        Node* p = st.top(); st.pop();
        if (!p) continue;
        res.push_back(p->val);
        st.push(p->right);   // 先右后左
        st.push(p->left);
    }
    return res;
}
// 中序
vector<int> inorder(Node* root) {
    vector<int> res;
    stack<Node*> st;
    Node* cur = root;
    while (cur || !st.empty()) {
        while (cur) { st.push(cur); cur = cur->left; }
        cur = st.top(); st.pop();
        res.push_back(cur->val);
        cur = cur->right;
    }
    return res;
}
// 后序（前序改造，根右左，再反转）
vector<int> postorder(Node* root) {
    vector<int> res;
    stack<Node*> st;
    st.push(root);
    while (!st.empty()) {
        Node* p = st.top(); st.pop();
        if (!p) continue;
        res.push_back(p->val);
        st.push(p->left);
        st.push(p->right);
    }
    reverse(res.begin(), res.end());
    return res;
}
```

**(2) 广度优先（BFS / 层序）**
- 用队列
- 一层一层遍历

#### 10.1.7 由遍历序列构造二叉树

- **中序 + 前序** → 唯一确定
- **中序 + 后序** → 唯一确定
- **前序 + 后序** → 不能唯一（除非满二叉树）
- 思路：前序/后序找根，中序分左右，递归

#### 10.1.8 二叉查找树（BST）

- **定义**：左子树所有节点 < 根 < 右子树所有节点
- **中序遍历得到升序序列**
- **操作时间复杂度**：O(h)，h = 树高
  - 平均 O(log n)
  - 最坏 O(n)（退化成链表）

**基本操作**：
- **查找**：从根开始，小左大右
- **插入**：找到空位插入
- **删除**（最复杂）：
  - **叶节点**：直接删
  - **单子树**：子树顶替
  - **双子树**：用**前驱（左子树最大）** 或**后继（右子树最小）** 替代，再删前驱/后继
- **最小/最大**：一直向左/右

#### 10.1.9 平衡二叉树（AVL）—— 重点

- **定义**：BST + 左右子树高度差 ≤ 1
- **平衡因子 BF**：BF = h(left) - h(right)
- **失衡四种情况**：
  - **LL**：插入点在失衡节点的左孩子的左子树
  - **RR**：插入点在失衡节点的右孩子的右子树
  - **LR**：插入点在失衡节点的左孩子的右子树
  - **RL**：插入点在失衡节点的右孩子的左子树
- **调整**：
  - **LL → 右单旋**
  - **RR → 左单旋**
  - **LR → 先左后右双旋**
  - **RL → 先右后左双旋**
- **插入/删除后**：从插入/删除点向上回溯，更新 BF，发现失衡立刻调整
- **时间复杂度**：所有操作 O(log n)
- **空间**：每个节点多存一个 BF

#### 10.1.10 红黑树（Red-Black Tree）

- **5 大性质**：
  1. 每个节点是红或黑
  2. 根是黑
  3. 叶（NIL）是黑
  4. 红节点的子节点必须黑（不能连续红）
  5. 从任一节点到叶的所有路径黑节点数相同
- **保证**：树高 ≤ 2·log(n+1)
- **操作**：旋转 + 变色
- **应用**：C++ `std::map` / `std::set`、Java `TreeMap`、Linux CFS 调度
- **比 AVL 优势**：插入/删除旋转次数少（最多 3 次）

#### 10.1.11 伸展树（Splay Tree）

- **思想**：刚访问的节点"伸展"到根
- **三种操作**：
  - **Zig**：单次旋转（一字型 / 之字型尽头）
  - **Zig-Zig**：一字型连续两次旋转
  - **Zig-Zag**：之字型两次旋转
- **自适应性**：访问频繁的节点靠近根
- **摊还时间**：O(log n)
- **应用**：缓存、文本编辑器撤销

#### 10.1.12 堆（Heap）—— 见第 8 章

#### 10.1.13 Huffman 树（哈夫曼树）

- **带权路径长度 WPL**：`Σ wᵢ · lᵢ`（wᵢ = 权值，lᵢ = 路径长）
- **构造**：每次取两个最小权值的节点合并，O(n log n)
- **应用**：Huffman 编码（数据压缩）
- **性质**：WPL 最小的二叉树

#### 10.1.14 树的应用

- **文件系统**：目录树
- **数据库索引**：B 树 / B+ 树
- **表达式求值**：表达式树
- **决策树**：机器学习

#### 10.1.16 二叉树遍历的非递归实现（必背）

**前序遍历（迭代）**：
```cpp
void preOrder(TreeNode* root) {
    if (!root) return;
    stack<TreeNode*> st;
    st.push(root);
    while (!st.empty()) {
        TreeNode* cur = st.top(); st.pop();
        cout << cur->val << ' ';           // 根
        if (cur->right) st.push(cur->right);  // 右先入栈
        if (cur->left)  st.push(cur->left);   // 左后入栈
    }
}
```
**思路**：根入栈即访问 → 弹出 → 先压右再压左（让左先出栈）

**中序遍历（迭代）—— 重点**：
```cpp
void inOrder(TreeNode* root) {
    stack<TreeNode*> st;
    TreeNode* cur = root;
    while (cur || !st.empty()) {
        while (cur) { st.push(cur); cur = cur->left; }  // 一路向左
        cur = st.top(); st.pop();
        cout << cur->val << ' ';  // 出栈即访问
        cur = cur->right;
    }
}
```
**思路**：每个节点入栈时是"路过"，出栈时才访问

**后序遍历（迭代）—— 最难**：

// 方法 1：前序变体（根右左 → 反转即左右根）

#### 10.1.17 由遍历序列构造二叉树

**前序 + 中序 → 树（必背）**

**原理**：
- 前序第一个是根
- 在中序中找到根位置，左边是左子树，右边是右子树
- 递归构造左右子树

**示例**：前序 = [1,2,4,7,3,5,6,8]，中序 = [4,7,2,1,5,3,8,6]
- 根 = 1（前序首）
- 中序中 1 的位置分割：[4,7,2] | [5,3,8,6]
- 左子树：前序 [2,4,7]，中序 [4,7,2]
- 右子树：前序 [3,5,6,8]，中序 [5,3,8,6]
- 递归...

**代码模板**：
```cpp
TreeNode* buildFromPreIn(vector<int>& pre, vector<int>& in,
                         int pl, int pr, int il, int ir) {
    if (pl > pr) return nullptr;
    TreeNode* root = new TreeNode(pre[pl]);
    int k = il;
    while (k <= ir && in[k] != pre[pl]) k++;  // 找根在中序中的位置
    int leftSize = k - il;  // 左子树节点数
    root->left  = buildFromPreIn(pre, in, pl+1, pl+leftSize, il, k-1);
    root->right = buildFromPreIn(pre, in, pl+leftSize+1, pr, k+1, ir);
    return root;
}
```

**中序 + 后序 → 树**（同理，后序最后一个是根）

**前序 + 后序** ❌ 不能唯一构造（缺中间分割信息）

#### 10.1.18 BST 删除 3 种情况详解（Lab 重点）

**情况 1：删除叶子节点**
- 直接删除，返回 nullptr
```cpp
if (!root->left && !root->right) { delete root; return nullptr; }
```

**情况 2：删除节点只有左子树或只有右子树**
- 用子树顶替
```cpp
if (!root->right) { auto* l = root->left; delete root; return l; }
if (!root->left)  { auto* r = root->right; delete root; return r; }
```

**情况 3：删除节点左右子树都有** —— 最复杂
- 找**前驱**（左子树最大值）或**后继**（右子树最小值）顶替
- 删除前驱/后继节点（递归调用）

```cpp
// 用后继（右子树最小值）顶替
TreeNode* findMin(TreeNode* node) {
    while (node->left) node = node->left;
    return node;
}
TreeNode* deleteNode(TreeNode* root, int key) {
    if (!root) return nullptr;
    if (key < root->val) root->left  = deleteNode(root->left, key);
    else if (key > root->val) root->right = deleteNode(root->right, key);
    else {
        // 找到目标节点
        if (!root->left)  return root->right;
        if (!root->right) return root->left;
        // 情况 3：找后继
        TreeNode* succ = findMin(root->right);
        root->val = succ->val;
        root->right = deleteNode(root->right, succ->val);
    }
    return root;
}
```

**为什么找前驱或后继？** —— 保持 BST 性质（左 < 根 < 右）
- 前驱：左子树最大值 < 根 < 右子树所有值
- 后继：左子树所有值 < 根 < 右子树最小值

#### 10.1.19 AVL 树 4 种旋转详解（重点）

**失衡类型**：插入或删除后，某节点左右子树高度差 > 1

**4 种失衡 + 旋转方案**：

**(1) LL 型（左左）—— 右旋**
- 在失衡节点的**左孩子的左子树**插入导致失衡
```
        5 (失衡)
       /
      3
     /
    2

右旋后：
      3
     / \
    2   5
```
```cpp
TreeNode* rotateRight(TreeNode* y) {
    TreeNode* x = y->left;
    TreeNode* T2 = x->right;
    x->right = y;
    y->left = T2;
    // 更新高度
    y->h = 1 + max(getH(y->left), getH(y->right));
    x->h = 1 + max(getH(x->left), getH(x->right));
    return x;
}
```

**(2) RR 型（右右）—— 左旋**
- 在失衡节点的**右孩子的右子树**插入导致失衡
```
    3 (失衡)
     \
      5
       \
        7

左旋后：
      5
     / \
    3   7
```
```cpp
TreeNode* rotateLeft(TreeNode* x) {
    TreeNode* y = x->right;
    TreeNode* T2 = y->left;
    y->left = x;
    x->right = T2;
    x->h = 1 + max(getH(x->left), getH(x->right));
    y->h = 1 + max(getH(y->left), getH(y->right));
    return y;
}
```

**(3) LR 型（左右）—— 先左旋再右旋**
- 在失衡节点的**左孩子的右子树**插入
```
        5 (失衡)
       /
      3
       \
        4

步骤 1：对 3 做左旋 → 变成 LL 型
        5
       /
      4
     /
    3

步骤 2：对 5 做右旋 → 平衡
      4
     / \
    3   5
```
**代码**：`return rotateRight(root);` 之前先 `root->left = rotateLeft(root->left);`

**(4) RL 型（右左）—— 先右旋再左旋**
- 在失衡节点的**右孩子的左子树**插入
```
    3 (失衡)
     \
      5
     /
    4

步骤 1：对 5 做右旋 → 变成 RR 型
    3
     \
      4
       \
        5

步骤 2：对 3 做左旋 → 平衡
      4
     / \
    3   5
```
**代码**：`return rotateLeft(root);` 之前先 `root->right = rotateRight(root->right);`

**AVL 插入模板**：
```cpp
TreeNode* insert(TreeNode* root, int key) {
    if (!root) return new TreeNode(key);
    if (key < root->val) root->left  = insert(root->left, key);
    else if (key > root->val) root->right = insert(root->right, key);
    else return root;  // 不插重复

    // 更新高度
    root->h = 1 + max(getH(root->left), getH(root->right));
    int balance = getBalance(root);

    // 4 种失衡
    if (balance > 1 && key < root->left->val)        return rotateRight(root);   // LL
    if (balance < -1 && key > root->right->val)       return rotateLeft(root);    // RR
    if (balance > 1 && key > root->left->val) {       // LR
        root->left = rotateLeft(root->left);
        return rotateRight(root);
    }
    if (balance < -1 && key < root->right->val) {     // RL
        root->right = rotateRight(root->right);
        return rotateLeft(root);
    }
    return root;
}
```

**AVL 复杂度**：
- 查找：O(logn)（平衡保证）
- 插入：O(logn)（含旋转）
- 删除：O(logn)
- 高度差：≤ 1

#### 10.1.20 红黑树 5 大性质详解

**5 大性质（必背）**：
1. **每个节点要么红要么黑**
2. **根节点是黑的**
3. **叶子节点（NIL）是黑的**
4. **红色节点的两个孩子必须都是黑的**（不能有连续红节点）
5. **从任一节点到其每个叶子的所有路径都包含相同数目的黑节点**（黑高相同）

**红黑树 vs AVL**：
| | AVL | 红黑树 |
|---|---|---|
| 平衡度 | 严格（高度差 ≤ 1）| 宽松（最长路径 ≤ 2 倍最短）|
| 查找 | ✅ 更快（更平衡）| 稍慢 |
| 插入/删除 | 旋转多（O(logn) 但常数大）| 旋转少（更优）|
| 应用 | 数据库索引 | Java `TreeMap` / C++ `map` / Linux 调度 |

**插入 5 种情况**（简版）：
- 设插入节点为 N，父节点为 P，叔节点为 U，祖父为 G
- **Case 1**：P 黑 → 直接插入，无需调整
- **Case 2**：P 红 + U 红 → P、U 变黑，G 变红，把 G 当新 N 向上递归
- **Case 3**：P 红 + U 黑 + N 是 P 的外侧孩子 → G 旋转（LL/RR）+ 变色
- **Case 4**：P 红 + U 黑 + N 是 P 的内侧孩子 → 先旋转成 Case 3，再按 Case 3 处理
- **Case 5**：根节点 → 染黑

**删除 6 种情况**（超复杂，考试不要求手写）

#### 10.1.21 Huffman 树与编码

**应用**：数据压缩（如 zip、jpeg）

**构造过程**：
1. 把每个字符看作单节点树（带权 = 频次）
2. 每次选两棵**最小权**的树合并，新树权 = 两子树权之和
3. 重复直到只剩一棵树

**示例**：字符 A(5), B(9), C(12), D(13), E(16), F(45)
```
合并 1: A(5) + B(9) = 14 → [14]
合并 2: C(12) + D(13) = 25 → [25, 14]
合并 3: 14 + E(16) = 30 → [25, 30]
合并 4: 25 + F(45) = 70 → [70, 30]
合并 5: 30 + 70 = 100 → [100]
```

**WPL（带权路径长度）** = ∑ wi × li = 5×3 + 9×3 + 12×2 + 13×2 + 16×2 + 45×1 = 224

**Huffman 编码**：
- 左 0 右 1
- A: 000, B: 001, C: 010, D: 011, E: 10, F: 1

**特性**：出现频率高的字符编码短（压缩）

### 11.1 理论全掌握

#### 11.1.1 树的存储（一般树 → 二叉树）

#### 11.1.2 Trie 树（字典树 / 前缀树）—— 重点

- **定义**：每个节点有多个孩子指针（按字符数 26/52/64），专门为字符串集合设计
- **结构**：
  ```cpp
  struct TrieNode {
      TrieNode* children[26] = {nullptr};
      bool isEnd = false;     // 是否单词结尾
  };
  ```
- **操作**：
  - **插入**：从根开始，按字符走，没孩子就建；结尾标 `isEnd`
  - **查找**：按字符走，能走通 + 结尾是 `isEnd` 才算存在
  - **前缀匹配**：能走通即可（不管 `isEnd`）
- **时间复杂度**：O(L)，L = 字符串长度
- **空间复杂度**：O(总字符数 × 字符集大小)
- **应用**：
  - 字符串集合存储
  - 前缀匹配（自动补全、搜索建议）
  - 词频统计
  - 拼写检查
  - IP 路由（最长前缀匹配）
- **优点**：查找前缀效率高
- **缺点**：空间浪费（特别是大量无共同前缀的字符串）
- **压缩 Trie**：合并只有一个孩子的链
- **后缀树 / 后缀数组**：字符串子串问题

#### 11.1.3 B 树（B-Tree）—— 重点

- **动机**：磁盘 I/O 优化（减少访问次数）
- **m 阶 B 树定义**：
  - 每个节点最多 m 个孩子
  - 除根外，每个节点 ≥ ⌈m/2⌉ 个孩子
  - 根节点 ≥ 2 个孩子（除非只有 1 个 key）
  - 有 k 个孩子的节点有 k-1 个 key
  - 所有叶节点在同一层
  - 节点内 key 升序
- **高度**：`h = O(log_m n)`
- **m 越大 → 树高越小 → I/O 越少**
- **操作**：
  - 查找：从根开始，二分（节点内有序）
  - 插入：找到叶节点插入；满了就**分裂**
  - 删除：合并或借位
- **应用**：
  - 数据库索引
  - 文件系统
  - 外存数据结构

#### 11.1.4 B+ 树（B+ Tree）

- **vs B 树**：
  - B+ 树的**数据只在叶节点**
  - B+ 树的叶节点用**链表**相连
  - B+ 树内部节点的 key 是子树 key 的副本（不是数据）
- **优点**：
  - 范围查询快（链表连接）
  - 非叶节点只存索引，能存更多 key → 树更矮
  - 全树遍历方便（直接走链表）
- **应用**：
  - **MySQL InnoDB** 的主键索引
  - 关系数据库
- **聚簇索引 vs 非聚簇索引**：
  - 聚簇：数据按主键顺序存储（InnoDB）
  - 非聚簇：数据单独存，索引只存指针（MyISAM）

#### 11.1.5 多叉树比较

| 结构 | 用途 | 时间复杂度 | 空间 |
|---|---|---|---|
| Trie | 前缀匹配 | O(L) | 字符集 × 节点数 |
| B 树 | 外存索引 | O(log_m n) | 紧凑 |
| B+ 树 | 数据库 | O(log_m n) | 更紧凑 |
| 红黑树 | 内存平衡 | O(log n) | 节点 2-3 倍开销 |
| AVL | 内存平衡 | O(log n) | 节点 BF |

#### 11.1.6 常见考点

- Trie 树的插入/查找
- B 树的阶、节点数、高度
- B 树 vs B+ 树的区别
- 数据库索引原理
- 磁盘 I/O 与树高的关系

## 第 12 章 图

### 12.1 理论全掌握

#### 12.1.1 图的定义

- **图 G = (V, E)**：顶点集 V + 边集 E
  - V：顶点（vertex）的非空有限集
  - E：边（edge）的有限集
- **有向图 / 无向图**
- **加权图 / 无权图**
- **简单图 vs 多重图**：是否有重边/自环
- **稀疏图 / 稠密图**：边数远小于 V² / 接近 V²

#### 12.1.2 图的术语

- **度（Degree）**：
  - 无向图：`deg(v)` = 与 v 关联的边数
  - 有向图：`入度 in(v)` + `出度 out(v)` = `deg(v)`
- **邻接（Adjacent）**：两点有边相连
- **路径（Path）**：顶点序列，相邻顶点有边
- **简单路径**：顶点不重复
- **回路 / 环（Cycle）**：起点 = 终点
- **连通 / 强连通**：
  - 无向图：连通（任何两点有路径）
  - 有向图：强连通（任何两点双向可达）
- **连通分量 / 强连通分量**
- **生成树**：连通图含所有顶点的极小连通子图（V-1 条边）
- **完全图**：每对顶点都有边，n 顶点的无向完全图 Kₙ 有 n(n-1)/2 条边
- **子图 / 生成子图**

#### 12.1.3 握手定理

- 无向图：`Σ deg(v) = 2|E|`
- 有向图：`Σ in(v) = Σ out(v) = |E|`
- 度为奇数的顶点个数是偶数

#### 12.1.4 图的存储

**(1) 邻接矩阵**
- 适合**稠密图**
- 空间 O(V²)
- 判边存在 O(1)
- 遍历邻居 O(V)
```cpp
vector<vector<int>> adj(V, vector<int>(V, 0));
// 加边：adj[u][v] = w;
```

**(2) 邻接表**
- 适合**稀疏图**
- 空间 O(V+E)
- 判边存在 O(度数)
- 遍历邻居 O(度数)
```cpp
vector<vector<pair<int,int>>> adj(V);   // (邻居, 权值)
// 加边：adj[u].push_back({v, w});
```

**(3) 十字链表**（有向图）
- 节点 = `{data, firstIn, firstOut}`
- 边节点 = `{tail, head, hLink, tLink, weight}`
- 空间 O(V+E)

**(4) 邻接多重表**（无向图）
- 一条边只存一次
- 节省空间

#### 12.1.5 图的遍历

**(1) DFS（深度优先搜索）**
- 用**栈**（或递归）
- 步骤：从起点开始，访问 → 标记 → 递归访问未访问的邻居
- **时间复杂度**：O(V+E)（邻接表）/ O(V²)（邻接矩阵）
- **应用**：
  - 连通性
  - 路径查找
  - 拓扑排序（后序）
  - 连通分量
  - 强连通分量
```cpp
void dfs(int u) {
    visited[u] = true;
    for (int v : adj[u]) {
        if (!visited[v]) dfs(v);
    }
}
```

**(2) BFS（广度优先搜索）**
- 用**队列**
- 步骤：起点入队 → 弹出 → 访问 → 未访问邻居入队
- **时间复杂度**：O(V+E)
- **应用**：
  - 无权图最短路径
  - 层次遍历
  - 连通分量
```cpp
void bfs(int s) {
    queue<int> q;
    q.push(s); visited[s] = true; dist[s] = 0;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : adj[u]) {
            if (!visited[v]) {
                visited[v] = true;
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}
```

#### 12.1.6 拓扑排序

- **适用**：DAG（有向无环图）
- **Kahn 算法**（BFS）：
  1. 选入度 = 0 的顶点
  2. 删除该顶点 + 其出边
  3. 重复
- **DFS 法**：后序遍历，再反转
- **时间**：O(V+E)
- **检测环**：拓扑序列个数 < V → 有环
- **应用**：
  - 课程排课
  - 编译依赖
  - 任务调度

```cpp
vector<int> topo(int n) {
    vector<int> indeg(n, 0);
    for (int u = 0; u < n; u++)
        for (auto [v, w] : adj[u]) indeg[v]++;
    queue<int> q;
    for (int i = 0; i < n; i++) if (indeg[i] == 0) q.push(i);
    vector<int> res;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        res.push_back(u);
        for (auto [v, w] : adj[u]) {
            if (--indeg[v] == 0) q.push(v);
        }
    }
    return res;   // size < n 表示有环
}
```

#### 12.1.7 最短路径

| 算法 | 适用 | 边权 | 时间 | 思想 |
|---|---|---|---|---|
| BFS | 无权 | - | O(V+E) | 队列层序 |
| **Dijkstra** | 单源、非负权 | ≥ 0 | O((V+E) log V)（堆优化） | 贪心 |
| Bellman-Ford | 单源、可负权 | 任意 | O(VE) | 动态规划 |
| **Floyd-Warshall** | 全对 | 任意 | O(V³) | 动态规划 |
| SPFA | 单源、负权 | 任意 | 平均 O(E) | Bellman-Ford 队列优化 |

**(1) Dijkstra（重点）**
- **限制**：边权非负
- **步骤**：
  1. `dist[s] = 0`，其他 `= ∞`
  2. 选 `dist` 最小的未访问顶点 `u`
  3. 遍历 `u` 的邻居 `v`，尝试松弛：`dist[v] = min(dist[v], dist[u] + w(u,v))`
  4. 标记 `u` 已访问
  5. 重复直到所有顶点访问完
- **堆优化**：用 priority_queue 存 `(dist, v)`，O((V+E) log V)
```cpp
vector<long long> dist(n, LLONG_MAX);
dist[s] = 0;
priority_queue<pair<long long,int>, vector<pair<long long,int>>, greater<>> pq;
pq.push({0, s});
vector<bool> visited(n, false);
while (!pq.empty()) {
    auto [d, u] = pq.top(); pq.pop();
    if (visited[u]) continue;
    visited[u] = true;
    for (auto [v, w] : adj[u]) {
        if (dist[u] + w < dist[v]) {
            dist[v] = dist[u] + w;
            pq.push({dist[v], v});
        }
    }
}
```

**(2) Bellman-Ford**
- **可检测负环**
- **步骤**：松弛 V-1 轮所有边
- **时间**：O(VE)
- **优化**：第 i 轮只考虑上一轮被更新的节点的出边（SPFA）

**(3) Floyd-Warshall**
- **全对最短路**
- **状态**：`dist[i][j]` = i 到 j 当前最短距离
- **转移**：`dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])` for k = 0..V-1
- **时间**：O(V³)
- **空间**：O(V²)
- **可检测负环**：`dist[i][i] < 0` 则有负环
```cpp
for (int k = 0; k < n; k++)
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]);
```

#### 12.1.8 最小生成树（MST）

- **适用**：连通**无向加权**图
- **性质**：
  - MST 有 V-1 条边
  - 不唯一（边权相同时）
  - **切割性质**：跨任意切割的最短边必在某个 MST 中
  - **环性质**：环上最长的边不在 MST 中

| 算法 | 思路 | 时间 | 适用 |
|---|---|---|---|
| **Prim** | 从一个顶点出发，长边扩展 | O(E log V) | 稠密图 |
| **Kruskal** | 按边权排序，并查集 | O(E log E) | 稀疏图 |

**(1) Prim**
- 维护"已加入 MST 的顶点集合"和"候选边"
- 每次选候选边中权值最小的，把对应顶点加入
- 用堆优化

**(2) Kruskal**
- 边按权值升序
- 依次考虑：若两端点不连通（并查集）→ 加入 MST
- 加到 V-1 条边为止
- **并查集（Union-Find）**：路径压缩 + 按秩合并，O(α(n))

```cpp
// Kruskal
struct Edge { int u, v, w; };
vector<Edge> edges;
sort(edges.begin(), edges.end(), [](auto& a, auto& b){ return a.w < b.w; });
vector<int> parent(n);
for (int i = 0; i < n; i++) parent[i] = i;
int find(int x) { return parent[x] == x ? x : parent[x] = find(parent[x]); }
int mst = 0, cnt = 0;
for (auto& e : edges) {
    int pu = find(e.u), pv = find(e.v);
    if (pu != pv) {
        parent[pu] = pv;
        mst += e.w;
        if (++cnt == n-1) break;
    }
}
```

#### 12.1.9 强连通分量（SCC）

- **定义**：极大强连通子图
- **Kosaraju 算法**：
  1. DFS 原图，按完成时间倒序
  2. 在反图上按这个顺序再 DFS，每个连通块就是一个 SCC
  3. 时间 O(V+E)
- **Tarjan 算法**：单次 DFS，用栈和 low 标记

#### 12.1.10 关键路径（AOE 网）

- **AOE 网**：边表示活动、点表示事件
- **关键路径**：从源点到汇点的最长路径（决定工程最短工期）
- **关键活动**：关键路径上的活动（无时差）

#### 12.1.11 图的应用

| 场景 | 算法 |
|---|---|
| 工程调度（依赖关系） | 拓扑排序 |
| 地图导航 | Dijkstra / Floyd |
| 网络布线 | 最小生成树 |
| 社交关系 | BFS/DFS（连通分量） |
| 任务依赖 | 关键路径 |
| 城市间最短 | Floyd |
| 单源最短路（含负权） | Bellman-Ford / SPFA |
| 网页排名 | PageRank（图的迭代） |
| 推荐系统 | 图嵌入 / 随机游走 |

#### 12.1.12 常见考点

- 邻接矩阵 / 邻接表的转换
- DFS / BFS 遍历（手写）
- 拓扑排序（Kahn / DFS）
- 最短路（Dijkstra / Floyd）
- 最小生成树（Prim / Kruskal）
- 强连通分量（Kosaraju / Tarjan）

## 附录 C: 常见坑位速查

| 坑 | 避坑 |
|---|---|
| `mid*mid` 溢出 int | 用 `long long` |
| 环形队列 front==rear 判空还是判满 | 用 count 变量更稳 |
| 链表反转没保存 next | 一定要先存 `nxt = cur->next` |
| BST 删除双子树 | 用前驱或后继，再删前驱/后继 |
| 中缀表达式负数 | 表达式开头 / `(` 后 / 运算符后：补 0 |
| 输出空格 | 行内空格分隔，行末不空格 |
| 输入 `-1` 结束 | 一定要在读入循环里 break |
| 排序中间结果 | 在**插入/选择次数 = n/2** 时输出 |
| 堆 1-indexed vs 0-indexed | 父子关系公式差 1 |
| 哈希表删除 | 链地址法可直接删；开放定址要墓碑 |
| 哈希表取模 | 质数 p |
| 顺序表扩容 | Lab 题目固定容量即可（题目给 <100 等） |
| 字符串 hash | BKDR 公式 `h = h*seed + ch` |
| Floyd 二维数组 | `dist[i][j] = min(dist[i][j], dist[i][k]+dist[k][j])` |
| DFS 递归深度 | 树深 ≤ 10⁴，递归可能栈溢出（Lab 不涉及超大） |
| AVL 旋转 | 失衡点在祖先，回溯到第一个失衡的调整 |
| 红黑树 | 5 大性质，插入/删除旋转+变色 |
| B 树分裂 | 中间 key 上升，左右两半 |
| 拓扑排序 | Kahn 用入度，DFS 用后序 |
| MST | 切割性质、环性质 |
| 关键路径 | 最早开始 = 入点最大；最晚开始 = 出点最小 - 持续 |
| 强连通 | Kosaraju 两遍 DFS（反图） |
