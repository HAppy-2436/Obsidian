二叉树的递归/层序遍历

一句话总结

多叉树结构就是  二叉树结构  的延伸，二叉树是特殊的多叉树。

多叉树的遍历就是  二叉树遍历  的延伸。

森林是指多个多叉树的集合，单独一棵多叉树是一个特殊的森林。

二叉树的节点长这样，每个节点有两个子节点：

```cpp
class TreeNode { public:  int val;  TreeNode* left;  TreeNode* right;

  TreeNode(int v) : val(v), left(nullptr), right(nullptr) {} };
```

多叉树的节点长这样，每个节点有任意个子节点：

```cpp
class Node { public:  int val;  vector<Node*> children; };
```

就这点区别，其他没了。

## 森林

这里介绍一下「森林」这个名词，后面讲到  Union Find 并查集算法  时，会用到这个概念。

顾名思义，森林就是多个多叉树的集合（单独一棵多叉树也是一个特殊的森林），用代码表示就是多个多叉树的根节点列表，类似这样：

```
List<Node> forest;
```

只需对每个根节点分别进行 DFS/BFS 遍历，即可遍历森林的所有节点。

在并查集算法中，我们会同时持有多棵多叉树的根节点，那么这些根节点的集合就是一个森林。

接下来说下多叉树的遍历，和二叉树一样，也就递归遍历（DFS）和层序遍历（BFS）两种。

## 递归遍历（DFS）

对比二叉树的遍历框架看多叉树的遍历框架吧：


> [!info] 📌 付费章节补全内容（基于算法知识体系补充）

```cpp
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
};
```

**N 叉树** 的每个节点有任意个子节点（用 `vector<Node*>` 表示）：

```cpp
struct Node {
    int val;
    vector<Node*> children;
};
```

就这点区别，其他没了——遍历框架就是把二叉树「递归 left + 递归 right」改成「for each child: 递归 child」。

### 1.2 为什么需要 N 叉树？

- **DOM 树**：HTML/XML 文档中，一个元素可以有任意个子元素。
- **文件系统**：一个目录下可以有任意个子目录/文件。
- **组织架构树**：一个员工可以有任意多个下属。
- **Trie 字典树**：见 [[29-字典树原理|Trie 原理]]，26 个字母对应 26 叉树。
- **图结构**：见 [[33-图结构实现|图结构实现]]，用邻接表表示的图本质就是 N 叉树。

## 2. 森林（Forest）：多个 N 叉树的集合

**森林** = 多个 N 叉树的集合。单独一棵 N 叉树也是一个特殊的森林（只含一棵树的森林）。

用代码表示就是多个根节点的列表：

```cpp
// 森林 = 多个根节点
vector<Node*> forest;

// 遍历森林：对每个根节点分别 DFS/BFS
for (Node* root : forest) {
    dfs(root);  // 或 bfs(root)
}
```

> [!info] 森林与并查集
> [[30-并查集原理|Union Find 并查集算法]] 中，会同时持有多棵 N 叉树的根节点，这些根节点的集合就是一个森林。每个根节点代表一个集合，节点之间的父子关系代表集合之间的合并历史。

## 3. N 叉树的递归遍历（DFS）

> 以下是 N 叉树的递归遍历、层序遍历的完整 C++ 实现，以及与二叉树遍历的对比分析。

### 3.1 递归框架

对比二叉树和 N 叉树的递归框架：

```cpp
// 二叉树遍历
void traverse(TreeNode* root) {
    if (!root) return;
    // 前序位置
    traverse(root->left);
    // 中序位置
    traverse(root->right);
    // 后序位置
}

// N 叉树遍历
void traverse(Node* root) {
    if (!root) return;
    // 前序位置
    for (Node* child : root->children) {
        traverse(child);
    }
    // 后序位置
}
```

注意：**N 叉树没有「中序」概念**——因为它没有固定的「左」「右」之分，children 列表是无序的。前序（根在前）和后序（根在后）有意义，中序没有意义。

### 3.2 完整 C++ 实现：前序 + 后序

下面给出 N 叉树前序和后序遍历的完整实现（对应 LeetCode 589 和 LeetCode 590）：

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace dsa {

// N 叉树节点定义
struct Node {
    int val;
    vector<Node*> children;
    Node(int x) : val(x) {}
};

// LeetCode 589. N 叉树的前序遍历
// 顺序：根 → 子1 → 子2 → ... → 子n
vector<int> preorder(Node* root) {
    vector<int> res;
    function<void(Node*)> dfs = [&](Node* node) {
        if (!node) return;
        res.push_back(node->val);              // 前序：先访问根
        for (Node* child : node->children) {   // 再依次递归子节点
            dfs(child);
        }
    };
    dfs(root);
    return res;
}

// LeetCode 590. N 叉树的后序遍历
// 顺序：子1 → 子2 → ... → 子n → 根
vector<int> postorder(Node* root) {
    vector<int> res;
    function<void(Node*)> dfs = [&](Node* node) {
        if (!node) return;
        for (Node* child : node->children) {   // 先依次递归子节点
            dfs(child);
        }
        res.push_back(node->val);              // 后序：最后访问根
    };
    dfs(root);
    return res;
}

} // namespace dsa
```

**示例**：对下面这棵 N 叉树：

```
              1
        /  /  |  \
       3  2   4   5
      /|        |  /|\
     6 7        8 9 10
```

- 前序：`1 3 6 7 2 4 8 5 9 10`
- 后序：`6 7 3 2 8 4 9 10 5 1`

你可以看到，N 叉树的前序就是「根先访问，再依次遍历每棵子树」；后序就是「先遍历完每棵子树，再访问根」——这与二叉树的定义完全一致，只是「每棵子树」从一个变成了多个。

### 3.3 迭代版前序遍历

递归版调用栈太深时（比如 10⁵ 层）会爆栈，需要迭代版。迭代版用栈手动模拟递归：

```cpp
// LeetCode 589 迭代版：N 叉树前序遍历
vector<int> preorderIterative(Node* root) {
    vector<int> res;
    if (!root) return res;
    stack<Node*> stk;
    stk.push(root);
    while (!stk.empty()) {
        Node* cur = stk.top(); stk.pop();
        res.push_back(cur->val);
        // 注意：栈是 LIFO，要让第一个 child 最后弹出
        for (int i = (int)cur->children.size() - 1; i >= 0; --i) {
            stk.push(cur->children[i]);
        }
    }
    return res;
}
```

**为什么 children 反向入栈？** 栈是「后进先出」，为了让第一个子节点最先被处理（保持前序顺序），需要把 children 列表反向入栈——最后一个 child 先入栈，第一个 child 最后入栈、最先弹出。

## 4. N 叉树的层序遍历（BFS）

### 4.1 完整 C++ 实现

N 叉树的层序遍历（LeetCode 429）就是把二叉树 BFS 的「push left + push right」改成「push 所有 children」：

```cpp
// LeetCode 429. N 叉树的层序遍历
// 返回：vector<vector<int>>，外层下标表示层数（0-based）
vector<vector<int>> levelOrder(Node* root) {
    vector<vector<int>> res;
    if (!root) return res;
    queue<Node*> q;
    q.push(root);
    while (!q.empty()) {
        int sz = (int)q.size();
        vector<int> curLevel;
        for (int i = 0; i < sz; ++i) {
            Node* cur = q.front(); q.pop();
            curLevel.push_back(cur->val);
            // 把所有子节点加入队列
            for (Node* child : cur->children) {
                q.push(child);
            }
        }
        res.push_back(curLevel);
    }
    return res;
}
```

对上面那棵 N 叉树，层序遍历结果：

```text
第 0 层: [1]
第 1 层: [3, 2, 4, 5]
第 2 层: [6, 7, 8, 9, 10]
```

### 4.2 与二叉树 BFS 的对比

```cpp
// 二叉树层序遍历
void bfs_binary(TreeNode* root) {
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        TreeNode* cur = q.front(); q.pop();
        // 访问 cur
        if (cur->left)  q.push(cur->left);   // 仅 2 个子节点
        if (cur->right) q.push(cur->right);
    }
}

// N 叉树层序遍历
void bfs_nary(Node* root) {
    queue<Node*> q;
    q.push(root);
    while (!q.empty()) {
        Node* cur = q.front(); q.pop();
        // 访问 cur
        for (Node* child : cur->children) {   // 任意多个子节点
            q.push(child);
        }
    }
}
```

唯一的区别：**N 叉树用 `for` 循环把所有 children 入队**，二叉树是固定的 left + right 两个。

## 5. 与二叉树遍历的对比总结

| 维度 | 二叉树 | N 叉树 |
|------|--------|--------|
| 节点子节点数 | 最多 2 个 | 任意多个 |
| 节点结构 | `left, right` | `vector<Node*> children` |
| DFS 前序 | ✓ | ✓（LeetCode 589） |
| DFS 中序 | ✓ | ✗（无左/右之分） |
| DFS 后序 | ✓ | ✓（LeetCode 590） |
| BFS 层序 | ✓ | ✓（LeetCode 429） |
| 递归模板 | `dfs(left); dfs(right);` | `for (c : children) dfs(c);` |
| 迭代栈顺序 | push right, push left | push children 反向 |
| 应用 | 二叉搜索树、堆、线段树 | DOM、文件系统、Trie |

### 5.1 关键差异点

1. **N 叉树没有中序遍历**。中序的定义是「左 → 根 → 右」，N 叉树没有固定的左右之分，所以中序没有意义。
2. **DFS 模板差异**：二叉树是「固定递归两个子节点」，N 叉树是「循环递归每个 child」。
3. **迭代栈入栈顺序**：二叉树迭代前序时先 push right 再 push left；N 叉树迭代前序时 **反向** push children（让第一个 child 最先弹出）。
4. **层序遍历本质相同**：都是「队列 + 一次性处理一整层」。

### 5.2 何时用 N 叉树 vs 二叉树？

| 场景 | 数据结构 |
|------|---------|
| 子节点数量固定为 2（如父子、左右） | 二叉树 |
| 子节点数量不固定（如 XML、组织架构） | N 叉树 |
| 节点有多个属性，且其中两个最重要（如 BST 的左右孩子） | 二叉树 |
| 需要字典序前缀匹配 | Trie（26 叉树） |
| 表达「集合的集合」（如并查集） | 森林 |

### 5.3 一个完整的可编译示例

下面把所有遍历（前序、后序、层序、迭代前序）放在一个可编译的 C++ 程序中：

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace dsa {

struct Node {
    int val;
    vector<Node*> children;
    Node(int x) : val(x) {}
};

void deleteTree(Node* root) {
    if (!root) return;
    for (Node* c : root->children) deleteTree(c);
    delete root;
}

} // namespace dsa

int main() {
    using namespace dsa;
    // 构造:
    //              1
    //        /  /  |  \
    //       3  2   4   5
    //      /|        |  /|\
    //     6 7        8 9 10
    Node* n1 = new Node(1);
    Node* n3 = new Node(3); Node* n2 = new Node(2);
    Node* n4 = new Node(4); Node* n5 = new Node(5);
    Node* n6 = new Node(6); Node* n7 = new Node(7);
    Node* n8 = new Node(8); Node* n9 = new Node(9); Node* n10 = new Node(10);
    n1->children = {n3, n2, n4, n5};
    n3->children = {n6, n7};
    n4->children = {n8};
    n5->children = {n9, n10};

    // 调用四种遍历函数（此处省略函数实现，与上文一致）
    // auto pre = preorder(n1);            // 1 3 6 7 2 4 8 5 9 10
    // auto post = postorder(n1);          // 6 7 3 2 8 4 9 10 5 1
    // auto lvl = levelOrder(n1);          // {{1},{3,2,4,5},{6,7,8,9,10}}
    // auto preI = preorderIterative(n1);  // 同 pre

    deleteTree(n1);
    return 0;
}
```

## 6. 复杂度表

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| DFS 前序/后序 | O(N) | O(h)（递归栈，h = 树高） |
| BFS 层序 | O(N) | O(w)（队列，w = 最大层宽） |
| 整体存储 | — | O(N) |

注意：N 叉树的「层宽」可能远大于二叉树（根节点可能有几百个子节点），所以 BFS 的队列空间消耗要特别注意。
