## 二、逐模块讲解（要点）

### 1) 后端接口封装（`src/api/index.ts`）
- 是什么：所有 REST 请求的集中封装层，返回类型为 `ApiResponse<T>`。
- 关键点：
  - 分模块组织（认证、题目、题库、上传/任务、用户、LLM 设置、导出）
  - 筛选参数转换：前端 `FilterParams`  后端 query（subjects 用逗号拼接，dateRange 拆成 startDate/endDate）
  - 上传流程的三段接口：`uploadFile`（FormData） `processFile`（提交处理） `getTaskStatus`（轮询）
- 演示代码片段：展示 `fetchQuestions` 如何拼接 query

---

### 2) 上传页与队列
- 主要文件：`UploadPage.vue`、`UploadQueue.vue`、`ProcessingCard.vue`
- 用户交互：支持拖拽/点击/剪贴板粘贴三种方式上传
- 三段式处理流程（重点讲解）：
  1. 上传文件到服务器，得到 `fileId`（进度显示 10%30%）
  2. 提交 `fileId` 给后端开始处理，得到 `taskId`（进度 35%45%）
  3. 轮询 `getTaskStatus(taskId)` 每 2s 更新进度，直到 success 或 failed（进度 45%100%）
- 队列功能：拖拽排序、清除已完成、批量处理（串行）
- 错误与重试：失败显示错误信息，支持一键重试

---

### 3) 筛选组件与题库/提交列表
- 主要文件：`FilterBar.vue`、`QuestionListPage.vue`、`QuestionBankPage.vue`
- FilterBar 的职责：关键字防抖（500ms）、学科多选、难度、状态、日期范围（7 天/30 天 快捷）
- 数据流：FilterBar emit -> 父组件更新 store.filters -> 调用 store.loadQuestions()
- 页面功能：表格 / 卡片 双视图、分页、批量选择（添加标签 / 删除）

---

### 4) 公共工具函数（`src/utils/helpers.ts`）
- 常用函数：`formatFileSize`、`formatDateTime`、`truncateText`（会替换 LaTeX 为占位）、`generateId`、`debounce`、`daysAgo`
- 讲点：强调为什么抽离（复用、测试、统一格式）

---

### 5) 错误处理与容错
- 网络层统一拦截（`src/api/http.ts`）
  - 请求拦截：自动挂载 `Authorization: Bearer <token>`
  - 响应拦截：401 清除 token 并跳转登录；其他错误统一提示
- 上传层容错：入队时做格式/大小/重复校验；处理流程上用 try/catch，失败有友好文案（如 `NOT_A_QUESTION` 对应中文提示）；支持重试
- 改进建议（可在答辩提出）：WebSocket 替代轮询、并发上传池、token 刷新机制

## 四、可能被问的问题与建议回答
- Q: 为什么拆成三步而不是一次完成？
  - A: OCR+LLM 分析耗时，三步可提供更好 UX（即时上传反馈 + 后端异步处理 + 前端轮询/通知）。

- Q: 为什么轮询而不是 WebSocket？
  - A: 轮询实现简单，后端无额外 WebSocket 服务时更可靠；若要实时性和节省资源，可以用 WebSocket 替代。

- Q: 并发怎么控制？
  - A: 当前实现串行处理，避免瞬时并发压力；可使用并发池（例如 p-limit）限制同时处理数量（例如 3）

- Q: 如何保证 API Key 的安全？
  - A: 前端只显示设置界面，真实的密钥应由后端加密存储与使用（不要在浏览器长期存储敏感密钥）。

- Q: LLM 返回不稳定如何处理？
  - A: 后端应有重试策略与返回明确的 errorCode；前端展示友好提示并允许用户重试或人工校正
