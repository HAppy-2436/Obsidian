# 前端答辩防身指南（常见问题与装杯黑话）

这份指南专门为你应对本科学长提问准备。左边是通俗的交互解释，右边是专业术语包装（用来装杯）。答辩时可以随时看里面的触发词与我对你的小提示来组织语言。

##  一、 常见问答（通俗版，本科学长最爱问的交互问题）

### 1. 【上传相关交互】
- **触发词**：怎么传文件的、可以一张张传吗、支持什么格式
- **你的回答要点**：
  1. 支持三种方法：点按钮选文件、直接把文件拖进来、或者按 `Ctrl+v` 粘贴剪贴板里刚截图的图片。
  2. 格式支持 JPG、PNG 图片，也支持 PDF（我们会给 PDF 生成一个默认的缩略图）。
  3. 大小限制了 10MB，而且我们在加进上传列表前会检查文件名和大小，防止重复添加。
  - *(背诵小动作：代码在 `UploadPage.vue` 里的 `handleFilesSelected` 和 `handlePaste` 函数)*

### 2. 【进度条怎么动的 / 怎么知道处理完了没】
- **触发词**：进度条怎么做的、识别慢怎么办、假进度条？
- **你的回答要点**：
  1. 进度条不是瞎跑的，是分三步：刚传图片到服务器算 10%~30%，告诉后端开始识别算 30%~45%。
  2. 然后前端会每隔 2 秒去问一次后端好了没（这叫轮询）。
  3. 后端会返回一个它的识别进度，前端把它折算成 45%~100% 之间的进度条显示给用户。
  - *(背诵小动作：核心机制是 `setInterval/setTimeout` 轮询，在 `UploadPage.vue` 的 `pollTask` 里)*

### 3. 【拖拽列表 / 队列怎么实现的】
- **触发词**：怎么能拖动啊、列表怎么排序、用什么库
- **你的回答要点**：
  1. 我们用了一个很流行的组件叫 `vuedraggable`，它是基于 Sortable.js 封装的。
  2. 我只要把数组绑给它，用户拖拽完，它会自动把数组顺序更新，我们立刻就能在界面看到新排序。
  - *(背诵小动作：在 `UploadQueue.vue` 里用 `<draggable>` 标签包住了我们的卡片组件)*

### 4. 【表格怎么筛选和搜索的】
- **触发词**：条件筛选怎么做的、搜索卡不卡、怎么防抖的
- **你的回答要点**：
  1. 搜索框做了防抖处理，就是用户连续打字时不会立刻发请求，停下半秒钟（500ms）才发送，减轻服务器压力。
  2. 科目下拉支持多选，选完后加上难度、状态、时间范围，组合成一个大对象，统一发给后端获取过滤好的数据。
  - *(背诵小动作：在 `FilterBar.vue` 里用了 `debounce` 工具函数)*

### 5. 【分页是怎么做的】
- **触发词**：翻页前端做还是后端做、数据多会不会卡
- **你的回答要点**：
  1. 是服务端分页，不是把所有数据一次性拿回来的假分页。
  2. 每次点击下一页或者改变每页显示几条，我们就会用 Vue 的 `watch` 监听这些变化，然后重新请求那页的数据。
  - *(背诵小动作：在 `QuestionListPage.vue` 里 `watch([currentPage, pageSize], ...)` )*

### 6. 【数学公式是怎么渲染到页面上的】
- **触发词**：公式怎么显示、为什么公式不乱码、KaTeX
- **你的回答要点**：
  1. 我们用了一个叫 `KaTeX` 的库包，比老牌的 MathJax 渲染快得多。
  2. 我们写了个组件把包含美元符号 `$$` 和 `$` 的长文本找出来，然后替换成漂亮的公式 HTML 节点。如果碰到有语法错误的公式，我们也不让它崩溃，直接原样当做普通代码展示作为兜底（try-catch）。
  - *(背诵小动作：在 `LatexRenderer.vue` 里用的正则替换然后调了 `katex.renderToString`)*

### 7. 【报错了前端怎么提示 / 网络错误处理】
- **触发词**：提交失败怎么办、登录过期怎么办
- **你的回答要点**：
  1. 我在全局网络请求（Axios）的地方统一做了拦截，所以代码不用到处写报什么错。
  2. 只要后端返回未经授权 401，我就让他直接跳回登录页；如果是上传的图片根本没有题目，轮询时拿到 `NOT_A_QUESTION` 的错误码，我就会弹个红字未识别到题目，请上传包含试卷题目的图片。
  - *(背诵小动作：在 `http.ts` 的 `interceptors` 里统一管，特殊错误在轮询函数里特判)*

### 8. 【API架构与稍有深度的网络请求问题】（预判学长装杯专用）
- **触发词**：跨域是怎么解决的、怎么带 Token 的、为什么不用 WebSocket、文件是怎么发给后端的
- **你的回答要点**：
  1. **关于跨域 (CORS)**：本地开发时，我们在 `vite.config.ts` 里配置了 proxy 代理（将 `/api` 开头的请求转发到 `localhost:8080` 后端），避开了浏览器的同源策略限制；线上部署时同理用 Nginx 转发或者把前后端扔在同一个域下。
  2. **关于 Token 与权限**：登录成功后拿到 token 存在 `localStorage`，我在 `src/api/http.ts` 写了**请求拦截器（Request Interceptor）**，每次发请求都会在 Header 里自动带上 `Authorization: Bearer <token>`，这样不用每个接口都去手写传 token。
  3. **关于为什么轮询而不用 WebSocket (长连接)**：由于大模型和 OCR 处理一张试卷差不多只要十几秒，我们综合评估后采用了每 2 秒**长轮询 (Polling)** 的方案。实现成本低，也不用维护麻烦的 WebSocket 连接池，对于这个量级的并发完全足够了（主打一个实用主义）。
  4. **关于文件上传的特殊性**：普通接口传 JSON，但发文件时，我在 `api/index.ts` 里专门用原生的 `FormData` 对象把文件包起来，并且让 Axios 设置请求头为 `multipart/form-data`，这样后端才能正确接收二进制流。
  5. **如果你被挑刺代码怎么那么多 any**：可以直接回击：我们引入了泛型 `ApiResponse<T>`，把所有接口的返回数据都做了严格的强类型约束，点进 `src/api/index.ts` 全是强类型的，根本不会有隐式 any 的问题。

##  二、 【终极装杯指南】新手伪装大佬的词汇与句式替换
学长喜欢听黑话，把你大白话的总结替换成这些专业术语，瞬间拉开差距：

### 1. 核心缩写/黑话词典（听懂学长在问啥）
- **SPA (Single Page Application)**：单页应用（咱们的 Vue 项目就是 SPA，页面不会刷新，只替换组件）。
- **CORS / 跨域**：浏览器限制不同域名互相访问（用 Proxy 代理解决的）。
- **Token / JWT (JSON Web Token)**：那个放在请求头里用来证明我是我的长字符串。
- **TS (TypeScript)**：带类型检查的 JS，我们项目最大的亮点，保证了类型安全 (Type Safety)。
- **Store / 状态管理**：Pinia，用来把所有组件都需要的变量存起来（替代了老一辈常用的 Vuex）。
- **组合式 API (Composition API)**：我们 `script setup` 这种写法，比老 Vue2 的选项式写法更优雅。
- **拦截器 (Interceptor)**：统一拦截进出的网络请求，做加 Token 和判错的地方。
- **兜底 / 优雅降级**：报错了程序不崩溃，而是给个温和的提示或者默认画面（比如公式出错时的 try-catch）。
- **轮询 (Polling)**：每隔 X 秒去服务器问一次进度（对应那个拉取处理进度的机制）。
- **防抖 (Debounce)**：连续操作只认最后一次（比如搜索框停打字 500ms 后才发请求）。
- **抽象 / 封装**：把一坨代码单独剥离出去变成组件，比如 `FilterBar`（可以说为了高内聚低耦合）。

### 2. 词汇降维打击（把大白话换成说辞）
-  我把代码分开放了   我把各个模块抽象成了独立的 UI 组件，实现了**高内聚低耦合**。
-  它报错了我就提示一下   我们在全局做了一套**异常捕获与容错机制**，保证了前端的**鲁棒性 (Robustness)**。
-  进度条是一点点走的   采用了**异步任务的长轮询机制**，结合虚假进度算法优化了**用户体验 (UX)**。
-  我存到本地了   通过 `localStorage` 做了本地**持久化存储**。
-  表格翻页是用后端的   没有采用全量加载，而是做了**服务端分页 (Server-side Pagination)**，减轻了渲染压力。

### 3. 万能太极句式（遇到没把握、或者被挑刺时用的模板）
- **遇到被问为什么不这样写？（比如为啥不用 WebSocket）**：
   考虑到目前的**业务场景和并发量**，我们在两种方案之间做了一个 **Trade-off（权衡）**。早期更追求**敏捷开发（快速出东西）**，所以采用了现有的方案，后面有时间一定会往那方面重构。
- **遇到被问这里会不会有性能问题？**：
   目前在 **MVP（最小可行性产品）** 阶段，性能瓶颈还不明显；但如果数据量激增，我们会考虑引入 **虚拟列表 (Virtual List) / 更深度的缓存策略** 来做性能层面的优化。
- **遇到纯粹不懂的底层盲区**：
   这部分的**底层具体实现**我主要依赖了生态内比较成熟的工业级库（比如 axios / vuedraggable / KaTeX），站在了巨人的肩膀上，把主要精力花在了**业务逻辑抽象**和**状态流转**上。
